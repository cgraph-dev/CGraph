## Milestone Summary

**Key Decisions:**

- Phoenix context-based DDD with 57 bounded contexts for clean domain separation
- Guardian + JOSE for JWT with family tracking, rotation, and theft detection
- Post-quantum E2EE (PQXDH + ML-KEM-768) with backward-compatible X3DH fallback
- Postgres-native webhook idempotency (unique constraint + ON CONFLICT) over distributed locks
- Session tokens hashed before storage; OAuth bypasses 2FA by design
- PgBouncer as app sidecar (not separate Fly.io process) — matches Alloy pattern
- Argon2 tuned (t_cost=2, m_cost=15) for ~100-150ms auth latency
- Client-assisted CRDT compaction via Oban + PubSub (avoids Yjs NIF)
- Per-file random AES-256-GCM keys wrapped with session ratchet key
- Member-specific channel overrides trump role overrides (Discord permission model)

**Issues Resolved:**

- Payout race condition (FOR UPDATE row locks)
- Information leakage via inspect(reason) in 30 controller responses
- Apple JWS + Google RTDN unverified webhook payloads
- WebSocket topic mismatch blocking all group messaging (channel:_ vs group:_)
- Repo.get! causing 500 crashes on invalid user input (11 locations)
- Auth p95 latency exceeding 300ms SLO (Argon2 defaults too expensive)
- Elixir/OTP version mismatch between Dockerfile and .tool-versions
- CRDT state growing unbounded (compact_updates was a no-op)

**Issues Deferred:**

- 5 pre-existing test failures (E2EE, Phase12, Payout aggregate, forums_extended)
- Overall test coverage 33.8% (revenue modules well-covered at 94-100%)
- 6 controllers > 400 LOC (refactor candidates)
- External security audit ($25K-$120K budget not allocated)
- Database sharding (PostgreSQL handles current scale)
