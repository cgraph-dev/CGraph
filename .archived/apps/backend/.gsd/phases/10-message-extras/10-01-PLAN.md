# Plan 10-01: Message Forwarding — Full Stack

## Context

Web already has a forward UI (`forward-message-modal.tsx`, 276 lines) and handler in
`useConversationActions.ts` that does client-side "fake forwarding" — it re-sends message content
with `{ forwarded: true, originalMessageId }` in metadata. This works but has no server-side
tracking.

Mobile has no forward UI at all.

This plan adds proper backend support and wires mobile.

## Existing Code to Modify

- `apps/backend/lib/cgraph/messaging/message.ex` — add forwarded_from fields to schema
- `apps/backend/lib/cgraph/messaging/core_messages.ex` — add forward_message function
- `apps/backend/lib/cgraph_web/controllers/api/v1/message_controller.ex` — add forward action
- `apps/backend/lib/cgraph_web/router/` — add forward route
- `apps/web/src/modules/chat/hooks/useConversationActions.ts` — replace client-side forward with API
  call
- `apps/web/src/modules/chat/components/message-bubble/message-bubble.tsx` — add forwarded
  attribution
- `apps/mobile/src/components/conversation/message-actions-menu.tsx` — add Forward action

## Tasks

### Backend

1. **Migration: add forwarded_from fields to messages**
   - `forwarded_from_id` (uuid, FK to messages, nullable) — the original message
   - `forwarded_from_user_id` (uuid, FK to users, nullable) — original sender (denormalized for
     display)
   - Index on `forwarded_from_id`

2. **Update Message schema**
   - Add `forwarded_from_id`, `forwarded_from_user_id` fields
   - Add `belongs_to :forwarded_from, Message` association
   - Add `belongs_to :forwarded_from_user, User`
   - Include in changeset cast list

3. **Create `forward_message/3` in core_messages.ex**
   - Params: `(user, original_message_id, target_conversation_ids)`
   - Validate: user has access to original message's conversation
   - For each target conversation: create a new message with:
     - `content` = original message content
     - `forwarded_from_id` = original message ID
     - `forwarded_from_user_id` = original sender ID
     - `sender_id` = forwarding user ID
   - Broadcast via channel to each target conversation
   - Return list of created messages

4. **Add forward action to message_controller.ex**
   - `POST /api/v1/messages/:id/forward`
   - Body: `{ conversation_ids: [uuid, ...] }`
   - Validate: max 5 targets per forward (prevent spam)
   - Returns: `{ forwarded_count: N }`

5. **Add route**
   - `post "/messages/:id/forward", MessageController, :forward`

6. **Include forwarded_from in message JSON**
   - Update `message_json.ex` to include `forwarded_from_id`, `forwarded_from_user_id`,
     `forwarded_from_user_name` in response

### Shared Types

7. **Create `packages/shared-types/src/messages.ts`**
   ```typescript
   export interface ForwardedMessageInfo {
     originalMessageId: string;
     originalSenderId: string;
     originalSenderName: string;
   }
   ```
   Export from package index.

### Web

8. **Update forward handler to use API**
   - In `useConversationActions.ts`, replace the client-side re-send pattern with
     `POST /api/v1/messages/:id/forward`
   - The `forward-message-modal.tsx` already handles conversation selection — just change the submit
     handler

9. **Add forwarded attribution to message bubble**
   - In message-bubble.tsx, check for `forwarded_from_id` or `forwarded_from_user_id`
   - Show "↪ Forwarded from [Name]" banner above the message content
   - Small text, muted color, with forward icon

### Mobile

10. **Add Forward action to message-actions-menu**
    - Add "Forward" option to the long-press action menu
    - Pass `onForward` callback prop

11. **Create `forward-message-modal.tsx`**
    - Bottom sheet with conversation search/picker
    - Multi-select conversations (max 5)
    - Message preview at top
    - "Forward" button calls API
    - Follow existing mobile modal patterns

12. **Add forwarded attribution to mobile message bubble**
    - Same "↪ Forwarded from [Name]" banner as web
    - Adapt styling for mobile

- Forward a message from mobile → same result
