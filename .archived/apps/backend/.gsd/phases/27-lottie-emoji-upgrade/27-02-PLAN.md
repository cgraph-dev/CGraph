<objective>
Build the Noto Emoji Animation manifest and seeding infrastructure.

Purpose: The Google Fonts CDN hosts animated Noto Emoji as Lottie JSON files at predictable URLs:
`https://fonts.gstatic.com/s/e/notoemoji/latest/{codepoint}/lottie.json`. Not every Unicode emoji
has a Noto animation (~400+ do). This plan creates: (1) a scraper that probes the CDN to discover
which emojis have animations, (2) a manifest JSON file mapping all animated codepoints to their CDN
URLs, (3) a Mix task to seed the `lottie_assets` table, and (4) enrichment of the existing
emoji_unicode16.json with animation URLs.

Output: Noto Emoji manifest JSON, scraper module, seed task, enriched emoji dataset. </objective>

  Create `CGraph.Animations.NotoScraper` that verifies which Unicode emojis have Noto animations.

**How the CDN works (verified):**

- Base URL: `https://fonts.gstatic.com/s/e/notoemoji/latest/{codepoint}/`
- Available formats if animated: `lottie.json` (HTTP 200, ~37KB), `512.webp` (200), `512.gif` (200)
- Non-animated emojis return HTTP 404
- Codepoint format: lowercase hex, no `U+` prefix (e.g., `1f600` for 😀)
- Multi-codepoint emojis use `_` separator (e.g., flag 🏴‍☠️ = `1f3f4_200d_2620_fe0f`)

**Implementation:**

```elixir
defmodule CGraph.Animations.NotoScraper do
  @cdn_base "https://fonts.gstatic.com/s/e/notoemoji/latest"

  def verify_codepoint(codepoint) do
    url = "#{@cdn_base}/#{codepoint}/lottie.json"
    case Req.head(url, receive_timeout: 5_000) do
      {:ok, %{status: 200} = resp} ->
        size = get_content_length(resp)
        {:ok, %{codepoint: codepoint, lottie_url: url, file_size: size}}
      {:ok, %{status: 404}} -> {:error, :not_found}
      {:error, reason} -> {:error, reason}
    end
  end

  def scrape_all(codepoints, opts \\ []) do
    concurrency = Keyword.get(opts, :concurrency, 10)
    # Use Task.async_stream with controlled concurrency to avoid hammering CDN
    codepoints
    |> Task.async_stream(&verify_codepoint/1, max_concurrency: concurrency, timeout: 30_000)
    |> Enum.reduce([], fn
      {:ok, {:ok, result}}, acc -> [result | acc]
      _, acc -> acc
    end)
  end

    # Scrape CDN for each
    # Return list of animated entries
  end

  def emoji_to_hex(emoji) do
    emoji
    |> String.to_charlist()
    |> Enum.map(&Integer.to_string(&1, 16))
    |> Enum.map(&String.downcase/1)
    |> Enum.join("_")
  end
end
```

  Create a comprehensive `noto_emoji_manifest.json` by scraping the CDN. This is a one-time
  generation step (re-run when Google adds new animated emojis).

**Manifest structure:**

```json
{
  "version": "1.0.0",
  "generated_at": "2026-03-07T00:00:00Z",
  "cdn_base": "https://fonts.gstatic.com/s/e/notoemoji/latest",
  "total_animated": 423,
  "emojis": [
    {
      "codepoint": "1f600",
      "emoji": "😀",
      "name": "grinning face",
      "category": "Smileys & Emotion",
      "subcategory": "face-smiling",
      "keywords": ["grin", "happy", "smile"],
      "formats": {
        "lottie": "1f600/lottie.json",
        "webp": "1f600/512.webp",
        "gif": "1f600/512.gif"
      },
      "file_size_bytes": 37328
    }
  ]
}
```

Store relative paths in `formats` — the CDN base URL is stored once at top level. Clients construct
full URLs as `cdn_base + "/" + formats.lottie`.

**Process:**

1. Load all emojis from `emoji_unicode16.json`
2. Convert each to hex codepoint(s)
3. HEAD request to CDN to check availability
4. For available ones, GET lottie.json to extract file size
5. Write manifest to `priv/data/noto_emoji_manifest.json`

**Alternatively** (for initial version): Manually curate the most common animated emojis from the
Noto Emoji Animation website categories (Smileys, People, Animals, Food, Travel, Activities,
Objects, Symbols, Flags — at least the top ~50 per category). This avoids hammering the CDN and
gives us a known-good initial dataset. </action> <verify>`priv/data/noto_emoji_manifest.json`
exists, is valid JSON, contains 100+ entries.</verify> <done>Noto Emoji manifest covers all known
animated emojis from the CDN.</done> </task>

```elixir
defmodule Mix.Tasks.Cgraph.Lottie.Seed do
  use Mix.Task

  @shortdoc "Seed the lottie_assets table from the Noto Emoji manifest"

  def run(_args) do
    Mix.Task.run("app.start")
    manifest = load_manifest()
    cdn_base = manifest["cdn_base"]

    results = Enum.map(manifest["emojis"], fn emoji ->
      attrs = %{
        codepoint: emoji["codepoint"],
        emoji: emoji["emoji"],
        name: emoji["name"],
        category: emoji["category"],
        subcategory: emoji["subcategory"],
        keywords: emoji["keywords"] || [],
        lottie_url: cdn_base <> "/" <> emoji["formats"]["lottie"],
        webp_url: cdn_base <> "/" <> emoji["formats"]["webp"],
        gif_url: cdn_base <> "/" <> emoji["formats"]["gif"],
        file_size: emoji["file_size_bytes"],
        asset_type: "emoji",
        source: "noto"
      }

      CGraph.Animations.Lottie.upsert(attrs)
    end)

    {ok, err} = Enum.split_with(results, &match?({:ok, _}, &1))
    Mix.shell().info("Seeded #{length(ok)} animated emojis (#{length(err)} errors)")
  end
end
```
