## Context

**Coin Shop:** Backend has `CoinShopController` with:

- `GET /api/v1/coins/bundles` — lists available coin bundles with Stripe prices
- `POST /api/v1/coins/checkout` — creates Stripe checkout session for coin purchase

The `CoinBundles` module will be fixed in Phase 20 (compile-time env issue). After that fix, these
endpoints return real Stripe price data.

Neither web nor mobile has a coin shop service. The mobile has IAP service infrastructure but it
isn't connected to the coin shop specifically.

**AI Features:** Backend has AI endpoints (summarize, smart replies, moderate, sentiment analysis)
but no frontend service calls them. These are value-add features that enhance the chat experience.

## Tasks

1. **Create web coin shop service**
   - `coinShopService.ts`: getBundles(), checkout(bundleId)
   - Checkout redirects to Stripe checkout URL returned by backend

2. **Create web coin shop store**
   - State: bundles[], loading, error
   - Action: fetchBundles, initiateCheckout

3. **Wire web gamification shop page**
   - Display real bundles with prices
   - Purchase button → Stripe checkout flow

4. **Wire mobile coin shop**
   - Update gamification service with coin shop endpoints
   - For iOS/Android: bridge to IAP for native purchase flow
   - Display bundle options with native pricing

5. **Create web AI service**
   - `aiService.ts`: summarize(), smartReply(), moderate(), analyzeSentiment()
   - Each calls corresponding `/api/v1/ai/*` endpoint

6. **Create mobile AI service**
   - `ai.ts`: same surface as web
   - Wire to chat/messaging features for smart replies and summarization

7. **Create AI hooks**
   - `useAI()` hook for both web and mobile
   - Manages loading state for async AI calls
   - Error handling with user-friendly messages
