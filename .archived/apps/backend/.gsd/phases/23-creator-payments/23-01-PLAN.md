## Context

Backend has a complete creator monetization API:

- `POST /api/v1/creator/onboard` — Start creator account
- `GET /api/v1/creator/status` — Creator status + settings
- `GET /api/v1/creator/balance` — Current balance (earnings.ex)
- `POST /api/v1/creator/payout` — Request payout (payout.ex, fixed in Phase 20)
- `GET /api/v1/creator/analytics` — Earnings analytics
- `POST /api/v1/creator/:id/subscribe` — Subscribe to creator
- `DELETE /api/v1/creator/:id/subscribe` — Unsubscribe
- Controller: `lib/cgraph_web/controllers/api/v1/creator_controller.ex`

Neither web nor mobile has any service layer for these endpoints. The Creator pages exist in both
apps but either show static content or are empty shells.

## Tasks

1. **Create web creator service**
   - New file: `modules/creator/services/creatorService.ts`
   - Typed request/response for each endpoint
   - Use shared HTTP client (apiClient or equivalent)

2. **Create web creator store**
   - New file: `modules/creator/stores/creatorStore.ts`
   - Zustand (or project's chosen state lib) store with:
     - State: isCreator, balance, payoutHistory, subscribers, analytics, loading, error
     - Actions: fetchStatus, fetchBalance, requestPayout, fetchAnalytics

3. **Create web creator hooks**
   - `useCreator()` — wraps store for status/onboarding
   - `useCreatorDashboard()` — wraps store for dashboard data + actions

4. **Wire web Creator Dashboard page**
   - Import `useCreatorDashboard()`
   - Replace any static/placeholder content with real data
   - Balance display, subscriber count, analytics charts, payout button

5. **Create mobile creator service**
   - New file: `services/creator.ts` (following existing service patterns like `services/forum.ts`)
   - Same endpoint coverage as web

6. **Wire mobile Creator Dashboard screen**
   - Use creator service in Creator Dashboard screen
   - Display real balance, analytics, payout functionality
