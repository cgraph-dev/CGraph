<task type="auto">
  <name>Task 1: Generate route manifest and identify errors</name>
  <files>
    apps/backend/route-audit.md
  </files>
  <action>
    1. Compile the backend to verify all route modules load:
       `cd apps/backend && mix compile --warnings-as-errors 2>&1`
       If compilation fails, fix the errors — a route that doesn't compile is a blocker.

    2. Generate the full route listing:
       `cd apps/backend && mix phx.routes CgraphWeb.Router 2>&1`
       Save the output. This lists every route with method, path, and controller#action.

    3. Start the backend in test/dev mode:
       `cd apps/backend && MIX_ENV=dev mix phx.server &`
       Wait for "Access CgraphWeb.Endpoint at http://localhost:4000"

    4. Test health endpoints (MUST all return 200):
       ```
       curl -s -o /dev/null -w "%{http_code}" http://localhost:4000/health
       curl -s -o /dev/null -w "%{http_code}" http://localhost:4000/ready
       ```

    5. Test auth endpoints with minimal payloads (expect 4xx for invalid data, NOT 500):
       ```
       curl -s -o /dev/null -w "%{http_code}" -X POST http://localhost:4000/api/v1/auth/register -H "Content-Type: application/json" -d '{}'
       curl -s -o /dev/null -w "%{http_code}" -X POST http://localhost:4000/api/v1/auth/login -H "Content-Type: application/json" -d '{}'
       curl -s -o /dev/null -w "%{http_code}" -X POST http://localhost:4000/api/v1/auth/refresh -H "Content-Type: application/json" -d '{}'
       curl -s -o /dev/null -w "%{http_code}" -X POST http://localhost:4000/api/v1/auth/forgot-password -H "Content-Type: application/json" -d '{}'
       ```
       A 400/401/422 is PASS (endpoint works, rejects bad input). A 500 is FAIL.

    6. Test public endpoints:
       ```
       curl -s -o /dev/null -w "%{http_code}" http://localhost:4000/api/v1/tiers
       curl -s -o /dev/null -w "%{http_code}" http://localhost:4000/api/v1/emojis
       ```

    7. Test a protected endpoint without auth (should be 401, NOT 500):
       ```
       curl -s -o /dev/null -w "%{http_code}" http://localhost:4000/api/v1/me
       ```

    8. Create `apps/backend/route-audit.md` documenting:
       - Total routes from `mix phx.routes`
       - Health endpoint results (status codes)
       - Auth endpoint results (status codes)
       - Public endpoint results (status codes)
       - Protected endpoint results (status codes)
       - Any 500 errors found (controller, action, error message)

    9. Stop the dev server after testing.

    NOTE: The backend requires PostgreSQL. If `mix phx.server` fails due to DB connection, ensure
    the database exists: `cd apps/backend && mix ecto.create && mix ecto.migrate`

  </action>
  <verify>
    Check route-audit.md exists: `cat apps/backend/route-audit.md | head -30`
    Health check: `curl -s -o /dev/null -w "%{http_code}" http://localhost:4000/health` returns 200
  </verify>
  <done>
    Route manifest exists at apps/backend/route-audit.md.
    Health and ready endpoints return 200.
    Auth endpoints return 4xx (not 500) for invalid input.
  </done>

<task type="auto">
  <name>Task 2: Fix critical route errors</name>
  <files>
    apps/backend/lib/cgraph_web/router/health_routes.ex
    apps/backend/lib/cgraph_web/router/auth_routes.ex
    apps/backend/lib/cgraph_web/router/public_routes.ex
    apps/backend/lib/cgraph_web/router/user_routes.ex
  </files>
  <action>
    Based on the route audit results from Task 1, fix any routes returning 500 errors in the
    critical path (health, auth, public).

    Common fixes for Phoenix 500 errors:
    - Missing controller action → add a stub action returning `json(conn, %{error: "not implemented"})`
      with a 501 status
    - Missing view/JSON serializer → ensure `render/2` or `json/2` is used correctly
    - Database query errors → check Ecto schema matches migration
    - Missing plug dependency → verify pipeline plugs exist and are loaded
    - Pattern match failures in controller → add fallback clauses

    For non-critical routes (admin, AI, gamification, forum) that return 500:
    - Document them in route-audit.md as "deferred to later phase"
    - Do NOT fix them now — they are out of scope for Phase 1

    For auth routes specifically:
    - POST /api/v1/auth/register must accept `{email, password, username}` and return
      422 with validation errors for invalid input (not 500)
    - POST /api/v1/auth/login must accept `{email, password}` and return
      401 for invalid credentials (not 500)
    - POST /api/v1/auth/refresh must accept `{refresh_token}` and return
      401 for invalid token (not 500)

    After fixes, re-run the audit curl commands to verify 500s are resolved.

  </action>
  <verify>

    4xx for invalid input. No 500 errors on critical path routes.
    Run: `cd apps/backend && mix test test/cgraph_web/controllers/ --max-failures 5 2>&1 | tail -20`
  </verify>
  <done>
    All health endpoints return 200.
    All auth endpoints return proper 4xx for invalid input.
    No 500 errors on health, auth, or public route paths.
    route-audit.md updated with final results.
  </done>
