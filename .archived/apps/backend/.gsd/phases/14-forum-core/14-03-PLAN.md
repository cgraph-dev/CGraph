# Plan 14-03: Web Forum Wiring — Search, Categories, Comments, Store Gaps

## Context

The web forum frontend is **massive** (~43,000 lines across 376 files) and largely functional.
However, the infrastructure analysis revealed several store-level gaps where UI components exist but
store actions are missing:

1. **Search** — `forum-search/` component (248L) has full UI with filters, sort, history, keyboard
   nav — but takes a callback `onSearch` that isn't wired to any store action. No search results
   page or route exists.

2. **Category CRUD** — Category types exist in both type systems and `forum-category-list/`
   component (291L) renders categories. But the store has NO category management actions (create,
   update, delete, reorder). The backend has `categories.ex` (85L) with full CRUD + reorder.

3. **Comment mutations** — Socket hooks receive `onCommentEdited` and `onCommentDeleted` events, but
   there are no REST store actions to _initiate_ edits or deletes. Comments can only be created, not
   edited/deleted from the UI.

4. **Create forum payload** — `CreateForumData` only has `name`, `description`, `isNsfw`,
   `isPrivate` — no `categoryId` or `ForumSettings` fields. The backend accepts much more.

5. **Thread prefixes fetch** — `fetchThreadPrefixes` returns hardcoded defaults instead of API data.

**What exists (web store):**

- `forumStore.core.ts` (283L) — fetch/create posts, comments, voting
- `forumStore.features.ts` (291L) — prefixes, ratings, attachments, polls, subscriptions
- `forumStore.forumCrud.ts` (~140L) — create/update/delete forum, subscribe, leaderboard
- `forumStore.moderation.ts` (~200L) — pin/lock/delete/move, multi-quote
- `forumStore.admin.ts` (258L) — user groups, warnings, bans, mod queue
- `forumHosting-actions.ts` (306L) — board/thread listing for board view

## Tasks

### Task 1: Add search actions and hook

**Files:** `apps/web/src/modules/forums/store/forumStore.types.ts`,
`apps/web/src/modules/forums/store/forumStore.core.ts`,
`apps/web/src/modules/forums/hooks/useForumSearch.ts`

**Actions:**

1. Add to `ForumState` types:
   ```typescript
   // Search types
   interface ForumSearchResult {
     type: 'thread' | 'post' | 'comment';
     id: string;
     title?: string;
     contentPreview: string;
     author: { id: string; username: string; avatar?: string };
     forum: { id: string; name: string; slug: string };
     board?: { id: string; name: string; slug: string };
     score: number;
     rank: number;
     createdAt: string;
     highlights?: string[];
   }
   interface ForumSearchFilters {
     type?: 'thread' | 'post' | 'comment' | 'all';
     forumId?: string;
     boardId?: string;
     authorId?: string;
     dateFrom?: string;
     dateTo?: string;
     sort?: 'relevance' | 'newest' | 'oldest' | 'most_votes';
   }
   // Add to ForumState:
   searchResults: ForumSearchResult[];
   searchQuery: string;
   searchFilters: ForumSearchFilters;
   searchLoading: boolean;
   searchHasMore: boolean;
   searchCursor?: string;
   searchForums: (query: string, filters?: ForumSearchFilters) => Promise<void>;
   searchMore: () => Promise<void>;
   clearSearch: () => void;
   ```
2. Implement `searchForums` in `forumStore.core.ts`:
   - GET `/api/v1/search/forums?q=${query}&type=${filters.type}&forum_id=${filters.forumId}...`
   - Set `searchResults`, update `searchCursor`, `searchHasMore`
   - `searchMore` appends using cursor pagination
   - `clearSearch` resets all search state
3. Create `useForumSearch.ts` hook:
   - Debounce search input (300ms)
   - Sync search query + filters with URL search params (`?q=...&type=...`)
   - Return `{ results, loading, hasMore, filters, setFilters, search, loadMore, clear }`
