# Plan 14-02: Full-Text Search + Reputation Propagation

## Context

### Search Gap

The existing `search.ex` (63 lines) only searches reddit-style `Post` entities using `ILIKE` pattern
matching. This has critical limitations:

- No relevance ranking (ILIKE returns boolean match, not ranked results)
- No stemming (searching "running" won't match "run")
- No phrase search capability
- **No search for Thread or ThreadPost** (the MyBB-style content ‚Äî the primary forum content type)
- No search suggestions or faceted filtering

The `SearchController` (261L) has a `search_posts` action but no dedicated forum search endpoint.
The web frontend has a full `forum-search/` component (248L) with filters, sort, history, and
keyboard navigation ‚Äî but it takes an `onSearch` callback that isn't wired to any store action. The
mobile has `searchForums()` in its search service.

### Reputation Gap

The voting system is comprehensive ‚Äî `voting.ex` (148L), `forum_voting.ex` (225L), `thread_posts.ex`
vote functions all correctly update `score`, `upvotes`, `downvotes` on the voted entity (post,
comment, thread). However, **none of them update the content author's reputation**. `ForumMember`
has `reputation`, `reputation_positive`, `reputation_negative` fields that are **never written to**
by any voting context.

**What exists:**

- `search.ex` (63L) ‚Äî ILIKE-only on Post, no Thread/ThreadPost search
- `search_controller.ex` (261L) ‚Äî Global search with `/search/posts`, but no forum-specific search
- `voting.ex` (148L) ‚Äî `vote_on_post/3`, `vote_on_comment/3` ‚Äî updates entity score only
- `forum_voting.ex` (225L) ‚Äî Forum-level voting with eligibility checks
- `thread_posts.ex` (250L) ‚Äî `vote_thread/3`, `vote_post_by_id/3` ‚Äî updates score on thread/post
- `forum_member.ex` (160L) ‚Äî Has `reputation`, `reputation_positive`, `reputation_negative` fields
  (never written)
- `members.ex` (331L) ‚Äî Membership management, no reputation update functions

## Tasks

### Task 1: Create migration for tsvector columns and GIN indexes

**Files:** `apps/backend/priv/repo/migrations/TIMESTAMP_add_forum_fulltext_search.exs`

**Actions:**

1. Generate migration with `mix ecto.gen.migration add_forum_fulltext_search`
2. Add `search_vector` tsvector column to:
   - `forum_threads` ‚Äî weighted: title (A) + content (B)
   - `forum_thread_posts` ‚Äî weighted: content (B)
   - `forum_posts` ‚Äî weighted: title (A) + content (B)
   - `forum_comments` ‚Äî weighted: content (C)
3. Create GIN indexes on each `search_vector` column
4. Create trigger functions to auto-update tsvector on INSERT/UPDATE:
   ```sql
   CREATE OR REPLACE FUNCTION forum_threads_search_vector_update() RETURNS trigger AS $$
   BEGIN
     NEW.search_vector := setweight(to_tsvector('english', coalesce(NEW.title, '')), 'A') ||
                          setweight(to_tsvector('english', coalesce(NEW.content, '')), 'B');
     RETURN NEW;
   END;
   $$ LANGUAGE plpgsql;
   ```
5. Add triggers to all four tables
6. Backfill existing data: `UPDATE forum_threads SET search_vector = setweight(...)`

### Task 2: Rewrite search.ex with full-text search

**Files:** `apps/backend/lib/cgraph/forums/search.ex`

**Actions:**

1. Replace ILIKE with tsvector search:
   ```elixir
   def search_threads(query, opts \\ []) do
     tsquery = to_tsquery(query)

     from(t in Thread,
       where: fragment("? @@ ?", t.search_vector, ^tsquery),
       order_by: [desc: fragment("ts_rank(?, ?)", t.search_vector, ^tsquery)],
       select_merge: %{rank: fragment("ts_rank(?, ?)", t.search_vector, ^tsquery)}
     )
     |> apply_filters(opts)
     |> CursorPagination.paginate(opts)
   end
   ```
2. Add `search_thread_posts/2` ‚Äî search within thread replies
3. Add `search_posts/2` ‚Äî search reddit-style posts (upgrade existing)
4. Add `search_comments/2` ‚Äî search comment content
5. Add `search_all/2` ‚Äî unified search across all entity types, returning mixed results with type
   indicators
6. Support filters: `forum_id`, `board_id`, `author_id`, `date_from`, `date_to`, `has_poll`,
   `is_pinned`
7. Support sort: `relevance` (default, ts_rank), `newest`, `oldest`, `most_votes`
8. Handle search query parsing: strip special chars, handle phrases in quotes, OR with `|`
9. Private helper `to_tsquery/1` that converts user input to safe tsquery:
   ```elixir
   defp to_tsquery(query) do
     query
     |> String.trim()
     |> String.replace(~r/[^\w\s"|-]/, "")
     |> String.split(~r/\s+/)
     |> Enum.join(" & ")
     |> then(&fragment("to_tsquery('english', ?)", ^&1))
   end
   ```

### Task 4: Reputation propagation from votes

**Files:** `apps/backend/lib/cgraph/forums/members.ex`, `apps/backend/lib/cgraph/forums/voting.ex`,
`apps/backend/lib/cgraph/forums/thread_posts.ex`

**Actions:**

1. Add to `members.ex`:
   ```elixir
   def update_reputation(forum_id, user_id, delta) when delta in [1, -1] do
     field = if delta > 0, do: :reputation_positive, else: :reputation_negative

     from(m in ForumMember,
       where: m.forum_id == ^forum_id and m.user_id == ^user_id
     )
     |> Repo.update_all(inc: [{field, 1}, {:reputation, delta}])
   end
   ```
2. In `voting.ex` `vote_on_post/3`:
   - After successfully creating/updating vote, call
     `Members.update_reputation(forum_id, post.author_id, vote_value)`
   - On vote removal, reverse:
     `Members.update_reputation(forum_id, post.author_id, -old_vote_value)`
   - On vote change (up‚Üídown), apply delta:
     `Members.update_reputation(forum_id, post.author_id, -2)` (reverse old + apply new)
3. Same pattern in `voting.ex` `vote_on_comment/3`
4. Same pattern in `thread_posts.ex` `vote_thread/3` and `vote_post_by_id/3`:
   - Look up thread/post author, resolve forum_id from board, call `Members.update_reputation/3`
5. Self-vote prevention: existing `forum_voting.ex` already prevents self-votes; ensure voting.ex
   does too
