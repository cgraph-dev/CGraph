<objective>
Replace the remove-from-list deletion behavior with soft-delete placeholder rendering on both web and mobile, so recipients see a "message deleted" indicator instead of the message silently vanishing.
Purpose: Meets SC-2 — users see a clear deletion indicator, maintaining conversation context and timeline integrity.
Output: Both platforms render a non-interactive placeholder for deleted messages; message ordering preserved.
</objective>

<context>
**What exists:**
- Backend already does soft-delete: sets `deleted_at` timestamp + `deleted_for_everyone` flag, broadcasts `message_deleted` with message id
- Web: `deleteMessage` in chatStore.message-ops.ts sends DELETE; `message_deleted` handler in conversationChannel.ts calls `removeMessage()` which filters message out of array
- Mobile: `deleteMessage` in chatStore.ts sends DELETE; `message_deleted` event handler filters message from list
- Shared types: `Message` has `deletedAt?: string` field already

**Key references:**

- Web message bubble: apps/web/src/modules/chat/components/message-bubble/message-bubble.tsx
  (renders message content)
- Web socket handler: apps/web/src/lib/socket/conversationChannel.ts (message_deleted handler)
- Web store ops: apps/web/src/modules/chat/store/chatStore.message-ops.ts (removeMessage mutation)
- Mobile store: apps/mobile/src/stores/chatStore.ts (message_deleted handler + deleteMessage action)
- Mobile message bubble:
  apps/mobile/src/screens/messages/conversation-screen/components/message-bubble.tsx </context>
