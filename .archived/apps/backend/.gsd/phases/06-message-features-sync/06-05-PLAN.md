<objective>
Wire the mobile chatStore to read from and write to WatermelonDB for all message operations, bridging the existing sync infrastructure with the in-memory Zustand store so messages persist and sync across devices.
Purpose: Completes SC-5 — user reads messages on web, switches to mobile, and messages are synced via the existing WatermelonDB sync engine.
Output: chatStore reads from WatermelonDB on init, writes all mutations (send, edit, delete, receive) to WatermelonDB, and the sync engine keeps mobile in sync with server state.
</objective>

<context>
**Depends on 06-02 + 06-04 outputs:**
- Soft-delete now preserves messages with `deletedAt` set (06-02) — WatermelonDB must mirror this behavior
- Edit history is now recorded and typed (06-04) — WatermelonDB Message model needs `is_edited` column updates

**What exists (WatermelonDB infrastructure — all built, none wired):**

- Database singleton: apps/mobile/src/lib/database/index.ts
- Schema v1: apps/mobile/src/lib/database/schema.ts — messages table with columns: id,
  conversation_id, sender_id, content, message_type, sent_at, delivered_at, read_at, deleted_at,

- Backend: `SyncQueries` module with `list_user_messages_since/2`,
  `list_deleted_message_ids_since/2`

- No code path reads from or writes to WatermelonDB.
- The sync engine runs independently but chatStore doesn't consume its data.
