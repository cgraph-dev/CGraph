# Plan 19-01: Wallet Auth Polish

## Objective

Polish existing Ethereum wallet authentication to production quality: upgrade challenge messages to
SIWE (EIP-4361) standard and add WalletConnect v2 multi-wallet support on both web and mobile. After
this plan, wallet auth supports MetaMask, WalletConnect (100+ wallets), and Coinbase Wallet with
industry-standard SIWE messages.

## AUTH-12 (Privy) — DESCOPED

Privy was descoped from Phase 19 because CGraph already has a complete custom auth infrastructure:

- **OAuth**: Google, Apple, Facebook, TikTok — all implemented and working
- **Wallet**: EIP-191 signature verification backend + MetaMask web UI (this plan polishes it)
- **Sessions**: JWT access/refresh tokens via Guardian
- **2FA**: TOTP-based two-factor authentication
- **E2EE**: Custom identity chain for end-to-end encryption (Privy can't integrate with this)

Adding Privy would introduce: vendor lock-in ($99-299/mo), two parallel auth systems to maintain,
and incompatibility with CGraph's custom E2EE identity chain. The ROI is negative.

## Requirements Covered

- **AUTH-13**: Ethereum wallet signature auth — POLISH (SIWE upgrade, WalletConnect, mobile wallet)

## Current State

- **AUTH-13**: ~85% — Full EIP-191 backend (`wallet_authentication.ex`, 10 endpoints in
  `wallet_auth_controller.ex`) + MetaMask web UI (`useLoginForm.ts`). Missing: SIWE standard message
  format, WalletConnect v2 support, mobile wallet connection, wallet state persistence.

## Wave 1 Execution Order

1. T1 → T2 (backend SIWE upgrade, sequential)
2. T8 (shared types, parallel with backend)
3. T3 → T4 → T5 (web WalletConnect, after backend SIWE ready)
4. T6 → T7 (mobile WalletConnect, parallel with web)

## Verification

```bash
# Backend: SIWE challenge format
curl -X POST http://localhost:4000/api/v1/auth/wallet/challenge \
  -H "Content-Type: application/json" \
  -d '{"address": "0x..."}'
# Returns: SIWE-formatted message with domain, nonce, issued-at, expiration

# Backend: SIWE signature verification
curl -X POST http://localhost:4000/api/v1/auth/wallet/verify \
  -H "Content-Type: application/json" \
  -d '{"address": "0x...", "signature": "0x...", "message": "<siwe_message>"}'
# Returns: {"access_token": "...", "refresh_token": "..."}

# Web: Login page shows MetaMask + WalletConnect + Coinbase Wallet options
# Web: WalletConnect QR code opens, connects to mobile wallet, signs SIWE message
# Mobile: "Connect Wallet" opens WalletConnect modal, deep-links to wallet app
```
