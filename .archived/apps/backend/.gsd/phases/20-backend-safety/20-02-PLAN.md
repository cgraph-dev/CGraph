## Context

21 controllers use `inspect(reason)` in error responses, leaking Ecto changeset errors, Stripe API
errors, and internal state to users. This is both a security risk (information disclosure) and poor
UX.

33 locations use `Repo.get!` which raises `Ecto.NoResultsError` (500) on missing records. Most are
fine for internal use, but at least 3 receive user-controlled IDs and must gracefully handle missing
records.

`coin_bundles.ex` uses `System.get_env` at compile time via module attributes, meaning all Stripe
price IDs are nil when the Docker image is built in CI (where Stripe env vars aren't set).

`@tier_mapping` in stripe_webhook_controller.ex (L333, L342) is defined but either unused or has
stale mappings — it should be removed or fixed.

`Earnings.get_balance/1` (L54-55) does a Repo.get then accesses .balance — not atomic under
concurrent balance changes.

## Tasks

1. **Create safe error response helper**
   - Add `CgraphWeb.ErrorHelpers.safe_error_message/1` that maps known error atoms to user-friendly
     strings
   - Unknown errors map to "An unexpected error occurred"
   - Log original error via `Logger.error`

2. **Replace inspect(reason) in all 21 controllers**
   - Search: `grep -rn "inspect(reason)\|inspect(error)\|inspect(err)" lib/cgraph_web/controllers/`
   - Replace each with call to safe error helper
   - Ensure `Logger.error("#{action} failed: #{inspect(reason)}")` preserves debugging

3. **Fix critical Repo.get! locations**
   - `paid_subscription.ex:36` — user subscribes to creator by ID
   - `creator_controller.ex:111` — user views creator profile by ID
   - `coin_checkout.ex:143` — user purchases bundle by ID
   - Pattern: `case Repo.get(Schema, id) do nil -> {:error, :not_found}; record -> ... end`

4. **Review remaining 30 Repo.get! locations**
   - Categorize as user-input (must fix) vs system-internal (safe to keep with comment)
   - Fix all user-input ones, comment all system-internal ones

5. **Fix CoinBundles compile-time env**
   - Move `System.get_env("STRIPE_...")` from module attributes to function body using
     `Application.get_env(:cgraph, :stripe_coin_prices)`
   - Add runtime.exs config entry for coin bundle Stripe price IDs
   - Add startup validation that warns if price IDs are missing

6. **Clean up dead code**
   - Remove `@tier_mapping` from stripe_webhook_controller.ex if unused
   - If used, fix the mappings to match current Stripe plan IDs

7. **Fix Earnings.get_balance/1 atomicity**
   - Replace struct fetch + field access with single `Repo.one` query selecting balance
   - Or use `Repo.get` with `:lock` option in transaction contexts
