## Context

Revenue-critical code has **zero test coverage**:

- Creator monetization: 9 files + 1 controller, 0 tests
- Stripe webhook handler: 4 tests total, only testing signature verification
- IAP: no tests for receipt verification or purchase flow

`coveralls.json` exists in the repo but coverage isn't wired to CI.

These are the features that handle real money. Bugs here = lost revenue or incorrect charges.

## Tasks

1. **Write creator payout tests**
   - Test the `Repo.transaction` + `FOR UPDATE` flow from Phase 20
   - Test successful payout creates payout record + updates balance
   - Test concurrent race condition (two Tasks, only one succeeds)
   - Test insufficient balance rejection
   - Test minimum payout threshold

2. **Write creator earnings tests**
   - Test `get_balance/1` returns correct amount
   - Test balance calculation after multiple earnings entries
   - Test atomic balance read (Phase 20 fix)

3. **Write paid subscription tests**
   - Test subscribe to creator
   - Test duplicate subscription handling
   - Test unsubscribe
   - Test self-subscribe rejection
   - Test subscriber count after sub/unsub

4. **Write creator controller tests**
   - Full CRUD flow through HTTP endpoints
   - Auth required (test 401 without token)
   - Test each endpoint with valid and invalid params

5. **Expand Stripe webhook tests**
   - Add test for each event type the webhook handles
   - Verify database state changes after webhook processing
   - Test idempotency (same event ID processed twice)

6. **Write IAP controller tests**
   - Test Apple receipt verification (valid, tampered, expired)
   - Test Google purchase token verification (valid, invalid)
   - Test JWS/RTDN verification from Phase 20

7. **Establish coverage baseline**
   - Run `mix coveralls` and record baseline percentage
   - Configure coveralls.json for CI
   - Add coverage check to CI pipeline (fail if below baseline)
