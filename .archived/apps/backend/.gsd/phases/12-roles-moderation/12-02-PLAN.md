# Plan 12-02: Ban/Kick + Content Reporting — Group Moderation Pipeline

## Context

**Ban/Kick (GROUP-06):** Backend has full ban/kick infrastructure:

- `group_ban.ex` (41L): schema with reason, expires_at, banned_by
- `members.ex` (281L): `ban_member/3`, `ban_member/4`, `unban_user/2`, `list_bans/1`,
  `remove_member/1`
- `group_member_controller.ex` (317L): kick endpoint (`POST /members/:id/kick`), audit logging
- Web: `confirm-action-modal.tsx` (119L) for ban/kick confirmation
- Mobile: `group-members-screen.tsx` (257L) with basic member actions

**Gaps:** Kick endpoint has no reason param. No ban duration UI. No ban list management on mobile.

**Reporting (MOD-01/02):** Backend has comprehensive reporting:

- `report.ex` (79L): 13 categories, 6 target types
- `reports.ex` (321L): create_report with duplicate detection, priority scoring
- `enforcement.ex` (312L): review_report with 5 action types, user restrictions
- Web: `report-dialog.tsx` (297L) + moderation queue/log (296L)
- Mobile: `group-moderation-screen.tsx` (290L) basic moderation

**Gaps:** No "Report" button on group messages. Moderation is platform-admin scoped only, not
group-moderator scoped. No mobile report submission screen.

## Tasks

### Task 1: Enhance kick/ban with structured reasons and ban management

**Files:** `apps/backend/lib/cgraph/groups/members.ex`,
`apps/backend/lib/cgraph_web/controllers/api/v1/group_member_controller.ex`,
`apps/backend/lib/cgraph/groups/audit_log.ex`

**Action:**

1. In `group_member_controller.ex`:
   - Change `kick/2` to accept optional `"reason"` param from request body
   - Pass reason to audit log: change `AuditLog.log(:member_kicked, ...)` to include
     `%{reason: reason}` in metadata
   - Add `ban/2` action if not already exists:
     - Accept `reason` (string), `duration_hours` (integer, optional — nil = permanent)
     - Calculate `expires_at` from `duration_hours` if provided
     - Call `Members.ban_member/4` with moderator tracking
     - Log to audit as `:member_banned` with reason + duration
   - Add `list_bans/2`: `GET /groups/:id/bans` → returns active bans with banner info
   - Add `unban/2`: `DELETE /groups/:id/bans/:user_id` → calls `Members.unban_user/2`
2. In `members.ex`:
   - Ensure `ban_member/4` accepts `expires_at` param and stores it on `group_ban`
   - Add `list_active_bans/1` that filters by `expires_at IS NULL OR expires_at > now()`
3. Add routes in router: `POST /groups/:id/members/:member_id/ban`, `GET /groups/:id/bans`,
   `DELETE /groups/:id/bans/:user_id`

**Verify:** `mix compile` passes. Kick with reason → audit log has reason. Ban with duration →
group_ban has expires_at. List bans returns active bans only.

**Done:** Kick/ban accept reasons, ban supports expiry, ban list endpoint exists.

### Task 2: Build group-scoped moderation context and controller

**Files:** `apps/backend/lib/cgraph/groups/moderation.ex` (NEW),
`apps/backend/lib/cgraph_web/controllers/api/v1/group_moderation_controller.ex` (NEW)

**Action:**

1. Create `groups/moderation.ex`:
   - `list_group_reports/2` — query `reports` where `target_type` is "message" or "user" AND the
     target belongs to the group (join through messages.channel_id → channels.group_id, or through
     group_members). Accept filters: status (pending/reviewed/dismissed), page/limit.
   - `review_group_report/4` — (report_id, moderator_id, action, opts) — delegate to
     `Moderation.Enforcement.review_report/3` but verify the moderator is in the group with
     `manage_messages` or `ban_members` permission first.
   - `get_group_report/2` — get single report with preloads (reporter, target, review_actions)
   - `group_moderation_stats/1` — count pending reports, recent actions, for the group

2. Create `group_moderation_controller.ex`:
   - `index/2` — list group reports (paginated, filterable by status)
   - `show/2` — single report detail
   - `review/2` — take action on a report (action: dismiss/warn/remove_content/mute/ban + notes)
   - `stats/2` — moderation dashboard stats for the group
   - All actions require group membership + appropriate permission (manage_messages for content
     actions, ban_members for user actions)

3. Add routes:
   `resources "/groups/:group_id/moderation/reports", GroupModerationController, only: [:index, :show]` +
   `POST /groups/:group_id/moderation/reports/:id/review` + `GET /groups/:group_id/moderation/stats`

**Verify:** `mix compile` passes. List reports for a group returns only reports targeting content in
that group. Review action changes report status.

**Done:** Group-scoped moderation exists separate from platform-admin moderation.
