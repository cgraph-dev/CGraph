# Plan 09-02: DND Schedule UI & Timezone-Aware Quiet Hours

## Context

The backend already has `quiet_hours_enabled`, `quiet_hours_start`, `quiet_hours_end` fields on
`user_settings` with `in_quiet_hours?/1` logic that handles overnight wrapping. The
`should_notify?/2` function checks quiet hours before delivery. However:

1. **No UI exists** to configure quiet hours (web or mobile)
2. **No quick DND toggle** — only presence status picker has "dnd" option
3. **UTC-only** — `in_quiet_hours?` uses `Time.utc_now()`, not user's timezone
4. **No duration-based DND** (e.g., "mute for 2 hours")

## Existing Code to Modify

- `apps/backend/lib/cgraph/accounts/user_settings.ex` — add `timezone` and `dnd_until` fields
- `apps/backend/lib/cgraph/accounts/settings.ex` → `in_quiet_hours?/1` — use timezone
- `apps/web/src/modules/settings/components/panels/notification-settings-panel.tsx` — add DND
  section or link
- `apps/mobile/src/screens/settings/notification-settings-screen.tsx` — add DND section
- `apps/web/src/components/layout/` — add DND quick-toggle to nav
- Presence system — broadcast DND state changes

## Tasks

### Backend

1. **Add timezone + dnd_until fields to user_settings**
   - Migration: add `timezone` (string, default "UTC"), `dnd_until` (utc_datetime, nullable)
   - `timezone` — IANA timezone string (e.g., "America/New_York")
   - `dnd_until` — instant DND expiry (nil = not in instant DND)
   - Add to `UserSettings` schema and changeset

2. **Update `in_quiet_hours?/1` for timezone awareness**
   - Read `settings.timezone` (default "UTC")
   - Convert current UTC time to user's local time using `Calendar` or `Timex` (check if available,
     else use stdlib)
   - Compare local time against `quiet_hours_start`/`quiet_hours_end`
   - Also check `dnd_until`: if set and in future, suppress notifications regardless of schedule

3. **Update `should_notify?/2` for instant DND**
   - Add `dnd_until` check: if `dnd_until` is set and
     `DateTime.compare(DateTime.utc_now(), dnd_until) == :lt`, return false
   - This is independent of quiet hours — instant DND is a separate override

4. **Add DND duration endpoint**
   - `POST /api/v1/settings/dnd` — body: `{duration_minutes: number}` — sets
     `dnd_until = now + duration`
   - `DELETE /api/v1/settings/dnd` — clears `dnd_until`
   - `GET /api/v1/settings/dnd` — returns current DND state and expiry
   - Add to `settings_controller.ex`

5. **Write tests**
   - `in_quiet_hours?` with timezone conversion (EST 10pm = UTC 3am)
   - `dnd_until` suppression and expiry
   - API endpoint tests

### Web

6. **Create `dnd-schedule-panel.tsx`**
   - Toggle: "Scheduled Quiet Hours" (on/off)
   - Time pickers: start time, end time (24h format)
   - Timezone display: auto-detected from browser, editable
   - Save calls `PUT /api/v1/settings/notifications` with `quiet_hours_enabled`,
     `quiet_hours_start`, `quiet_hours_end`, `timezone`

7. **Create `dnd-toggle.tsx` (nav bar quick toggle)**
   - Bell/moon icon button in top nav bar (next to notifications bell)
   - Click: toggle DND immediately (calls `POST /api/v1/settings/dnd {duration_minutes: null}` for
     indefinite, or shows dropdown)
   - Dropdown options: "For 1 hour", "For 2 hours", "For 8 hours", "Until tomorrow morning", "Until
     I turn it off"
   - Active state: icon changes color/style, optional tooltip "DND until X"
   - Syncs with presence (set presence to "dnd" when active)

8. **Add DND section to notification settings panel**
   - Link to DND schedule configuration
   - Show current DND status

### Mobile

9. **Create `dnd-schedule-screen.tsx`**
   - Same config as web: toggle, time pickers, timezone
   - iOS-style time pickers (DateTimePicker)
   - Timezone auto-detected from device

10. **Create `dnd-quick-toggle.tsx`**
    - Accessible from notification settings or conversation list header
    - Bottom sheet with duration options (same as web dropdown)
    - Updates presence status to "dnd"

11. **Add DND to mobile notification settings navigation**
    - Link from notification settings screen to DND schedule screen

### Presence Integration

12. **Sync DND with presence system**
    - When DND is activated (quick toggle or quiet hours active), update user's presence status to
      "dnd"
    - When DND expires or is turned off, restore previous presence status
    - Other users see DND indicator (already supported — "dnd" status shows red dot)

## Verification

- Set quiet hours 10pm-7am in EST → at 11pm EST receive no notifications; at 8am EST notifications
  resume
- Quick-toggle DND for 2 hours → no notifications for 2 hours → auto-resume
- Quick-toggle DND indefinitely → manually turn off → notifications resume
- Other users see DND presence indicator during DND
- Verify both web and mobile
