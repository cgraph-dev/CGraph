# Plan 04-01: Unified Design Tokens + WCAG Audit

**Phase:** 04-design-mobile **Wave:** 1 (parallel with 04-03) **Depends on:** ŌĆö **Requirements:**
DESIGN-01, DESIGN-02

## Goal

Consolidate web's two conflicting color systems into a single CSS-variable-driven token layer, unify
the two competing ThemeProviders, audit all color pairs for WCAG AA compliance, and fix failing
combinations.

## Context (from discovery)

**Two divergent color systems on web:**

- Tailwind config (`tailwind.config.js`): emerald-based greens (`primary-500: #10b981`)
- Theme engine (`themes.ts`): indigo-based (`primary: #6366f1`, `secondary: #818cf8`)

These are different palettes applied in different contexts ŌĆö Tailwind classes vs CSS variables.

**Two competing ThemeProviders:**

- `ThemeProvider` (theme-context.tsx): manages light/dark/system via CSS class toggle
- `ThemeProviderEnhanced` (theme-provider-enhanced.tsx): manages 7+ themes via CSS variables
- Both persist to same localStorage key (`cgraph-theme`) with different data shapes

**5+ hardcoded color locations:** tailwind.config.js, theme-globals.css, themes.ts, presets.ts,
index.css

**WCAG failures found:**

- Mobile `#10b981` (primary) as bubble background with white text = 3.2:1 (fails AA 4.5:1)
- Matrix theme `#008822` (textMuted) on `#000000` = 3.2:1 (fails AA)
- `#10b981` as inputFocus on `#ffffff` = borderline for text

## Tasks

### Task 1: Establish single color token source

Create `apps/web/src/lib/theme/tokens.ts` as the ONE source of truth for all color values. Define
semantic tokens:

```
// Surface tokens
bg.primary, bg.secondary, bg.tertiary, bg.inverse
// Text tokens
text.primary, text.secondary, text.muted, text.inverse, text.on-primary, text.on-error
// Interactive tokens
interactive.primary, interactive.hover, interactive.active, interactive.disabled
// Feedback tokens
feedback.error, feedback.warning, feedback.success, feedback.info
// Component tokens
chat.bg, chat.bubble.sent, chat.bubble.received, sidebar.bg, sidebar.text
```

Each token has a dark AND light value. Token file is the canonical reference.

### Task 2: Unify ThemeProviders

Merge `ThemeProvider` and `ThemeProviderEnhanced` into a single provider:

- Keep the CSS-variable injection approach from `ThemeProviderEnhanced`
- Keep the light/dark/system detection from `ThemeProvider`
- Single localStorage key with consistent data shape
- Remove the duplicate provider ŌĆö keep one file
- Update `App.tsx` or layout to use the unified provider

### Task 3: Wire Tailwind to CSS variables

Update `tailwind.config.js` to reference CSS variables instead of hardcoded hex:

```js
colors: {
  primary: {
    DEFAULT: 'var(--token-primary)',
    hover: 'var(--token-primary-hover)',
    // ...
  }
}
```

This way Tailwind classes like `bg-primary` and `text-primary` automatically respond to theme
changes.

### Task 4: WCAG AA audit and fix

Audit all token pairs using contrast ratio calculation:

For each theme (Dark, Light, Matrix, Holo variants):

1. Calculate contrast ratio for every text/bg combination
2. Flag any pair below 4.5:1 for normal text or 3:1 for large text
3. Adjust failing colors to meet AA minimums

**Known fixes needed:**

- Mobile `bubbleSent` (#10b981) with white text ŌåÆ darken to `#059669` (~4.7:1) or use dark text
- Matrix `textMuted` (#008822) on black ŌåÆ lighten to `#00b33c` (~5.2:1)
- Document all contrast ratios in a comment block in tokens.ts

### Task 5: Sync mobile tokens

Update mobile `themeStore.ts` to align color values with the new canonical tokens:

- Match primary, background, text values to web tokens
- Fix `bubbleSent` WCAG failure
- Keep mobile's architecture (Zustand + AsyncStorage) ŌĆö only sync VALUES, not architecture
- Document any platform-specific deviations (e.g., mobile statusBar colors)

### Task 6: Clean up redundant color definitions

- Remove hardcoded hex values from `theme-globals.css` (use CSS variables from engine)
- Remove duplicate definitions in `presets.ts` store (reference tokens)
- Update `index.css` base layer to use token variables instead of hardcoded dark colors
