# Plan 13-01: Mobile WebRTC Wiring + Call History API

## Context

The mobile app has beautiful call UI screens (~2,000 lines across voice, video, and history screens)
but **all WebRTC state is simulated with `setTimeout()`**. The web app has fully working P2P WebRTC
calls. The backend signaling infrastructure is complete (GenServer rooms + Phoenix Channels).

**What exists:**

- `webrtcService.ts` (604L) has types and structure matching `react-native-webrtc` interfaces but no
  actual native module installed
- `call-screen.tsx` (644L), `voice-call-screen.tsx` (638L) have full UI but fake state transitions
- `call-history-screen.tsx` (577L) uses MOCK_CALLS array instead of backend API
- Backend `Calls` context has full CRUD + pagination for `call_history` table
- Backend has no REST controller for call history (only used through WebSocket channels)

**What's needed:**

1. Install `react-native-webrtc` native module
2. Replace simulated connections with real RTCPeerConnection + Phoenix Channel signaling
3. Add backend REST controller for call history
4. Wire mobile call history to real API with callback button

## Tasks

### Task 1: Install react-native-webrtc and create call service

**Files:** `apps/mobile/package.json`, `apps/mobile/src/services/callService.ts`,
`apps/mobile/src/stores/callStore.ts`

**Actions:**

1. Add `react-native-webrtc` dependency to mobile package.json
2. Create `callService.ts` API client:
   - `getCallHistory(cursor?: string, limit?: number)` â†’ GET /api/v1/calls/history
   - `getCall(callId: string)` â†’ GET /api/v1/calls/:id
3. Create `callStore.ts` Zustand store:
   - State: `activeCall`, `callHistory`, `isLoading`, `incomingCall`
   - Actions: `startCall`, `answerCall`, `endCall`, `fetchCallHistory`, `setIncomingCall`
   - Subscribe to WebSocket events for incoming calls

### Task 2: Wire mobile WebRTC service to real peer connections

**Files:** `apps/mobile/src/lib/webrtc/webrtcService.ts`

**Actions:**

1. Import real `RTCPeerConnection`, `RTCSessionDescription`, `RTCIceCandidate`, `mediaDevices` from
   `react-native-webrtc`
2. Replace any simulated connection logic with actual:
   - `mediaDevices.getUserMedia({audio: true, video: callType === 'video'})` for local stream
   - `new RTCPeerConnection(iceServers)` with STUN/TURN from backend
   - ICE candidate gathering and relay through `call:room_id` Phoenix Channel
   - SDP offer/answer exchange through channel events
3. Wire `ontrack` handler to surface remote MediaStream
4. Add `getDisplayMedia()` for screen sharing (where available on mobile)
5. Handle network transitions (WiFi â†’ cellular) gracefully with ICE restart

### Task 3: Replace simulated call state in mobile screens

**Files:** `apps/mobile/src/screens/calls/call-screen.tsx`,
`apps/mobile/src/screens/calls/voice-call-screen.tsx`,
`apps/mobile/src/screens/calls/video-call-screen/use-video-call.ts`

**Actions:**

1. In `call-screen.tsx`:
   - Remove all `setTimeout()` state simulation
   - Import and use `callStore` for real connection state
   - Connect to Phoenix Channel `call:room_id` on mount
   - React to actual `oniceconnectionstatechange` for connected/disconnected/failed
2. In `voice-call-screen.tsx`:
   - Wire remote audio to speaker output
   - Replace simulated mute/unmute with real `track.enabled` toggle
   - Show real call duration from store
3. In `use-video-call.ts`:
   - Replace `setTimeout(() => setCallState('connected'), 2000)` with real RTCPeerConnection state
   - Wire `RTCView` to actual remote MediaStream from peer connection
   - Camera flip via `videoTrack._switchCamera()` (react-native-webrtc API)

### Task 4: Backend call history REST controller

**Files:** `apps/backend/lib/cgraph_web/controllers/api/v1/call_controller.ex`,
`apps/backend/lib/cgraph_web/router.ex`

**Actions:**

1. Create `CallController` with:
   - `index/2` â€” GET /calls/history â€” cursor-paginated call history for authenticated user

   - `show/2` â€” GET /calls/:id â€” single call detail
2. Add routes to router.ex under `/api/v1` authenticated scope:
   ```elixir
   resources "/calls", CallController, only: [:index, :show]
   ```
3. Return JSON: `{data: [...], meta: {cursor, has_more}}`

### Task 5: Wire mobile call history to real API

**Files:** `apps/mobile/src/screens/calls/call-history-screen.tsx`

**Actions:**

1. Remove `MOCK_CALLS` constant
2. Use `callStore.fetchCallHistory()` on mount and pull-to-refresh
3. Map backend response to existing UI format (participants â†’ avatars, duration, type)
4. Add "Call Back" button to each history item â†’ triggers `callStore.startCall(userId, callType)`
5. Add infinite scroll with cursor pagination
6. Show empty state when no call history
