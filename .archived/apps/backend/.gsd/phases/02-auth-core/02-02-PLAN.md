  <action>
    Create a shared password validation utility and update the web register form to enforce the same
    rules as mobile (`apps/mobile/src/screens/auth/register-screen/use-register.ts`).

    1. Create `apps/web/src/modules/auth/utils/password-validation.ts`:
       ```typescript
       export interface PasswordValidationResult {
         isValid: boolean;
         errors: string[];
       }

       export function validatePassword(password: string): PasswordValidationResult {
         const errors: string[] = [];
         if (password.length < 8) errors.push('Password must be at least 8 characters');
         if (!/[a-z]/.test(password)) errors.push('Must contain a lowercase letter');
         if (!/[A-Z]/.test(password)) errors.push('Must contain an uppercase letter');
         if (!/[0-9]/.test(password)) errors.push('Must contain a number');
         if (!/[!@#$%^&*()_+\-=\[\]{};':"\\|,.<>\/?]/.test(password))
           errors.push('Must contain a special character');
         return { isValid: errors.length === 0, errors };
       }
       ```

    2. Update `useRegisterForm.ts`:
       - Import `validatePassword` from the new utility
       - Replace the simple `password.length < 8` check with `validatePassword(password)`
       - Show individual validation errors (not just generic "password too short")
       - Keep the existing password match check (`password !== confirmPassword`)

    3. Optionally add a `PasswordRequirements` visual indicator component showing which rules pass/fail
       (similar to the existing `PasswordStrengthMeter` used on reset-password page)

    Match the mobile validation rules exactly:
    - Min 8 characters
    - At least one lowercase letter
    - At least one uppercase letter
    - At least one number
    - At least one special character
