# Plan 11-02: Group CRUD, Channels & Invites — E2E Wiring

## Context

Extensive infrastructure exists across all three platforms:

**Backend (complete):**

- `group_controller.ex` (141 lines): full CRUD — index, show, create, update, delete, audit_log
- `channel_controller.ex` (110 lines): CRUD for channels within groups
- `invite_controller.ex` (154 lines): create, index, delete, show_by_code, accept invites
- `operations.ex` (198 lines): `create_group` creates group + default #general channel + audit log
- Routes fully defined in `messaging_routes.ex` (nested REST resources)
- Tier-based group limits: free=5, premium=50, enterprise=unlimited

**Web (mostly complete):**

- `create-group-modal.tsx` (182 lines): form with name, description, visibility toggle
- `channel-list.tsx` (140 lines): renders channels with categories
- `create-channel-modal.tsx` (155 lines): channel creation with name, type, category selection
- `invite-modal.tsx` (121 lines): create/manage tabs for invite links
- `group-settings.tsx` (159 lines): settings panel with tabs
- `groups-page.tsx` (88 lines): main page with ServerList + ChannelList + ContentArea
- `group-actions.ts` (260 lines): full Zustand actions — createGroup, joinGroup, leaveGroup,
  updateGroup, deleteGroup, createInvite, fetchGroups, etc.

**Mobile (mostly complete):**

- `groupsService.ts` (654 lines): full API client with types + transformers
- `groupStore.ts` (469 lines): Zustand store with socket integration
- `group-screen.tsx` (281 lines), `channel-screen.tsx` (311 lines), `group-channels-screen.tsx` (291
  lines)
- `group-invites-screen.tsx` (234 lines), `group-settings-screen.tsx` (162 lines)
- `group-list-screen/` directory with empty state, animated header, morphing cards

**Gaps to address:**

1. Avatar/banner upload not wired in web create-group-modal (backend has upload endpoints but web
   likely skips them)
2. Category management (create/reorder) may not be wired in web/mobile
3. Invite join flow may use inconsistent endpoints (web uses /invites/:code/join but mobile uses
   both joinGroup and joinGroupByInvite)
4. Channel type selection in create modal may be limited
5. Group list on web needs to consistently fetch and render user's groups

## Tasks

### Web — Group Creation

1. **Verify create-group-modal.tsx wiring**
   - Confirm `createGroup()` action calls `POST /api/v1/groups` with
     `{ name, description, is_public }`
   - Verify form validation (name required, max length)
   - After creation, navigate to the new group's default channel
   - Add avatar upload: after group creation, call `POST /api/v1/groups/:id/avatar` (if backend
     supports, or add file input to form)
   - Verify newly created group appears in group list sidebar

2. **Verify groups-page.tsx renders group list**
   - Ensure `fetchGroups()` is called on mount
   - ServerList sidebar renders all user groups with icons
   - Clicking a group loads its channels
   - Active group is highlighted

### Web — Channel Management

3. **Verify create-channel-modal.tsx wiring**
   - Confirm it calls the correct API: `POST /api/v1/groups/:groupId/channels`
   - Verify channel type selector (text, voice, announcement, stage)
   - Verify category selection dropdown populated from group's categories
   - New channel appears in channel list after creation

4. **Verify channel-list.tsx category rendering**
   - Channels grouped by category with collapsible sections
   - Uncategorized channels in default section
   - Channels sorted by position within each category
   - Channel type icons (# for text, speaker for voice, megaphone for announcement)

5. **Wire category creation**
   - If not already wired: add "Create Category" option in channel list
   - Calls `POST /api/v1/groups/:id/categories` with name
   - New category appears in list

### Web — Invites

6. **Verify invite-modal.tsx create flow**
   - "Create" tab calls `createInvite()` action → `POST /api/v1/groups/:id/invites`
   - Returns invite code, constructs link: `https://cgraph.app/invite/{code}`
   - Copy-to-clipboard button works
   - Configurable: max uses dropdown, expiry dropdown

7. **Verify invite-modal.tsx management**
   - "Manage" tab lists active invites via `GET /api/v1/groups/:id/invites`
   - Shows code, uses/maxUses, expiry, creator
   - Revoke button calls `DELETE /api/v1/groups/:id/invites/:inviteId`

8. **Verify join-by-invite flow**
   - `joinGroup()` action calls `POST /api/v1/invites/:code/join`
   - On success, group is added to store, `justJoinedGroupName` triggers celebration animation
   - User is redirected to the group's default channel
   - Wire an "Enter Invite Code" input somewhere accessible (group list? social page?)

### Web — Group Settings

9. **Verify group-settings.tsx edit flow**
   - Overview tab: edit name, description, visibility → calls `updateGroup()` →
     `PATCH /api/v1/groups/:id`
   - Changes persist and reflect immediately in group header
   - Channels tab: reorder channels → calls `updateChannelOrder()`
   - Delete group option (owner only) → calls `deleteGroup()` → redirects away

### Mobile — Group Creation

10. **Verify group creation flow**
    - Empty state "Create Group" button triggers creation flow
    - Uses `createGroup()` from groupsService → `POST /api/v1/groups`
    - After creation, navigate to group screen
    - Group appears in group list

11. **Verify group-screen.tsx**
    - Loads group details and channels on mount
    - Renders channel list organized by categories
    - Navigation to channel-screen on channel tap
    - Header shows group name, avatar, member count

### Mobile — Channels

12. **Verify channel management on mobile**
    - Channel list in group-screen shows categories + channels
    - Create channel option available to admins/owners
    - Uses `createChannel()` from groupsService
    - Channel types supported

### Mobile — Invites

13. **Verify group-invites-screen.tsx**
    - Create invite: calls `createGroupInvite()` → generates code + share link
    - Share button uses React Native Share API
    - Join by code: input field + join button → calls `joinGroupByInvite(code)`
    - Existing invites listed with uses/expiry

14. **Verify mobile group settings**
    - `group-settings-screen.tsx` allows editing group name/description
    - Uses `updateGroup()` from groupsService
    - Member management accessible (separate screen)
