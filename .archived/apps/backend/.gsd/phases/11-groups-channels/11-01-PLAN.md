# Plan 11-01: Group Channel Messaging — WebSocket Alignment & E2E Wiring

## Context

The backend GroupChannel (`apps/backend/lib/cgraph_web/channels/group_channel.ex`, 270 lines) is
fully implemented with join/access-check, message history on join, new_message/typing/edit/delete
handlers, rate limiting (10/10s), and presence.

However, there is a **critical topic mismatch** that prevents any real-time messaging from working:

- **Backend:** `user_socket.ex` routes `"group:*"` to `GroupChannel`, which pattern-matches
  `"group:" <> channel_id`
- **Web:** `groupChannel.ts` constructs topic as `"channel:${channelId}"` — no `"channel:*"` route
  exists → **FAILS**
- **Mobile groupStore:** constructs topic as `"channel:${channelId}"` — same issue → **FAILS**
- **Mobile channel-screen:** constructs topic as `"group:${groupId}:channel:${channelId}"` → matches
  `"group:*"` but channel_id extracts as `"abc:channel:def"` → **WRONG**
- **Mobile useRealtimeChannel:** uses same composite format → **WRONG**

All clients must send `"group:{channelId}"` to match the backend.

Additionally, the message flow needs E2E verification: REST API fetches channel messages, WebSocket
receives real-time updates, and stores update correctly.

## Tasks

### Web — Fix WebSocket Topic

1. **Fix `groupChannel.ts` topic format**
   - Change `const topic = \`channel:${channelId}\`` to `const topic = \`group:${channelId}\``
   - Update `leaveGroupChannel()` to use same `group:` prefix
   - Verify `channels.has(topic)` guard uses correct key

2. **Verify `group-channel.tsx` hooks into socket**
   - Ensure `joinGroupChannel()` is called when navigating to a channel
   - Ensure `leaveGroupChannel()` is called on unmount/channel switch
   - Ensure `fetchChannelMessages()` is called for initial history
   - Verify the existing `useEffect` for channel join/leave cycle

3. **Verify web message rendering pipeline**
   - `new_message` event → `normalizeMessage()` → `addChannelMessage()` → store update → UI
     re-render
   - `message_updated` → `updateChannelMessage()` → UI update
   - `message_deleted` → `removeChannelMessage()` → UI update
   - Ensure `channelId` field on ChannelMessage is properly set from backend response

### Mobile — Fix WebSocket Topic

4. **Fix `groupStore.ts` socket join**
   - Change `const topic = \`channel:${channelId}\`` to `const topic = \`group:${channelId}\``
   - Update any leaveChannel calls to use `group:` prefix

5. **Fix `channel-screen.tsx` socket join**
   - Change
     `socketManager.joinChannel(\`group:${groupId}:channel:${channelId}\`)`to`socketManager.joinChannel(\`group:${channelId}\`)`
   - Change corresponding `leaveChannel` call to match
   - Verify message event handlers are wired (new_message, typing, etc.)

6. **Fix `useRealtimeChannel.ts` group channel topic**
   - Change the topic construction at the bottom of the file from
     `\`group:${groupId}:channel:${channelId}\``to`\`group:${channelId}\``
   - Verify the hook exposes push() for sending messages

7. **Verify mobile message send path**
   - Confirm channel-screen sends messages via Phoenix channel push("new_message") or REST POST
   - Backend GroupChannel.handle_in("new_message") creates message and broadcasts to all members
   - Verify incoming messages update groupStore.channelMessages correctly

### Shared — Message Payload Alignment

8. **Verify backend message JSON matches frontend types**
   - `MessageJSON.render("message.json")` must include: `id`, `content`, `sender_id`, `channel_id`,
     `inserted_at`, `updated_at`, plus `sender` object
   - Web `ChannelMessage` type (in group-types.ts) must match backend response shape
   - Mobile `ChannelMessage` type (in groupStore.ts) must match backend response shape
   - If ChannelMessage types use `channelId` but backend sends `channel_id`, verify
     normalizeMessage() handles the conversion

9. **Verify typing indicators**
   - Web: `setTypingUser()` action wired to `typing` event from GroupChannel
   - Mobile: typing handler in channel-screen or groupStore processes `typing` event
   - Both platforms: send typing push to channel on input change with throttle

10. **Verify presence**
    - Backend GroupChannel tracks presence on join (`CGraph.Presence.track`)
    - Web logs presence_state/presence_diff — verify this renders member list or online indicator
    - Mobile: verify presence state is tracked and displayed
