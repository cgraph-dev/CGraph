    granted_at (utc_datetime). Unique index on (user_id, milestone_key).

    ReputationRewards context module:
    - @milestones map: %{"helpful_votes_100" => %{check: fn, reward: 100}, "posts_1000" => ...}
    - check_and_grant/1 (user_id): iterates milestones, checks if user qualifies AND hasn't been
      granted yet, calls Nodes.credit_nodes/4 for each new milestone
    - already_granted?/2 (user_id, milestone_key): checks reputation_rewards table
    - list_user_rewards/1: returns all granted milestones for a user
    - Milestone definitions:
      * helpful_votes_100: user received 100+ helpful votes → 100 Nodes
      * helpful_votes_500: 500+ helpful votes → 300 Nodes
      * posts_1000: 1000+ forum posts → 500 Nodes
      * posts_5000: 5000+ posts → 2000 Nodes
      * friends_50: 50+ friends → 50 Nodes
      * friends_200: 200+ friends → 200 Nodes
      * account_1year: account age >= 1 year → 100 Nodes
      * account_2year: account age >= 2 years → 500 Nodes

    ⚠️ REMOVED milestones (no tracking infrastructure exists):
    - voice_hours_100 — no voice hour tracking exists in codebase (deferred)
    - groups_admin_3 — no count_admin_groups/1 query exists (deferred)

    ⚠️ DEPENDENCY PATHS (verified exact module locations):
    - Post count: CGraph.Forums.UserContent.get_user_post_stats/1 → .total_posts
    - Friend count: CGraph.Accounts.Friends.Queries.get_friend_stats/1 → .friends
      (NOT CGraph.Friends — that module doesn't exist)
    - Helpful votes: CGraph.Forums.Reputation calc uses upvotes from ForumMember
    - Account age: user.inserted_at compared to DateTime.utc_now()

    ReputationRewardWorker: Oban worker in :reputation_rewards queue, daily cron at 4 AM UTC.
    Queries users with recent activity (last 7 days), calls check_and_grant/1 for each.
    Batches of 100 users. Logs results via Logger.info.

    Add queue: :reputation_rewards to Oban config in config.exs (concurrency: 2).
