# Plan 18-04 — Infrastructure Scale & Hardening

## Goal

Ensure CGraph can handle 10,000+ concurrent users: wire feature flag admin management, audit and
harden rate limiting, scale load tests to 10K WebSockets with realistic traffic patterns, and
configure connection backpressure. Backend infrastructure is mature (feature_flags.ex 368L,
rate_limiter.ex 363L, k6 suite 5 scripts); this plan completes the admin layer and validates at
scale.

## Why This Ordering

Tasks proceed: (0) feature flag admin API + routes, (1) feature flag frontend SDK + admin panel, (2)
rate limiting audit + per-tier limits, (3) WebSocket connection backpressure, (4) 10K WebSocket load
test, (5) realistic traffic load test, (6) rich media load test, (7) results documentation.

```elixir
defmodule CGraphWeb.API.Admin.FeatureFlagController do
  use CGraphWeb, :controller

  alias CGraph.FeatureFlags

  def index(conn, _params) do
    flags = FeatureFlags.all_flags()
    render(conn, :index, flags: flags)
  end

  def show(conn, %{"id" => name}) do
    case FeatureFlags.get_flag(name) do
      {:ok, flag} -> render(conn, :show, flag: flag)
      {:error, :not_found} -> conn |> put_status(:not_found) |> json(%{error: "not_found"})
    end
  end

  def create(conn, %{"flag" => params}) do
    admin = conn.assigns.current_user
    case FeatureFlags.create_flag(params, audit: %{changed_by: admin.id}) do
      {:ok, flag} -> conn |> put_status(:created) |> render(:show, flag: flag)
      {:error, changeset} -> conn |> put_status(:unprocessable_entity) |> json(%{errors: changeset})
    end
  end

  def update(conn, %{"id" => name, "flag" => params}) do
    admin = conn.assigns.current_user
    case FeatureFlags.update_flag(name, Map.put(params, "changed_by", admin.id)) do
      {:ok, flag} -> render(conn, :show, flag: flag)
      {:error, reason} -> conn |> put_status(:unprocessable_entity) |> json(%{error: reason})
    end
  end

  def delete(conn, %{"id" => name}) do
    admin = conn.assigns.current_user
    case FeatureFlags.delete_flag(name, audit: %{changed_by: admin.id}) do
      :ok -> send_resp(conn, :no_content, "")
      {:error, reason} -> conn |> put_status(:unprocessable_entity) |> json(%{error: reason})
    end
  end

  # NOTE: FeatureFlags.get_history/1 does NOT exist yet — must be created in feature_flags.ex
  def history(conn, %{"id" => name}) do
    history = FeatureFlags.get_history(name)
    render(conn, :history, history: history)
  end
end
```

```typescript
export const useFeatureFlagStore = create<FeatureFlagState>((set, get) => ({
  flags: {},
  lastFetched: 0,

  fetchFlags: async () => {
    const now = Date.now();
    if (now - get().lastFetched < 5 * 60 * 1000) return; // 5-min cache

    set({ flags: response.data.flags, lastFetched: now });
```

**Web hook** — `useFeatureFlag.ts`:

```typescript
export function useFeatureFlag(flagName: string): {
  enabled: boolean;
  variant?: string;
  loading: boolean;
} {
  const { flags, fetchFlags } = useFeatureFlagStore();

  useEffect(() => {
    fetchFlags();
  }, []);

  const flag = flags[flagName];
  return {
    enabled: flag?.enabled ?? false,
    variant: flag?.variant,
    loading: Object.keys(flags).length === 0,
  };
}
```

```elixir
defp get_rate_limit(scope, conn) do
  base_limit = RateLimiter.get_limit(scope)
```

**Where:**

**How:**

**Bandit/Phoenix config** — in `runtime.exs`, configure for 10K+ connections:

```elixir
config :cgraph, CGraphWeb.Endpoint,
  http: [
    port: port,
    transport_options: [
      num_acceptors: 100,       # Match CPU cores × 4
      max_connections: 15_000,  # Allow 10K sockets + 5K HTTP headroom
      socket_opts: [
        {:backlog, 2048},       # TCP backlog for burst connections
        {:nodelay, true}        # Disable Nagle for real-time
      ]
    ]
  ]
```

**Connection monitor** — Track active WebSocket connections:

```elixir
defmodule CGraph.Cluster.ConnectionMonitor do
  use GenServer

  def start_link(_opts) do
    GenServer.start_link(__MODULE__, %{}, name: __MODULE__)
  end

  def connection_count do
    # Count active transport pids in Phoenix PubSub
    Phoenix.PubSub.node_name(CGraph.PubSub)
    |> :pg.get_members(CGraph.PubSub)
    |> length()
  end

  def at_capacity?(threshold \\ 0.9) do
    max = Application.get_env(:cgraph, :max_ws_connections, 10_000)
    connection_count() / max >= threshold
  end
end
```

**Socket-level backpressure** — in `endpoint.ex`, add a custom connect check:

```elixir
socket "/socket", CGraphWeb.UserSocket,
  websocket: [
    check_origin: false,
    timeout: 45_000,
    connect_info: [:peer_data, :x_headers],
    # Reject connections at capacity
    error_handler: {CGraphWeb.SocketErrorHandler, :handle_error, []}
  ]
```

If at capacity, return `{:error, :at_capacity}` with a 503-like WebSocket close code (1013).

### Task 5 — Realistic Traffic Load Test (INFRA-04)

**What:** Create a k6 test modeling realistic user journeys with think times, varying behavior, and
mixed HTTP + WebSocket traffic.

**Where:** `infrastructure/load-tests/k6/realistic-traffic.js` (new)

**How:**

Model user journey as a state machine:

```javascript
export default function () {
  // 1. Login (HTTP)
  const loginRes = http.post(
    `${BASE_URL}/api/v1/auth/login`,
    JSON.stringify({
      email: `user${__VU}@test.com`,
      password: 'testpass123',
    }),
    { headers: { 'Content-Type': 'application/json' } }
  );

  const token = loginRes.json('data.access_token');
  sleep(randomBetween(1, 3)); // Think time

  // 2. Browse conversations (HTTP)
  http.get(`${BASE_URL}/api/v1/conversations`, {
    headers: { Authorization: `Bearer ${token}` },
  });
  sleep(randomBetween(2, 5));

  // 3. Open WebSocket + join conversation (WS)
  const ws_url = `${WS_URL}/socket/websocket?token=${token}`;
  ws.connect(ws_url, {}, (socket) => {
    socket.on('open', () => {
      socket.send(joinMessage('conversation:1'));
    });

    // 4. Send messages burst (3-5 messages over 30s)
    for (let i = 0; i < randomBetween(3, 5); i++) {
      socket.send(
        JSON.stringify({
          topic: 'conversation:1',
          event: 'new_message',
          payload: { content: `Load test message ${i}` },
          ref: String(i + 2),
        })
      );
      sleep(randomBetween(3, 10));
    }

    // 5. Search messages (HTTP)
    http.get(`${BASE_URL}/api/v1/search/messages?q=hello&conversation_id=1`, {
      headers: { Authorization: `Bearer ${token}` },
    });
    sleep(randomBetween(2, 5));

    // 6. Idle period (1-3 minutes)
    sleep(randomBetween(60, 180));

    // 7. Disconnect
    socket.close();
  });
}
```

Options: ramp 50 → 500 VUs over 10 minutes, hold 5 minutes, ramp down.

**Verify:** Test completes with p95 HTTP < 500ms, p95 WS message < 200ms, error rate < 1%.
