# Plan 11-03: Explore Page & Channel Threads

## Context

### Group Discovery

**Backend (complete):**
- `group_repository.ex` has `list_discoverable/1` with sort options (popular, newest, name) and pagination
- `groups.ex` has `search_groups/1` facade
- Routes in `user_routes.ex`: `GET /search/groups`
- Routes in `messaging_routes.ex`: groups index lists user's groups

**Web (partial):**
- `discover-tab.tsx` (108 lines) in social pages — generic search across users/forums/groups
- Navigates to `/groups/:id` on group click, has Join button (but no join handler wired)
- No dedicated explore page with category browsing, featured groups, or filtering
- `group-actions.ts` has no `searchGroups()` or `fetchDiscoverableGroups()` action

**Mobile (partial):**
- `groupsService.ts` has `getPublicGroups()` with category/search/sort params and `getFeaturedGroups()`
- `discovery-section.tsx` in search-screen shows trending topics but not group discovery
- No dedicated explore-groups screen exists
- `groupStore.ts` has no discover/explore state

### Channel Threads

**Backend (complete):**
- `thread_channel.ex` (405 lines): full Phoenix channel for threads — join, replies, voting, polls, viewers
- `thread_controller.ex` (211 lines): create thread, list replies, get thread, update, delete, vote, poll
- Routes: nested under channels — `resources "/threads"` with replies
- Thread model exists with `parent_message_id`, `channel_id`, content, author

**Web (mostly complete):**
- `threadStore.ts` (212 lines): Zustand store — openThread, closeThread, fetchMoreReplies, sendThreadReply, fetchReplyCounts
- `thread-panel.tsx` (261 lines): side panel — thread header, reply list, send input
- `thread-reply-badge.tsx`, `thread-reply-count.tsx`: badges for parent messages
- `threadChannel.ts` (230 lines): Phoenix socket — joinThread, leaveThread with handlers

**Mobile (partial):**
- Thread-related UI may exist in some channel screens but needs verification
- No dedicated thread panel/bottom-sheet for mobile

**Gaps to address:**
1. Web needs a proper explore page (not just generic search tab) with categories, featured, and join wiring
2. Web group-actions.ts needs discoverGroups/searchGroups action
3. Mobile needs an explore-groups screen wired to getPublicGroups/getFeaturedGroups
4. Thread panel needs to be triggered from channel messages (verify "Reply in thread" action exists)
5. Mobile needs thread view for channel messages — likely bottom sheet or separate screen
6. Thread reply counts need to load for channel messages and show badges

## Tasks

### Web — Explore Groups Page

1. **Add `fetchDiscoverableGroups` action to group store**
   - New state: `discoverableGroups: Group[]`, `isLoadingDiscover: boolean`, `discoverSearch: string`
   - Action calls `GET /api/v1/groups/discover` (or `GET /search/groups`) with params: `{ search, sort, page, limit }`
   - Also add `fetchFeaturedGroups` if backend supports a featured endpoint

2. **Create `explore-groups.tsx` page**
   - Route: `/groups/explore` or integrate into social discover tab
   - Layout:
     - Search bar at top with debounced input (300ms)
     - Category filter chips (Gaming, Crypto, Art, Tech, etc.)
     - Sort selector: Popular / Newest / Name
     - Featured groups carousel (if available)
     - Grid/list of discoverable groups — each card shows: icon, name, description, member count, "Join" button
   - Join button calls existing `joinGroup(inviteCode)` for public groups or a new `joinPublicGroup(groupId)` action
   - Clicking a group navigates to `/groups/:id`
   - Pagination: load more on scroll

3. **Wire join from explore**
   - For public groups, backend should support `POST /api/v1/groups/:id/join` (direct join without invite code)
   - If this endpoint doesn't exist, add to group-actions.ts via existing `/groups/:id/members` POST
   - On success, add group to user's groups list, show celebration

4. **Update web routing**
   - Add `/groups/explore` route in app-routes.tsx
   - Add "Explore" link in groups sidebar or navigation

### Mobile — Explore Groups Screen

5. **Create `explore-groups-screen.tsx`**
   - Search bar with debounced input
   - Category filter horizontal scroll (chips)
   - Featured groups section (horizontal card carousel)
   - Public groups list (FlatList with pull-to-refresh, infinite scroll)
   - Each group card: icon, name, description, member count, join button
   - Uses `getPublicGroups({ search, category, sortBy })` and `getFeaturedGroups()` from groupsService
   - Join button: calls `joinGroup(groupId)` → navigates to group screen

6. **Add explore navigation**
   - Add explore screen to groups-navigator.tsx
   - Add "Explore" button in group list screen header or empty state
   - Wire into search-screen discovery section as a quick action

### Web — Channel Threads

7. **Verify thread opening from channel messages**
   - In channel message item component, ensure "Reply in thread" action exists
   - Clicking it calls `threadStore.openThread(messageId, channelId)`
   - Thread panel slides in from right side of channel view
   - Verify the thread panel renders alongside the channel content (not replacing it)

8. **Verify thread reply flow**
   - Thread panel shows parent message at top
   - Replies load via `fetchMoreReplies()` → `GET /api/v1/channels/:channelId/threads/:threadId/replies`
   - Send reply input at bottom → `sendThreadReply()` → `POST /api/v1/channels/:channelId/threads/:threadId/replies`
   - New replies appear in real-time via threadChannel socket
   - Verify threadChannel.ts `joinThread()` uses correct `"thread:{threadId}"` topic (matches backend)

9. **Wire thread reply count badges**
   - `thread-reply-count.tsx` component renders on channel messages that have threads
   - `fetchReplyCounts()` loads counts for visible messages
   - Badge shows "N replies" with last reply timestamp
   - Clicking badge opens thread panel

10. **Verify thread panel close and cleanup**
    - Close button leaves thread channel socket
    - Thread state clears from store
    - Channel view returns to full width

### Mobile — Channel Threads

11. **Create thread bottom sheet for mobile**
    - When user long-presses a channel message → "Reply in Thread" option
    - Opens a bottom sheet (or full-screen modal) showing:
      - Parent message at top
      - Thread replies list
      - Send reply input at bottom
    - Fetches replies from `GET /api/v1/channels/:channelId/threads/:threadId/replies`
    - Sends via `POST /api/v1/channels/:channelId/threads/:threadId/replies`
    - Real-time: join `"thread:{threadId}"` socket channel for live updates

12. **Thread reply count on mobile channel messages**
    - Show reply count indicator below channel messages that have threads
    - Tapping opens the thread bottom sheet
    - Fetch reply counts when loading channel messages
