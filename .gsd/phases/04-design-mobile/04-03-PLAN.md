# Plan 04-03: Mobile EAS Build Pipeline

**Phase:** 04-design-mobile **Wave:** 1 (parallel with 04-01) **Depends on:** — **Requirements:**
INFRA-08

## Goal

Fix Expo EAS configuration so mobile app builds for iOS simulator and Android emulator, launches,
and reaches the login screen. Add convenience build scripts.

## Context (from discovery)

**What works:**

- Expo SDK 54 with React Native 0.81.5, New Architecture enabled
- `eas.json` has 3 profiles (development, preview, production)
- `app.config.js` has dynamic environment config for API URLs
- `app.json` has full iOS privacy manifests and proper permissions
- EAS Updates configured

**What's broken/missing:**

- `EAS_PROJECT_ID` is a placeholder string (`'cgraph-production'`), NOT a real Expo project UUID
- Apple submit credentials are placeholders
- `app.json` vs `app.config.js` drift:
  - notification color: `#6366f1` (app.json) vs splash `#10b981` (app.config.js)
  - assetBundlePatterns: `["assets/**/*"]` (app.json) vs `["**/*"]` (app.config.js)
  - iOS buildNumber `"4"` vs Android versionCode `3`
- No convenience build scripts in `package.json`
- No CI workflow for EAS builds

## Tasks

### Task 1: Initialize EAS project

Run `eas init` to get a real project UUID (or configure via `EXPO_PUBLIC_EAS_PROJECT_ID` env var).

If `eas` CLI isn't installed: `npx eas-cli init`

Update `app.config.js` to use the real project ID from env or config:

```js
extra: {
  eas: {
    projectId: process.env.EAS_PROJECT_ID || 'real-uuid-from-eas-init',
  },
},
```

### Task 2: Fix app.json / app.config.js drift

Consolidate into `app.config.js` as the single config source:

- Sync version numbers: versionCode and buildNumber should match
- Sync colors: pick one notification/splash color and use consistently
- Sync asset patterns: use `["assets/**/*"]` (more explicit)
- Remove duplicated values from `app.json` where `app.config.js` overrides them
- Document env vars needed for builds in a comment or README

### Task 3: Add build convenience scripts

Add to `apps/mobile/package.json`:

```json
{
  "scripts": {
    "build:dev": "eas build --profile development --platform all",
    "build:dev:ios": "eas build --profile development --platform ios",
    "build:dev:android": "eas build --profile development --platform android",
    "build:preview": "eas build --profile preview --platform all",
    "build:preview:ios": "eas build --profile preview --platform ios",
    "build:preview:android": "eas build --profile preview --platform android"
  }
}
```

### Task 4: Verify development build

Run development build for both platforms:

```bash
cd apps/mobile
npx eas build --profile development --platform ios --local  # iOS simulator
npx eas build --profile development --platform android --local  # Android emulator
```

If `--local` builds aren't feasible (requires Xcode/Android SDK), verify at minimum:

1. `npx expo prebuild --clean` succeeds (generates native projects)
2. Native project files (ios/android directories) are properly generated
3. `npx expo start` launches Metro without errors
4. App loads in Expo Go or dev client and reaches login screen

### Task 5: Document build setup

Add a `BUILD.md` or section in the mobile README with:

- Required env vars (EAS_PROJECT_ID, API_URL variants)
- How to configure Apple credentials for iOS builds
- How to configure Google service account for Android
- Build commands for each profile
- OTA update commands (`eas update`)
