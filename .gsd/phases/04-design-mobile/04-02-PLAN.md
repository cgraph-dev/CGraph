# Plan 04-02: Web Light Mode

**Phase:** 04-design-mobile **Wave:** 2 (depends on 04-01) **Depends on:** 04-01 (unified tokens
with light/dark values) **Requirements:** DESIGN-05

## Goal

Make web app actually render in light mode. Currently the infrastructure exists (ThemeProvider
toggles CSS classes, ThemeEngine selects theme) but CSS is 100% dark-only. No `dark:` utilities, no
`.light` class overrides, no theme-aware base styles.

## Context (from discovery)

**What works:**

- ThemeProvider toggles `<html class="dark|light">`
- ThemeEngine has `THEME_LIGHT` with proper light colors
- `usePrefersDarkMode` hook detects system preference
- Theme store persists preference to localStorage

**What's broken:**

- `index.css` body: `@apply bg-dark-900 text-gray-100` ŌĆö hardcoded dark
- All component classes (`.btn-secondary`, `.card`, `.input`) use dark colors only
- Scrollbar styling hardcoded to dark background
- `.message-bubble` hardcoded to `bg-dark-700/50`
- No `dark:` Tailwind prefixes in global CSS
- ThemeSwitcher does `window.location.reload()` on change ŌĆö jarring

**After 04-01** the CSS variable system will be in place, so light mode is a matter of:

1. Making base CSS reference variables (not hardcoded hex)
2. Ensuring Tailwind `dark:` prefix works OR CSS variables swap values

## Tasks

### Task 1: Make base styles theme-aware

Update `index.css` to reference CSS variables from the token system (04-01):

```css
body {
  background-color: var(--token-bg-primary);
  color: var(--token-text-primary);
}

/* Scrollbar */
::-webkit-scrollbar-track {
  background: var(--token-bg-secondary);
}
::-webkit-scrollbar-thumb {
  background: var(--token-bg-tertiary);
}
```

Remove all hardcoded `bg-dark-*` and `text-gray-*` from base layer.

### Task 2: Update component utility classes

Update `.btn`, `.btn-primary`, `.btn-secondary`, `.card`, `.input`, `.message-bubble`, etc. in
`index.css` to use CSS variables:

```css
.card {
  background-color: var(--token-bg-secondary);
  border-color: var(--token-border);
}

.input {
  background-color: var(--token-bg-input);
  color: var(--token-text-primary);
  border-color: var(--token-border);
}
```

### Task 3: Verify theme-globals.css integration

Ensure `theme-globals.css` properly sets CSS variable values for both light and dark:

- Light theme: CSS variables inject light palette values
- Dark theme: CSS variables inject dark palette values
- Verify transitions work (`.theme-transitioning` class)

### Task 4: Fix ThemeSwitcher UX

Remove the `window.location.reload()` from theme switching. CSS variables should swap instantly
without a reload. If reload was a workaround for stale styles, the variable-based system eliminates
that need.

### Task 5: Visual QA ŌĆö light mode render check

Verify these pages render correctly in light mode:

- Login page
- Register page
- Settings page (appearance tab)
- Main chat layout (sidebar + message area)

Check for:

- No invisible text (white on white, dark on dark)
- All borders visible
- Focus rings visible
- Error/success states visible
- Scrollbars styled
