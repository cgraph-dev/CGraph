    Create migration with timestamp pattern YYYYMMDDHHMMSS:

    CREATE TABLE node_wallets:
    - user_id UUID PRIMARY KEY REFERENCES users(id) ON DELETE CASCADE
    - available_balance INTEGER NOT NULL DEFAULT 0  (spendable now)
    - pending_balance INTEGER NOT NULL DEFAULT 0  (earned, held 21 days)
    - lifetime_earned INTEGER NOT NULL DEFAULT 0
    - lifetime_spent INTEGER NOT NULL DEFAULT 0
    - inserted_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
    - updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()

    CREATE TABLE node_transactions:
    - id UUID PRIMARY KEY DEFAULT gen_random_uuid()
    - user_id UUID REFERENCES users(id) NOT NULL
    - amount INTEGER NOT NULL (positive = credit, negative = debit)
    - type VARCHAR(30) NOT NULL
      (purchase | tip_received | tip_sent | content_unlock | subscription_received |
       subscription_sent | withdrawal | cosmetic_purchase)
    - reference_id UUID (thread_id, user_id of tip target, etc.)
    - reference_type VARCHAR(30) (thread, user, subscription, etc.)
    - platform_cut INTEGER (20% retained by CGraph on earned transactions)
    - net_amount INTEGER (amount after platform cut)
    - hold_until TIMESTAMPTZ (21 days after earned; NULL for purchased nodes)
    - metadata JSONB
    - inserted_at TIMESTAMPTZ NOT NULL DEFAULT NOW()

    CREATE INDEX ON node_transactions (user_id, inserted_at DESC)
    CREATE INDEX ON node_transactions (type)
    CREATE INDEX ON node_transactions (hold_until) WHERE hold_until IS NOT NULL

    CREATE TABLE withdrawal_requests:
    - id UUID PRIMARY KEY DEFAULT gen_random_uuid()
    - user_id UUID REFERENCES users(id) NOT NULL
    - nodes_amount INTEGER NOT NULL
    - currency VARCHAR(3) NOT NULL DEFAULT 'EUR'
    - fiat_amount DECIMAL(10,2) NOT NULL (nodes * 0.008 = EUR)
    - status VARCHAR(20) NOT NULL DEFAULT 'pending' (pending | processing | completed | failed)
    - payout_reference VARCHAR(255) (Stripe Transfer ID or external ref)
    - inserted_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
    - completed_at TIMESTAMPTZ

  <files>apps/backend/lib/cgraph/nodes/</files>
  <action>
    Create new context module at apps/backend/lib/cgraph/nodes/:

    Schemas:
    - nodes/node_wallet.ex — Ecto schema for node_wallets (binary_id, timestamps :utc_datetime)
    - nodes/node_transaction.ex — Ecto schema for node_transactions (binary_id, timestamps :utc_datetime)
    - nodes/withdrawal_request.ex — Ecto schema for withdrawal_requests (binary_id, timestamps :utc_datetime)
    - nodes/node_bundles.ex — Bundle definitions (NOT a schema, a module with functions):
      Starter(500/€4.99/no bonus), Popular(1200/€9.99/+20%), Creator(2800/€19.99/+40%),
      Pro(6500/€39.99/+63%), Ultimate(17000/€99.99/+70%)

    Context module nodes/nodes.ex (CGraph.Nodes):
    - get_or_create_wallet(user_id) — returns wallet, creates if not exists
    - get_balance(user_id) — returns {available, pending}
    - credit_nodes(user_id, amount, type, opts) — add to available_balance (for purchases)
    - debit_nodes(user_id, amount, type, opts) — subtract from available_balance (for spending)
      Returns {:error, :insufficient_balance} if not enough
    - tip(sender_id, recipient_id, amount) — Ecto.Multi transaction:
      1. Debit sender available_balance by amount
      2. Calculate platform_cut = floor(amount * 0.20)
      3. net_amount = amount - platform_cut
      4. Credit recipient pending_balance by net_amount
      5. Create tip_sent transaction for sender (negative amount)
      6. Create tip_received transaction for recipient (positive net_amount, hold_until = now + 21 days)
      Returns {:ok, transaction} or {:error, :insufficient_balance}
    - unlock_content(user_id, thread_id) — Ecto.Multi transaction:
      1. Load thread, verify is_content_gated == true
      2. Check if already unlocked (check node_transactions for content_unlock + thread reference)
      3. Debit user available_balance by gate_price_nodes
      4. Credit thread author pending_balance by net_amount (after 20% cut)
      5. Create content_unlock transaction for buyer
      6. Create subscription_received transaction for author
      Returns {:ok, transaction} or {:error, :insufficient_balance | :already_unlocked | :not_gated}
    - release_held_nodes() — called by Oban worker:
      Find node_transactions where hold_until <= now AND type in tip_received/subscription_received
      Move matching amounts from pending_balance → available_balance
    - request_withdrawal(user_id, nodes_amount) —
      Validates available_balance >= nodes_amount, minimum 1000 nodes
      fiat = nodes_amount * 0.008
      Creates withdrawal_request with status: pending
      Debits available_balance

    All write operations use CGraph.Repo.
    All read operations use CGraph.ReadRepo.
    All multi-step operations use Ecto.Multi with proper rollback.

  </action>
