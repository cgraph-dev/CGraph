# Plan 16-02: Achievement Triggers & Quest Rotation Engine

## Context

GAME-02 requires achievements triggered by milestones. The backend has `AchievementSystem` (141 LOC) with list/progress/unlock + `AchievementRepository` (170 LOC) with DB queries, plus `Achievement` schema with 6 categories and 6 rarity tiers. The web has achievement display (1,272 LOC), notification toast (153 LOC), and mobile has achievement screens. **But there's no seed data and no trigger logic that checks milestones.**

GAME-03 requires daily/weekly quests. The backend has `QuestSystem` (147 LOC) with accept/claim flow, `Quest` schema with 5 types, `UserQuest` with objective map tracking. Web has quest panel (778 LOC) + quest page. **But there's no Oban worker for quest rotation and no template pool.**

**Critical gap:** No `AchievementTriggers` module exists to evaluate "has user sent 100 messages? → unlock Chatterbox achievement." Similarly, no code generates daily quests from templates.

<objective>
Implement achievement milestone triggers and quest rotation engine. Seed 30+ achievements and 20+ quest templates. Wire achievement checks into the XP event pipeline and automate quest generation.

Purpose: Without triggers, achievements never unlock. Without rotation, quests never appear. This activates the progression loop.
Output: AchievementTriggers, QuestRotationWorker, QuestTemplates modules; seed migration; notification wiring.
</objective>

  Create `CGraph.Gamification.AchievementTriggers`:
  ```elixir
  defmodule CGraph.Gamification.AchievementTriggers do
    alias CGraph.Gamification.{AchievementSystem, Repositories.AchievementRepository}
    alias CGraph.Repo
    
    @action_achievement_map %{
      message_sent: ["first_message", "chatterbox", "motor_mouth", "megaphone", "legendary_talker"],
      forum_thread: ["first_thread", "forum_regular", "forum_veteran"],
      forum_post: ["first_thread", "forum_regular"],  # posts also count
      forum_upvote_received: ["helpful_poster", "respected_voice", "forum_legend"],
      friend_added: ["first_friend", "social_butterfly", "social_maven", "networking_legend"],
      group_joined: ["group_explorer", "community_builder"],
      quest_completed: ["first_quest", "quest_master"]
    }
    
    @doc "Check all achievements relevant to the given action type."
    def check_all(user_id, action_type) when is_atom(action_type) do
      slugs = Map.get(@action_achievement_map, action_type, [])
      
      Enum.each(slugs, fn slug ->
        AchievementSystem.increment_achievement_progress(user_id, slug, 1)
      end)
    end
    
    @doc "Check level-based achievements (called on level-up)."
    def check_level_achievements(user, level) do
      level_slugs = [
        {5, "level_5"}, {10, "level_10"}, {25, "level_25"}, {50, "level_50"}
      ]
      
      Enum.each(level_slugs, fn {req_level, slug} ->
        if level >= req_level do
          AchievementSystem.unlock_achievement_by_slug(user, slug)
        end
      end)
    end
  end
  ```
  - `check_all/2` is called from XpEventHandler after each action (already wired by 16-01 via update_quest_progress, but we add achievement check too)
  - `increment_achievement_progress` already handles: does nothing if already unlocked, increments progress, auto-unlocks when progress >= max_progress
  - Add broadcast on unlock: when AchievementSystem.try_unlock_achievement returns {:ok, unlocked}, broadcast to PubSub
  
  **REVIEW NOTE:** `AchievementSystem.check_level_achievements/2` is currently a no-op stub (`def check_level_achievements(_user, _level), do: :ok`). Task 2 must replace this stub with actual level-based achievement checking logic.

  ```elixir
  defmodule CGraph.Gamification.QuestRotationWorker do
    use Oban.Worker, queue: :gamification, max_attempts: 3
    
    alias CGraph.Gamification.{Quest, QuestTemplates}
    alias CGraph.Repo
    
    @impl Oban.Worker
    def perform(%Oban.Job{args: %{"type" => "daily"}}) do
      expire_old_quests(:daily)
      generate_quests(:daily, QuestTemplates.pick_daily(3))
      broadcast_new_quests(:daily)
      :ok
    end
    
    def perform(%Oban.Job{args: %{"type" => "weekly"}}) do
      expire_old_quests(:weekly)
      generate_quests(:weekly, QuestTemplates.pick_weekly(3))
      broadcast_new_quests(:weekly)
      :ok
    end
  end
  ```
  - Register in application.ex Oban config:

  - `expire_old_quests/1` — set ends_at on previous day's/week's quests
  - `generate_quests/2` — insert Quest records with starts_at = now, ends_at = 24h/7d from now
  - `broadcast_new_quests/1` — PubSub broadcast "new_quests_available" to trigger client notifications
  
  NOTE: Must register `:gamification` queue in `config/config.exs` and `config/prod.exs` Oban config before worker can process jobs.
