<objective>
Bootstrap E2EE on app startup and make real-time message decryption reliable.

Purpose: Currently, `useE2EEStore.getState().initialize()` is never called at app startup, so `isInitialized` stays `false`. When encrypted messages arrive via WebSocket, they enter `decryptAndAddMessage`, see `isInitialized === false`, show "🔒 Decrypting…", then poll 10 times over 5 seconds — but E2EE never initializes because nobody triggered it. After 5s the message shows "⚠️ Unable to decrypt". On page refresh, the REST API returns server-decrypted plaintext so messages appear fine.

Output: E2EE initializes on login, real-time messages decrypt immediately or as soon as init completes.
</objective>

<task type="auto">
  <name>Task 2: Replace blind polling with event-driven retry</name>
  <files>apps/web/src/modules/chat/store/chatStore.messaging.ts</files>
  <action>
  Replace the `retryDecryptAfterInit` function that blindly polls 10 times every 500ms (total 5s max) with an event-driven approach using Zustand's `subscribe`:

  ```typescript
  async function retryDecryptAfterInit(message: Message, get: Get) {
    const TIMEOUT_MS = 15_000; // 15s timeout (E2EE auto-bootstrap can take time)

    return new Promise<void>((resolve) => {
      let timeoutId: ReturnType<typeof setTimeout>;

      const unsubscribe = useE2EEStore.subscribe((state) => {
        if (!state.isInitialized) return;

        // E2EE ready — attempt decryption
        unsubscribe();
        clearTimeout(timeoutId);

        attemptDecrypt(message, state)
          .then((plaintext) => {
            let protocolVersion: string | undefined;
            try {
              const proto = state.getSessionProtocol(message.senderId);
              if (proto) protocolVersion = String(proto);
            } catch { /* Non-critical */ }

            get().updateMessage({
              ...message,
              content: plaintext,
              isEncrypted: true,
              decryptionFailed: false,
              protocolVersion,
            });
            logger.log('Decrypted queued E2EE message after init');
          })
          .catch((error: unknown) => {
            logger.error('Retry decryption failed after init:', error);
            get().updateMessage({
              ...message,
              content: '⚠️ Unable to decrypt this message',
              isEncrypted: true,
              decryptionFailed: true,
            });
          })
          .finally(resolve);
      });

      // Timeout: if E2EE never initializes, mark as failed
      timeoutId = setTimeout(() => {
        unsubscribe();
        logger.error('E2EE did not initialize within timeout, marking message as decrypt-failed');
        get().updateMessage({
          ...message,
          content: '⚠️ Unable to decrypt this message',
          isEncrypted: true,
          decryptionFailed: true,
        });
        resolve();
      }, TIMEOUT_MS);

      // Check if already initialized (race condition: init completed between
      // the if-check in decryptAndAddMessage and subscribing here)
      const currentState = useE2EEStore.getState();
      if (currentState.isInitialized) {
        unsubscribe();
        clearTimeout(timeoutId);

        attemptDecrypt(message, currentState)
          .then((plaintext) => {
            let protocolVersion: string | undefined;
            try {
              const proto = currentState.getSessionProtocol(message.senderId);
              if (proto) protocolVersion = String(proto);
            } catch { /* Non-critical */ }

            get().updateMessage({
              ...message,
              content: plaintext,
              isEncrypted: true,
              decryptionFailed: false,
              protocolVersion,
            });
          })
          .catch((error: unknown) => {
            logger.error('Retry decryption failed:', error);
            get().updateMessage({
              ...message,
              content: '⚠️ Unable to decrypt this message',
              isEncrypted: true,
              decryptionFailed: true,
            });
          })
          .finally(resolve);
      }
    });
  }
  ```

  Key improvements:
  - Uses Zustand `.subscribe()` to react INSTANTLY when `isInitialized` flips to `true`
  - 15s timeout instead of 5s (auto-bootstrap involves key generation + API call)
  - Handles race condition where init completes between check and subscribe
  - No wasted CPU cycles polling every 500ms
  </action>
