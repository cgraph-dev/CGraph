<objective>
Decrypt encrypted messages that are loaded via the REST API (fetchMessages).

Purpose: Currently `fetchMessages` stores raw API messages without decryption. The backend returns `is_encrypted: true` messages with ciphertext in the `content` field. After page refresh, the backend conveniently returns decrypted content for the user's own sent messages, but incoming encrypted messages from the other party need client-side decryption. Currently they show ciphertext or "Unable to decrypt" because `fetchMessages` never runs them through the decrypt pipeline.

This plan operates on `chatStore.impl.ts` which is independent from Plan 01's files, so they can run in parallel.

Output: fetchMessages processes encrypted messages through the decrypt pipeline after loading them from the API.
</objective>

  1. Import `useE2EEStore` at the top of the file (alongside existing imports).
  2. Import `useAuthStore` if not already imported.
  3. After normalizing messages (`const newMessages = rawMessages.map(...)`), add:

  ```typescript
  // Decrypt encrypted messages from REST API
  const e2eeState = useE2EEStore.getState();
  const currentUserId = useAuthStore.getState().user?.id;

  const processedMessages = await Promise.all(
    newMessages.map(async (msg) => {
      // Skip non-encrypted messages
      if (!msg.isEncrypted) return msg;

      // Own sent messages: backend returns plaintext in content field for the sender
      // (the server stores encrypted_content separately). If content looks like
      // readable text (not base64 ciphertext), keep it as-is.
      if (currentUserId && msg.senderId === currentUserId) {
        return { ...msg, isEncrypted: true };
      }

      // For messages from others: attempt client-side decryption
      if (!e2eeState.isInitialized) {
        // E2EE not ready yet — mark as needing decryption, show placeholder
        return {
          ...msg,
          content: '🔒 Decrypting…',
          isEncrypted: true,
          decryptionFailed: false,
        };
      }

      try {
        const { attemptDecrypt } = await import('./chatStore.messaging');
        const plaintext = await attemptDecrypt(msg, e2eeState);
        return {
          ...msg,
          content: plaintext,
          isEncrypted: true,
          decryptionFailed: false,
        };
      } catch {
        // Decryption failed — the backend may have already returned plaintext
        // in the content field (some backends do server-side decryption for REST).
        // If content doesn't look like base64 ciphertext, keep it.
        if (msg.content && !looksLikeCiphertext(msg.content)) {
          return { ...msg, isEncrypted: true, decryptionFailed: false };
        }
        return {
          ...msg,
          content: '⚠️ Unable to decrypt this message',
          isEncrypted: true,
          decryptionFailed: true,
        };
      }
    })
  );
  ```

  4. Replace `newMessages` with `processedMessages` in the rest of the function (the `set()` call that stores merged messages).

  5. Add a helper function at module level:
  ```typescript
  /** Heuristic: base64 ciphertext is typically 40+ chars of [A-Za-z0-9+/=] with no spaces */
  function looksLikeCiphertext(content: string): boolean {
    if (content.length < 40) return false;
    // If it contains spaces, newlines, or common text patterns, it's probably plaintext
    if (/\s/.test(content)) return false;
    // Check if it looks like base64
    return /^[A-Za-z0-9+/=]+$/.test(content);
  }
  ```

  6. IMPORTANT: Export `attemptDecrypt` from `chatStore.messaging.ts` so it can be imported by `chatStore.impl.ts`. Currently it's a module-level function (not exported). Add `export` to its declaration.

  Note: The `fetchMessages` function needs to become `async` if it isn't already (it already is — it uses `await api.get()`).
