<objective>
Build the Pulse reputation backend — a community-scoped reputation system.
NOTE: An existing `CGraph.Reputation` context (144 LOC) handles per-user
reputation entries (from_user → to_user, +/- value). Pulse is SEPARATE:
it is community-scoped (per community_id), uses weighted voting, has 6 tiers,
and implements decay. The existing Reputation system remains for general
user-to-user rep; Pulse adds community-level scoring on top.

Create tables, write the Elixir context, implement weighted voting, tier calculation, and decay.
Expose API endpoints.

Pulse is community-scoped reputation. Each user has a separate score per community. Votes are
weighted by the voter's own Pulse (1x at 0 pulse, up to 4x at 900+). </objective>
