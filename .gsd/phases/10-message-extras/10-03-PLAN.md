# Plan 10-03: Message Extras Polish — Bookmark Actions, Disappearing Mobile, Shared Types

## Context

Discovery found MSG-13 (Pin) and MSG-14 (Bookmark) are fully implemented at the backend + page level, and MSG-17 (Disappearing) has full backend + web toggle. However:

1. **MSG-14**: No "Save" button in message action menus on either platform — users can't reach the save feature from a message
2. **MSG-17**: No disappearing messages toggle on mobile, and no visual timer indicator on messages set to disappear
3. **Shared types**: Forward, save, disappearing types need to be consolidated

This is a polish + integration plan.

## Tasks

### Save/Bookmark Action Buttons (MSG-14)

1. **Add Save to web message action menu**
   - In `apps/web/src/modules/chat/components/message-bubble/message-action-menu.tsx`
   - Add "Save" / "Unsave" toggle button (bookmark icon)
   - Call `POST /api/v1/saved-messages` with `{ message_id }` to save
   - Call `DELETE /api/v1/saved-messages/:id` to unsave
   - Need to check if message is already saved (call `GET /api/v1/saved-messages` or track locally)
   - Use `BookmarkIcon` (outline) / `BookmarkSolidIcon` (filled) for toggle state

2. **Add Save to mobile message action menu**
   - In `apps/mobile/src/components/conversation/message-actions-menu.tsx`
   - Add "Save message" / "Unsave message" action
   - Same API calls as web
   - Follow existing action pattern (icon + label)

### Disappearing Messages Mobile (MSG-17)

3. **Create mobile disappearing messages toggle**
   - At `apps/mobile/src/components/chat/disappearing-messages-toggle.tsx`
   - Same options as web: Off, 24 hours, 7 days, 30 days
   - Calls `PUT /api/v1/conversations/:id/ttl` with selected value
   - Shows current TTL setting
   - Follow existing mobile settings component pattern

4. **Add toggle to mobile conversation info/settings**
   - Find the mobile conversation details/info screen
   - Add "Disappearing Messages" section with the toggle

5. **Add timer icon to disappearing messages (web)**
   - In web message bubble, when message has `expiresAt`:
   - Show small clock/timer icon (⏱) next to timestamp
   - Optional tooltip: "Disappears at [time]"

6. **Add timer icon to disappearing messages (mobile)**
   - Same indicator in mobile message bubble
   - Small clock icon next to timestamp

### Pin Verification (MSG-13)

7. **Verify pin works E2E**
   - Read pin-related code paths to confirm wiring
   - Verify: web message menu has pin action → API call → broadcast → pin bar/panel shows it
   - Verify: mobile message menu has pin action → API call → broadcast → pinned bar shows it
   - If any gap found, fix it (expect none — discovery showed fully implemented)

### Shared Types Consolidation

8. **Extend `packages/shared-types/src/messages.ts`**
   - Add types that 10-01 may have started and fill remaining:
   ```typescript
   export interface SavedMessage {
     id: string;
     messageId: string;
     note?: string;
     savedAt: string;
   }

   export interface DisappearingConfig {
     ttl: number | null; // seconds: null=off, 86400=24h, 604800=7d, 2592000=30d
     expiresAt?: string; // ISO datetime for individual message
   }
   ```
   - Verify `ForwardedMessageInfo` exists from 10-01
   - Export all from package index

9. **Update web/mobile stores to use shared types where possible**
   - Import `SavedMessage`, `ForwardedMessageInfo` from `@cgraph/shared-types`
   - Only update where types are compatible (don't force breaking changes)

feat(10-03): message extras polish — save buttons, mobile disappearing, shared types

## Verification

- Web: Right-click message → "Save" → message appears in Saved Messages page
- Mobile: Long-press message → "Save" → message appears in Saved Messages screen
- Mobile: Open conversation settings → "Disappearing Messages" → set to 24h → send message → shows timer icon
- Web: Disappearing message shows timer icon next to timestamp
- Pin: Pin a message on web → appears in pinned bar/panel on both platforms
- All message extras (forward, pin, bookmark, link preview, disappearing) work E2E on both platforms
