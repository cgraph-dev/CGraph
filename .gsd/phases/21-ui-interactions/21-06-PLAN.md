   `{isOpen && <div className="dropdown-menu">...`).

3. Wrap with AnimatePresence and replace the container with motion.div:

```tsx
const reducedMotion = useReducedMotion();

<AnimatePresence>
  {isOpen && (
    <motion.div
      initial={reducedMotion ? undefined : { opacity: 0, scale: 0.95, y: -4 }}
      animate={{ opacity: 1, scale: 1, y: 0 }}
      exit={{ opacity: 0, scale: 0.95, y: -4 }}
      transition={
        reducedMotion
          ? { duration: 0 }
          : {
              type: 'spring',
              stiffness: 500,
              damping: 30,
            }
      }
      className="..." // preserve existing dropdown positioning classes
    >
      {/* existing dropdown items */}
    </motion.div>
  )}
</AnimatePresence>;
```

**Notification items** — These are rendered in a list. Add stagger entrance animation.

1. Read notification-item.tsx fully to understand its structure.

2. Add imports:

```ts
import { motion } from 'motion/react';
import { stagger } from '@cgraph/animation-constants';
```

3. Wrap the notification item's root element with motion.div:

```tsx
<motion.div
  initial={{ opacity: 0, y: 8 }}
  animate={{ opacity: 1, y: 0 }}
  transition={{
    duration: 0.2,
    delay: index * stagger.fast.staggerChildren,
  }}
>
  {/* existing notification item content */}
</motion.div>
```

If the component doesn't receive an `index` prop, add it as an optional prop:

```ts
interface NotificationItemProps {
  // ...existing props
  /** List index for stagger animation delay */
  index?: number;
}
```

Then pass `index` from the parent list component (the notifications page that maps over items).

**Notification badge** — Find where unread count badges are rendered (likely in
notification-header.tsx or sidebar).

If there's a badge showing unread count, add a key-based re-animation:

```tsx
<motion.span
  key={unreadCount}
  initial={{ scale: 1.3 }}
  animate={{ scale: 1 }}
  transition={{ type: 'spring', stiffness: 500, damping: 20 }}
  className="..." // existing badge classes
>
  {unreadCount}
</motion.span>
```

The `key={unreadCount}` forces re-mount on count change, triggering the bounce.

IMPORTANT:

- Do NOT change notification data fetching, filtering, or store logic
- Do NOT change notification item click/dismiss handlers
- Keep all existing className strings
- The stagger delay should be capped (e.g., `Math.min(index ?? 0, 10)`) to prevent long delays on
  large lists </action> <verify>
