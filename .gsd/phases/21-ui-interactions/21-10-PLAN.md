Target the most common hardcoded duration patterns. These are inline `duration:` values in framer-motion
transition objects that should reference shared presets.

1. Find all hardcoded durations:

```bash
grep -rn "duration: 0\.\(1\|15\|2\|25\|3\|4\|5\)" apps/web/src/ --include="*.tsx" --include="*.ts" | grep -v node_modules | grep -v animation-presets | grep -v animation-constants | head -40
```

2. For each common pattern, replace with the appropriate preset import:

| Hardcoded Value  | Replace With                                   |
| ---------------- | ---------------------------------------------- |
| `duration: 0.1`  | `duration: durations.instant.ms / 1000` (0.1s) |
| `duration: 0.15` | `duration: durations.fast.ms / 1000` (0.15s)   |
| `duration: 0.2`  | `duration: durations.normal.ms / 1000` (0.2s)  |
| `duration: 0.3`  | `duration: durations.slow.ms / 1000` (0.3s)    |
| `duration: 0.4`  | `duration: durations.smooth.ms / 1000` (0.4s)  |
| `duration: 0.5`  | `duration: durations.slower.ms / 1000` (0.5s)  |

Or use the `durationsSec` wrapper from `@/lib/animation-presets`: | `duration: 0.15` |
`duration: durationsSec.fast` | | `duration: 0.2` | `duration: durationsSec.normal` |

Prefer `durationsSec` from animation-presets when in component files (it's already a seconds
number). Prefer `durations.X.ms / 1000` when in preset definition files.

3. For files that don't already import from animation-presets or animation-constants, add the
   import:

```ts

```

Focus on the **highest-impact replacements**: files with 3+ hardcoded durations, and the most common
values (0.2, 0.3, 0.15). Don't spend time on one-off values in deeply nested components.

IMPORTANT:

- Do NOT change animation behavior — the replacement must use the equivalent numeric value
- Do NOT modify duration values that don't have a preset match (e.g., `duration: 0.07` or
  `duration: 1.2`)
- If a file already imports `durationsSec`, just replace the inline value
- If a file uses `durations` from @cgraph/animation-constants, use `durations.X.ms / 1000` </action>
