<task type="auto">
  <name>Task 1: Add haptics + spring press to mobile Button</name>
  <files>apps/mobile/src/components/button.tsx</files>
  <action>
The mobile Button uses `TouchableOpacity` with `Animated` from React Native (not Reanimated).

1. Add imports:

```ts
import * as Haptics from 'expo-haptics';
import ReanimatedAnimated, {
  useSharedValue,
  useAnimatedStyle,
  withSpring,
} from 'react-native-reanimated';
import { springs } from '@cgraph/animation-constants';
import { useReducedMotion } from 'react-native-reanimated';
```

2. Convert to use Reanimated for the press animation:

```ts
const scale = useSharedValue(1);
const reducedMotion = useReducedMotion();

const animatedStyle = useAnimatedStyle(() => ({
  transform: [{ scale: scale.value }],
}));

const handlePressIn = () => {
  if (!reducedMotion) {
    scale.value = withSpring(0.96, {
      stiffness: springs.snappy.stiffness,
      damping: springs.snappy.damping,
    });
  }
  Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Light);
};

const handlePressOut = () => {
  scale.value = withSpring(1, {
    stiffness: springs.snappy.stiffness,
    damping: springs.snappy.damping,
  });
};
```

3. Replace `TouchableOpacity` with `Pressable` wrapped in `ReanimatedAnimated.View`:

```tsx
<ReanimatedAnimated.View style={animatedStyle}>
  <Pressable
    onPress={onPress}
    onPressIn={handlePressIn}
    onPressOut={handlePressOut}
    disabled={disabled || loading}
    style={[buttonStyle, style]}
  >
    {/* existing content: icon, text, loading indicator */}
  </Pressable>
</ReanimatedAnimated.View>
```

Alternatively, if changing from TouchableOpacity to Pressable is too invasive:

- Keep TouchableOpacity
- Add `onPressIn` and `onPressOut` handlers for haptic + scale
- Wrap with ReanimatedAnimated.View for the scale animation

IMPORTANT:

- Do NOT change the ButtonProps interface (except to add optional onPressIn/onPressOut if needed)
- Do NOT change variant styles, size mapping, or theme logic
- Do NOT change the loading state rendering
- Keep backward compatibility — haptics are additive </action> <verify>
