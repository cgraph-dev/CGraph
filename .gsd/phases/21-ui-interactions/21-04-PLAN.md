<task type="auto">
  <name>Task 1: Add layoutId active indicator to nav items</name>
  <files>apps/web/src/layouts/app-layout/sidebar.tsx</files>
  <action>
The sidebar already imports `motion` and `AnimatePresence` from framer-motion (now motion/react after Plan 01).
It uses NavLink with `isActive` from react-router-dom.

Add these imports:

```ts
import { LAYOUT_IDS } from '@cgraph/animation-constants';
import { useMotionSafe } from '@/hooks/useMotionSafe';
```

For each nav item rendered via `NavLink`, add a conditional `motion.div` as the active background
indicator:

```tsx
{
  isActive && (
    <motion.div
      layoutId={LAYOUT_IDS.sidebarActiveIndicator}
      className="absolute inset-0 rounded-lg bg-white/[0.08]"
      transition={springs.smooth}
      style={{ zIndex: -1 }}
    />
  );
}
```

Where `springs` comes from `useMotionSafe()`.

Also add `whileHover` and `whileTap` to each nav item wrapper:

```tsx
<motion.div
  whileHover={hoverScale(1.02)}
  whileTap={tapScale(0.98)}
  transition={springs.snappy}
  className="relative"
>
  <NavLink ...>
    {/* existing icon + label content */}
    {isActive && (
      <motion.div layoutId={LAYOUT_IDS.sidebarActiveIndicator} ... />
    )}
  </NavLink>
</motion.div>
```

IMPORTANT:

- Do NOT modify store imports, logout handler, presence selector, or any business logic
- Do NOT change the NavLink routing behavior
- Keep existing badge rendering (unread counts, lock badges)
- Use `position: relative` on the nav item wrapper so the `absolute` indicator positions correctly
- The existing `location` prop is already available for determining active state

If the nav items are rendered in a loop/map, apply the pattern inside the map callback. If some nav
items already have motion wrappers, enhance them rather than duplicating. </action> <verify>
