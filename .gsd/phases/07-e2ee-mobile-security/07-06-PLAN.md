<objective>
Build safety number verification screens on web and mobile with QR code support.

Purpose: Safety numbers let users verify they're communicating with the intended recipient (not a
man-in-the-middle). The backend API exists (`/api/v1/e2ee/safety-number/:user_id`), the mobile hook
exists (`useE2EE.generateSafetyNumber`). This plan creates the UI screens for both platforms.
Satisfies E2EE-03.

Output: Web dialog and mobile screen showing 60-digit safety number + QR code. Users can compare
numbers visually or scan QR codes. </objective>
