Add interactive canvas-based background effects to the web app, matching the landing and auth page visual quality.

Purpose: The landing page has a 100-particle neural network with mouse-reactive physics. Auth pages
have a similar particle field + cyber grid. The main app currently uses a static GradientBackground
(blurred CSS orbs). This plan adds an interactive particle canvas behind the app content.

Output:

- `cinematic-background.tsx` — Canvas particle engine adapted from landing/auth patterns

<task type="auto">
  <name>Task 1: Create cinematic background component</name>
  <files>apps/web/src/components/ui/cinematic-background.tsx</files>
  <action>
    Create a performant canvas-based particle background:

    1. Port the particle physics engine from the landing page's GraphNetwork.ts
    2. Use `backgroundPresets.particleField.medium` as default (60 particles — lighter than landing's 100 for always-on app use)
    3. Implement:
       - Particle nodes with emerald/purple/cyan colors from backgroundPresets.colors.particles
       - Connection lines between nearby particles (within connectionDistance)
       - Mouse repulsion physics (gentle push when cursor approaches)
       - Smooth opacity on connections based on distance
    4. Performance requirements:
       - Use `requestAnimationFrame` with timestamp-based delta for frame-independent physics
       - Canvas sized to viewport with devicePixelRatio (capped at 2)
       - Auto-pause when document is hidden (`visibilitychange`)
       - Limit to 60fps via RAF
    5. Accept `intensity` prop: 'full' | 'medium' | 'subtle' | 'off' mapping to backgroundPresets
    6. Reduced motion: render static dots only (no movement, no connections)
    7. Style: position fixed, inset 0, z-index 0, pointer-events none

  </action>
  <verify>pnpm turbo typecheck --filter=@cgraph/web</verify>
  <done>CinematicBackground renders canvas with interactive particles</done>
</task>
