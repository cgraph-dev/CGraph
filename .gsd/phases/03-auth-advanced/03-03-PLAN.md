<objective>
Bridge session revocation to token invalidation so that revoking a session actually prevents the user from making authenticated requests.

Purpose: Currently `sessions.revoke_session/1` and `session_management.revoke_session/1` only mark
sessions as revoked in the DB but do NOT invalidate tokens in TokenManager.Store. This means revoked
sessions can still make API calls until the access token expires. Output: Session revocation
triggers token revocation. Full integration test coverage. </objective>

**1. Check TokenManager.Store API:**
- Determine how tokens are keyed in Store (by `jti`? by `session_id`?)
- If Store doesn't support revocation by session, add a lookup/revocation function:
  - Option A: If sessions store `jti` references → `Store.revoke(jti)` for each
  - Option B: If Store keys include session info → `Store.revoke_by_session_id(session_id)`
  - Option C: If sessions store token pairs → revoke both access + refresh JTIs
- Choose the approach that fits existing data model. Prefer the simplest.

**2. Modify `sessions.revoke_session/1`:**

- After setting `revoked_at` in DB, call token revocation:
  ```elixir
  # After successful DB update
  revoke_tokens_for_session(session)
  ```
- The `revoke_tokens_for_session/1` helper should find the session's token JTIs and call
  `Store.revoke/1` or `Store.delete/1` for each.
- If the session schema tracks `token_jti` or similar, use it directly.
- If not, look up tokens by user_id + session_id in Store.

**3. Modify `session_management.revoke_session/1`:**

- If it delegates to `sessions.revoke_session/1`, no change needed (bridge already applied).
- If it has its own implementation, add the same token revocation bridge.

**4. Modify `session_management.revoke_other_sessions/2`:**

- After revoking other sessions in DB, revoke their tokens too.
- Can iterate through revoked sessions and call the bridge for each.

**5. Handle edge cases:**

- Token not in Store (expired/evicted): `Store.delete/1` should be idempotent — confirm or make it
  so.
- Wrap token revocation in try/rescue if Store errors shouldn't prevent session revocation.
