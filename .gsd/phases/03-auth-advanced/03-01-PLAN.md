<task type="auto">
  <name>Task 2: GREEN — Implement 2FA login gate</name>
  <files>apps/backend/lib/cgraph_web/controllers/api/v1/auth_controller.ex, apps/backend/lib/cgraph_web/router/auth_routes.ex</files>
  <action>
**In auth_controller.ex:**

1. Modify `handle_successful_login/3` to check `user.totp_enabled`:
   - If `true`: Generate a random temp token, store it in Cachex `:two_factor_challenges` with
     key=token, value=%{user_id: user.id, lockout_key: lockout_key}, TTL=5 minutes. Return
     `%{status: "2fa_required", two_factor_token: token}` with 200.
   - If `false` (or nil): Continue existing flow — generate tokens, create session, return
     auth_response.

2. Add `verify_login_2fa/2` action:
   - Extract `two_factor_token` and `code` from params
   - Look up temp token in Cachex `:two_factor_challenges`
   - If not found: return 401 "Invalid or expired two-factor token"
   - Load user by stored user_id
   - Try `Security.TOTP.verify(user, code)` — if ok, generate tokens + session + return
     auth_response
   - If TOTP fails, try `Security.TOTP.use_backup_code(user, code)` — if ok, do same
   - If both fail: return 401 "Invalid verification code"
   - On success: delete the temp token from Cachex (single-use)
   - Apply rate limiting (reuse `safe_clear_attempts` with stored lockout_key)

3. Verify the `Cachex` cache name — check if `:two_factor_challenges` already exists in
   application.ex. If not, either reuse an existing Cachex instance with a namespaced key, or add a
   new Cachex to the supervision tree.

**In router/auth_routes.ex:** 4. Add route: `post "/login/2fa", AuthController, :verify_login_2fa`
in the unauthenticated scope alongside the existing login route.

**Decision: OAuth bypasses 2FA** — OAuth callbacks already verify identity via the provider
(Google/Apple verified the user). Do NOT add 2FA gate to OAuth flows. Document this in a code
comment in oauth.ex: `# OAuth bypasses 2FA — provider already verified identity`. </action>
