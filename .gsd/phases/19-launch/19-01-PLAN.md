# Plan 19-01: Wallet Auth Completion

## Objective

Complete wallet-based authentication with Privy embedded wallets (AUTH-12) and polish existing
Ethereum wallet signature auth to SIWE standard (AUTH-13). After this plan, users have 5 auth
methods: email/password, OAuth (Google/Apple), TOTP 2FA, Privy (embedded wallet + social), and
direct Ethereum wallet signature.

## Requirements Covered

- **AUTH-12**: Privy auth (embedded wallet + social login) — NEW
- **AUTH-13**: Ethereum wallet signature auth — POLISH (SIWE upgrade, multi-wallet, mobile)

## Current State

- **AUTH-12**: 0% — No Privy code exists anywhere in the codebase
- **AUTH-13**: ~85% — Full EIP-191 backend + MetaMask web UI exists. Missing: SIWE standard message
  format, mobile wallet connection, WalletConnect support

## Wave 1 Execution Order

1. T1 → T2 → T3 → T4 (backend Privy, sequential — each builds on previous)
2. T11 (shared types, parallel with backend)
3. T5 → T6 → T7 (web Privy, after backend ready)
4. T8 → T9 (mobile Privy, parallel with web)
5. T10 (SIWE upgrade, independent — can run parallel with T5-T9)
