# Plan 15-04: Emoji Packs + Post Icons + RSS Polish

## Context

**FORUM-13 Emoji Packs & Post Icons:** Backend has `CustomEmoji` (262L), `EmojiPack` (72L), `EmojiCategory` (85L), `PostIcon` (155L), `CustomEmojiController` (445L). Web has picker and upload components. **Gaps:** No pack import/export, no post icon selector in thread creation, no mobile pack browser, no animated preview. Mobile has basic emoji management.

**FORUM-14 RSS Feeds:** Backend has `rss.ex` (170L), `feeds.ex` (171L), `RssController` (480L). Web has `rss-feed-button.tsx` (27L) and RSS feeds page at `pages/settings/rss-feeds/`. **Mobile already has `rss-feeds-screen.tsx` (486L) at `screens/settings/rss-feeds-screen.tsx`** — tasks below extend this, not create from scratch. **Gaps:** No per-board RSS feeds, no feed config panel, thin tests. These are smaller gaps.

This is a lighter plan combining two related "content delivery" features.

## Tasks

### Task 1: Emoji pack import/export backend

**Files:** `apps/backend/lib/cgraph/forums/emoji_pack.ex`, `apps/backend/lib/cgraph_web/controllers/api/v1/custom_emoji_controller.ex`

**Actions:**
1. Add to `EmojiPack` context:
   - `export_pack/1` (pack_id) — returns JSON bundle: `{name, version, emojis: [{shortcode, image_url, animated}]}`
   - `import_pack/2` (forum_id, json_bundle) — creates pack + downloads/stores emoji images
   - `list_available_packs/0` — returns public packs (marketplace)
2. Add controller endpoints:
   - `GET /forums/:fid/emoji-packs/:id/export` → download JSON
   - `POST /forums/:fid/emoji-packs/import` → upload JSON + import
   - `GET /emoji-packs/marketplace` → browse public packs
3. Import validates: max 500 emojis per pack, max 256KB per image, shortcode uniqueness within forum

### Task 2: Post icon integration in thread creation

**Files:** `apps/backend/lib/cgraph/forums/threads.ex`, `apps/backend/lib/cgraph_web/controllers/api/v1/thread_controller.ex` (if exists)

**Actions:**
1. Ensure thread creation accepts `post_icon_id` parameter
2. Validate `post_icon_id` belongs to the board (per-board restriction from `PostIcon`)
3. Thread response includes post icon data (emoji, name, color)
4. Add `list_post_icons/1` (board_id) to context for board-specific icons

### Task 3: Emoji pack manager web UI

**Files:** `apps/web/src/modules/forums/components/emoji-picker/emoji-pack-manager.tsx`, `apps/web/src/modules/forums/store/forumStore.emoji.ts`

**Actions:**
1. `EmojiPackManager` component:
   - List installed packs with emoji count, version, premium badge
   - "Create Pack" form: name, description, emoji upload drop zone (bulk upload)
   - "Import Pack" button: file picker for JSON bundle
   - "Export Pack" button: downloads JSON bundle
   - Per-pack actions: edit name, add/remove emojis, toggle active, delete
   - Emoji grid view within pack: shows all emojis with shortcodes, animated badge
   - Approval queue for pending emojis (if emoji moderation enabled)
2. Store slice `forumStore.emoji.ts`:
   - `fetchPacks`, `createPack`, `importPack`, `exportPack`, `deletePack`
   - `fetchEmojis(packId)`, `uploadEmoji`, `deleteEmoji`, `approveEmoji`

### Task 4: Custom emoji picker enhancements

**Files:** `apps/web/src/modules/forums/components/emoji-picker/custom-emoji-picker.tsx`

**Actions:**
1. Group emojis by pack with collapsible headers
2. Add animated emoji preview (.gif/.apng rendered in picker with play-on-hover)
3. Search by shortcode and alias
4. "Recently Used" section at top (stored in localStorage)
5. Category tabs matching `EmojiCategory` from backend
6. Emoji favorites (star icon, synced to server via favorites endpoint)

### Task 5: Post icon selector component

**Files:** `apps/web/src/modules/forums/components/emoji-picker/post-icon-selector.tsx`

**Actions:**
1. `PostIconSelector` component:
   - Grid of available post icons for the current board
   - Each icon shows emoji + name + color badge
   - Click to select, appears next to thread title in creation form
   - "None" option to create thread without icon
   - Shows "(Board default)" for the default icon if configured
2. Integrate into thread creation form (conditionally shown when board has post icons)

### Task 6: Per-board RSS feeds backend

**Files:** `apps/backend/lib/cgraph/forums/rss.ex`, `apps/backend/lib/cgraph_web/controllers/api/v1/rss_controller.ex`

**Actions:**
1. Add to `rss.ex`:
   - `list_board_threads_for_rss/2` (board_id, limit) — queries threads for a specific board ordered by most recent
   - `board_rss_enabled?/1` (board_id) — checks if RSS is enabled for this board
2. Add controller endpoint:
   - `GET /forums/:fid/boards/:bid/rss` → generates RSS 2.0 XML for a board's threads
   - Includes: title as thread title, link, description from first 200 chars, pubDate, author, category
3. Add `rss_enabled` boolean to boards table (migration) with default true
4. `GET /forums/:fid/boards/:bid/rss/atom` → Atom feed variant

### Task 7: RSS feed configuration panel

**Files:** `apps/web/src/modules/forums/components/rss-feed/rss-feed-config.tsx`, `apps/web/src/modules/forums/store/forumStore.rss.ts`

**Actions:**
1. `RssFeedConfig` component:
   - List all boards with RSS toggle per board
   - Global feed toggle for forum-wide feed
   - Feed URL display with copy button
   - Feed format preference (RSS 2.0 / Atom)
   - Items per feed setting (10/25/50)
2. Store slice `forumStore.rss.ts`:
   - `fetchRssConfig`, `updateBoardRss`, `toggleGlobalRss`
3. Admin-only panel integrated into forum settings

1. Admin creates emoji pack, uploads 10 emojis, exports as JSON bundle
2. Import pack JSON on different forum creates pack with all emojis
3. Post icon selector appears in thread creation when board has icons configured
4. Thread created with post icon displays icon next to title
5. `GET /forums/:fid/boards/:bid/rss` returns valid RSS 2.0 XML
6. Disabling RSS for a board returns 404 for that board's feed
7. Mobile emoji pack browser shows packs with animated previews
8. Mobile RSS subscribe sheet copies feed URL to clipboard
9. Custom emoji picker groups by pack with search and favorites
