# Plan 15-02: Plugin Execution Runtime + Forum Automod

## Context

**FORUM-08 Plugin System:** The backend has `ForumPlugin` (107L) with 18 hook events, `Plugins` context (268L) with install/uninstall/toggle, `PluginController` (175L), and routes. The web has a plugin marketplace page and store. **What's missing is the execution runtime** — hooks are defined but nothing dispatches them when events occur (thread created, post created, vote cast, etc.).

**FORUM-11 Forum Moderation:** The backend has `moderation.ex` (212L) with queue/hide/delete/ban/flag, `ContentReport` (65L, 6 reasons: spam/harassment/hate_speech/misinformation/nsfw/other, 4 statuses), `Moderator` (33L), `Ban` (46L). Web has a comprehensive moderation store with 8 slices and UI components including automod settings. **Gaps:** Current automod is group-level (at `/groups/:groupId/automod`), needs forum-specific automod. No warning/strike system exists. Mobile moderation is groups-only.

## Tasks

### Task 1: Create plugin execution runtime

**Files:** `apps/backend/lib/cgraph/forums/plugin_runtime.ex`, `apps/backend/test/cgraph/forums/plugin_runtime_test.exs`

**Actions:**
1. Create `CGraph.Forums.PluginRuntime` module (using existing `CGraph.TaskSupervisor` from worker_supervisor.ex):
   ```elixir
   defmodule CGraph.Forums.PluginRuntime do
     @doc "Dispatch a hook event to all installed plugins that listen for it"
     def dispatch(forum_id, hook_event, payload) do
       forum_id
       |> Plugins.get_active_by_hook(hook_event)
       |> Enum.each(fn plugin ->
         Task.Supervisor.async_nolink(CGraph.TaskSupervisor, fn ->
           execute_hook(plugin, hook_event, payload)
         end)
       end)
     end
   ```
2. Supported hook events (from `ForumPlugin.hooks`):
   - `:thread_created`, `:thread_updated`, `:thread_deleted`
   - `:post_created`, `:post_updated`, `:post_deleted`
   - `:vote_cast`, `:poll_voted`
   - `:member_joined`, `:member_left`, `:member_banned`
   - `:report_filed`, `:moderation_action`
   - `:theme_changed`, `:settings_updated`
3. Each hook execution is wrapped in try/rescue — one plugin failure must not affect others
4. Add `execute_hook/3` that evaluates plugin's hook handler code or calls built-in behavior
5. Log hook execution results with structured logging
6. Write tests verifying dispatch, isolation, and error handling

### Task 2: Wire plugin dispatch into forum lifecycle

**Files:** `apps/backend/lib/cgraph/forums/threads.ex`, `apps/backend/lib/cgraph/forums/moderation.ex`, `apps/backend/lib/cgraph/forums/forum_voting.ex`

**Actions:**
1. After `create_thread/3` succeeds → `PluginRuntime.dispatch(forum_id, :thread_created, %{thread: thread})`
2. After `create_thread_post/3` succeeds → `PluginRuntime.dispatch(forum_id, :post_created, %{post: post})`
3. After `vote_thread/3` or `vote_post/3` → `PluginRuntime.dispatch(forum_id, :vote_cast, %{vote: vote})`
4. After `flag_content/3` → `PluginRuntime.dispatch(forum_id, :report_filed, %{report: report})`
5. After ban/unban → `PluginRuntime.dispatch(forum_id, :member_banned, ...)`
6. Dispatches are non-blocking (fire-and-forget via Task.Supervisor)

### Task 4: Forum-level automod rules

**Files:** `apps/backend/lib/cgraph/forums/forum_automod.ex`, `apps/backend/test/cgraph/forums/forum_automod_test.exs`

**Actions:**
1. Create `CGraph.Forums.ForumAutomod` context:
   - `get_rules/1` (forum_id) — fetch forum automod config
   - `update_rules/2` (forum_id, rules) — save rules
   - `check_content/2` (forum_id, content) — run content through all active rules
   - Rules: `word_filter` (banned words list), `link_filter` (blocked domains), `spam_detection` (repeat rate, min account age), `caps_filter` (max caps percentage)
2. `check_content/2` returns `{:ok, :clean}` or `{:flag, reason}` or `{:block, reason}`
3. Actions per rule: `:flag` (adds to mod queue), `:block` (prevents posting), `:shadow_ban` (post hidden from others)
4. Store rules in `forums` table `automod_rules :map` column (add via migration if needed)
5. Wire `check_content` into `create_thread/3` and `create_thread_post/3` as pre-check

### Task 5: Warning/strike system

**Files:** `apps/backend/lib/cgraph/forums/warning.ex`, `apps/backend/priv/repo/migrations/*_add_forum_warnings.exs`

**Actions:**
1. Create `Warning` schema:
   - `forum_id` — belongs_to Forum
   - `user_id` — belongs_to User
   - `issued_by_id` — belongs_to User (moderator)
   - `reason` — string
   - `points` — integer (severity: 1=mild, 5=severe)
   - `expires_at` — utc_datetime (optional, nil = permanent)
   - `acknowledged` — boolean
   - `revoked` — boolean
   - `revoked_by_id` — optional
2. Add to `moderation.ex`: `warn_user/4`, `list_warnings/2`, `revoke_warning/2`, `active_warning_points/2`
3. Auto-action thresholds: 3 points → temp mute (24h), 6 points → temp ban (7d), 10 points → permanent ban
4. Create migration for `forum_warnings` table
5. Add warning endpoints to moderation controller

**Actions:**

   - `queue/2` — GET /forums/:id/moderation/queue — flagged content for forum
   - `action/2` — POST /forums/:id/moderation/action — take action (approve/reject/hide/delete)

   - `automod/2` — GET /forums/:id/moderation/automod — get automod rules
   - `update_automod/2` — PUT /forums/:id/moderation/automod — update automod rules
   - `stats/2` — GET /forums/:id/moderation/stats — mod queue count, recent actions, active warnings
2. Authorization: forum moderator or admin
3. Add routes to `forum_routes.ex`

## Verification

1. `PluginRuntime.dispatch(forum_id, :thread_created, payload)` fires hooks on installed plugins
2. One plugin raising an exception does not prevent other plugins from executing
3. Forum automod catches banned words and flags content to mod queue
4. Warning issued to user accumulates points, auto-mute triggers at 3 points
5. `GET /forums/:id/moderation/queue` returns forum-specific flagged content
6. Web mod dashboard shows queue, can approve/reject items, can issue warnings
7. Forum automod settings save and enforce on new posts
8. Mobile mod screen shows queue with swipe actions
9. Plugin config panel renders plugin settings and saves changes
