# Plan 15-01: Forum Customization Engine — 50+ Options

## Context

FORUM-07 requires 50+ enumerable customization options. The backend has `ForumTheme` (96L) with colors, CSS, fonts, layout fields, `Forum` schema has custom_css/sidebar_html/header/footer fields, and `ThemeController` (159L) exists. The web has `forumThemeStore.ts` (200L) with 3 presets and a theme provider. Mobile has a basic settings screen.

**What's missing is the full enumeration of 50+ options, a unified customization center UI, several backend schemas (custom fields, rank images, karma names), and a dedicated forum theme CRUD controller** (existing `theme_controller.ex` handles user profile themes via `CGraph.Themes`, not forum themes).

Existing options count:
- Theme presets: 3 (dark-elite, cyberpunk, classic-mybb) → expand to 8+
- Colors: 6 fields on ForumTheme → 6 options
- Fonts: font_family field → 1 option  
- Layout: sidebar_position, content_width → 2 options
- Custom CSS: 1 field → 1 option
- Header: logo, background → 2 options
- Sidebar: raw HTML → needs widget system

Need to reach 50+ by adding: widget config (10+), post templates (5+), custom fields (configurable), badges (5+), karma names (3+), rank images (5+), and more granular theme controls.

### Task 2: Add custom fields schema and migration

**Files:** `apps/backend/lib/cgraph/forums/custom_field.ex`, `apps/backend/priv/repo/migrations/*_add_forum_custom_fields.exs`

**Actions:**
1. Create `CustomField` schema:
   - `forum_id` — belongs_to Forum
   - `name` — field label
   - `field_type` — enum: text, number, select, checkbox, date, url
   - `target` — enum: thread, post, profile (what entity this field attaches to)
   - `options` — array of strings (for select type)
   - `required` — boolean
   - `position` — integer for ordering
   - `visible_to` — array of group IDs (visibility control)
2. Create migration adding `custom_fields` table and `customization_options :map` column to `forums`
3. Add context functions: `create_field/2`, `update_field/2`, `delete_field/1`, `list_fields/2` (by forum + target)

### Task 3: Create forum customization controller and forum theme CRUD

**Files:** `apps/backend/lib/cgraph_web/controllers/api/v1/customization_controller.ex`, `apps/backend/lib/cgraph_web/controllers/api/v1/forum_theme_controller.ex`, `apps/backend/lib/cgraph_web/router/forum_routes.ex`

**Note:** The existing `theme_controller.ex` handles **user profile themes** (avatar borders, chat bubbles, visual effects) via `CGraph.Themes` — NOT forum themes. A new dedicated `ForumThemeController` is needed for forum theme CRUD, while `CustomizationController` handles the 55-option configuration.

**Actions:**
1. Create `ForumThemeController` — CRUD for `ForumTheme` records (create/show/update/delete presets)
2. Create `CustomizationController`:
   - `show/2` — GET /forums/:id/customization — returns all 55 option values
   - `update/2` — PUT /forums/:id/customization — update category of options (admin auth required)
   - `list_options/2` — GET /forums/:id/customization/options — returns option metadata (names, types, defaults)
   - `preview/2` — POST /forums/:id/customization/preview — returns computed CSS for live preview
2. Add routes to `forum_routes.ex`
3. Authorization: forum admin or owner only
