# Plan 15-05: Ranking Integration + Leaderboard + Gamification Bridge

## Context

**FORUM-16 Ranking & Leaderboard:** Backend has `RankingEngine` (252L) with 7 algorithms (hot, top, rising, etc.), `Leaderboard` (71L), `UserLeaderboard` (118L), `LeaderboardController` (181L). Gamification has `LeaderboardSystem` (211L) and `Leaderboard` (313L). Web has widgets and page stubs. Mobile has animated leaderboard screen (318L) with 4 animated sub-components (animated-list-item, animated-period-selector, animated-podium, animated-tab-bar).

**Gaps:**
- Forum `RankingEngine` and gamification `LeaderboardSystem` are completely separate — no bridge
- No Oban cron job for `update_all_rankings()` — **note: `:rankings` queue must be added to Oban config in `config.exs` and `prod.exs`**
- No custom karma names per forum
- No `ForumRank` schema for score→rank→image progression
- Forum leaderboard page is a 15-line stub
- No rank badges displayed in posts

## Tasks

### Task 1: Create ForumRank schema for rank progression

**Files:** `apps/backend/lib/cgraph/forums/forum_rank.ex`, `apps/backend/priv/repo/migrations/*_add_forum_ranks.exs`

**Actions:**
1. Create `ForumRank` schema:
   - `forum_id` — belongs_to Forum
   - `name` — rank name (e.g., "Newcomer", "Regular", "Veteran", "Legend")
   - `min_score` — integer threshold to achieve this rank
   - `max_score` — integer upper bound (nil for highest rank)
   - `image_url` — rank image/badge URL
   - `color` — hex color for rank badge
   - `position` — integer ordering
   - `is_default` — boolean (the "unranked" base rank)
2. Create migration for `forum_ranks` table
3. Context functions: `create_rank/2`, `update_rank/2`, `delete_rank/1`, `list_ranks/1`, `get_rank_for_score/2`
4. Seed default ranks: Newcomer (0), Regular (50), Veteran (200), Expert (500), Legend (1000)

### Task 2: Create Oban ranking update worker

**Files:** `apps/backend/lib/cgraph/workers/ranking_update_worker.ex`

**Actions:**
1. Create `CGraph.Workers.RankingUpdateWorker` using Oban:
   ```elixir
   use Oban.Worker, queue: :rankings, max_attempts: 3
   
   @impl Oban.Worker
   def perform(%Oban.Job{args: %{"forum_id" => forum_id}}) do
     RankingEngine.update_all_rankings(forum_id)
     # Note: reset_weekly_scores/1 lives in ranking_engine.ex, NOT user_leaderboard.ex
     RankingEngine.reset_weekly_scores(forum_id) # if weekly boundary crossed
     :ok
   end
   ```
2. Register cron job in application config:
   - `{"0 * * * *", CGraph.Workers.RankingUpdateWorker}` — hourly for active forums
   - `{"0 0 * * 1", CGraph.Workers.RankingUpdateWorker, args: %{type: "weekly_reset"}}` — weekly reset
3. Add `enqueue_ranking_update/1` for on-demand updates (admin trigger)

### Task 3: Bridge forum rankings to gamification

**Files:** `apps/backend/lib/cgraph/forums/ranking_engine.ex`, `apps/backend/lib/cgraph/gamification/leaderboard_system.ex`, `apps/backend/lib/cgraph/gamification/xp_transaction.ex`

**Actions:**
1. Add XP grant triggers for forum actions:
   - Thread created → +10 XP (`forum_thread_created` source)
   - Post/reply created → +5 XP (`forum_post_created` source)
   - Upvote received → +2 XP (`forum_upvote_received` source)
   - Best answer marked → +20 XP (`forum_best_answer` source)
   - Apply daily caps: max 200 XP from forum activity per day
   - **Note:** Daily XP cap is entirely new infrastructure — no existing cap/rate-limiting mechanism in the gamification system. Will need a new `forum_xp_daily_totals` table or Redis counter to track per-user daily XP from forum activity.
2. Add to `RankingEngine`:
   - `calculate_unified_score/2` (forum_id, user_id) — combines forum karma + gamification XP into weighted score
   - Weight formula: `forum_karma * 1.0 + xp * 0.1` (configurable per forum)
3. Add to `LeaderboardSystem`:
   - `get_forum_leaderboard/2` (forum_id, opts) — queries unified scores, returns ranked list
   - Opts: `period` (all_time | monthly | weekly | daily), `limit` (default 50), `cursor`

### Task 4: Extend leaderboard controller

**Files:** `apps/backend/lib/cgraph_web/controllers/api/v1/leaderboard_controller.ex`, `apps/backend/lib/cgraph_web/router/forum_routes.ex`

**Actions:**
1. Add endpoints:
   - `GET /forums/:id/leaderboard` — unified leaderboard with period filter
   - `GET /forums/:id/leaderboard/my-rank` — current user's rank, score, progress to next rank
   - `GET /forums/:id/ranks` — list configured ranks with thresholds
   - `PUT /forums/:id/ranks` — update ranks (admin)
   - `POST /forums/:id/leaderboard/refresh` — trigger ranking recalculation (admin)
2. Leaderboard response includes: position, user (name, avatar), score, rank (name, image, color), change (up/down from previous period)
