<task type="auto">
  <name>Task 3: Build Discovery context with feed ranking</name>
  <files>apps/backend/lib/cgraph/discovery/</files>
  <action>
    Create NEW CGraph.Discovery context module tree:

    1. discovery.ex (facade — delegates to sub-modules):
       - get_user_frequencies/1, update_frequencies/2
       - list_topics/0
       - get_community_health/1
       Delegates feed operations to Discovery.Feed

    2. topic.ex (Ecto schema for topics table):
       - Schema: name, icon, slug
       - Changeset with validations

    3. user_frequency.ex (Ecto schema for user_frequencies table):
       - Belongs to user + topic
       - Composite PK, weight 0-100

    4. post_metric.ex (Ecto schema for post_metrics table):
       - Schema: weighted_resonates, reply_depth_avg, read_time_signal, cross_community_refs

    5. feed.ex — Feed ranking with 5 modes:
       a. PULSE: Ranked by composite score (PostgreSQL computed):
          weighted_resonates * 0.40 + reply_depth_avg * 0.25 + read_time_signal * 0.20
          + cross_community_refs * 0.10 - (age_days * 0.05)
          Filter to forums the user follows (join user_frequencies for weight)
       b. FRESH: Chronological (newest first), filtered to user's frequency topics
       c. RISING: Posts from last 24h with highest engagement velocity
          (resonates/hour since creation)
       d. DEEP_CUT: Older posts (>3 days) with high weighted_resonates but
          low view_count — surfaces niche quality content
       e. FREQUENCY_SURF: Filter to a specific community's posts (community_id param)

       All modes use cursor pagination (CGraph.Forums.CursorPagination pattern).
       All modes query the existing `posts` or `threads` tables, joining
       `post_metrics` for ranking data.

    NOTE: This is SEPARATE from CGraph.Forums.Feeds (which handles basic
    public/home/popular feeds). Discovery.Feed adds ranked discovery feeds.
    Module: CGraph.Discovery (NOT CGraph.Forums.Discovery)

  </action>
  <verify>cd apps/backend && mix compile 2>&1 | tail -5</verify>
  <done>Discovery context with 5 feed modes implemented</done>

<task type="auto">
  <name>Task 4: Build community health scoring</name>
  <files>apps/backend/lib/cgraph/discovery/community_health.ex</files>
  <action>
    Implement backend-only community health score:

    CGraph.Discovery.CommunityHealth.compute(forum_id):
    - Query forum metrics from existing CGraph.Forums tables:
      - positive_ratio: positive reactions / total reactions
      - new_voice_ratio: posts by users with <90 days membership / total posts
      - reply_depth: avg reply depth (normalized, target > 2)
      - mod_action_penalty: min(1.0, mod_removal_rate * 5)
      - dominance_penalty: min(1.0, top_poster_share * 2)

    Health = positive_ratio * 0.30 + new_voice_ratio * 0.25
             + min(1.0, reply_depth / 3) * 0.25
             - mod_action_penalty * 0.10 - dominance_penalty * 0.10

    Cache in ETS (table :community_health_cache, TTL 15 minutes).
    NEVER expose raw score to API — used internally for feed ranking.
    Used by Discovery.Feed.rank_pulse/3 to boost healthy community content.

    NOTE: Uses existing forum data. Queries against forums, thread_posts,
    forum_members tables. No new tables needed for this.

  </action>
  <verify>cd apps/backend && mix compile 2>&1 | tail -5</verify>
  <done>Community health scoring functional with ETS cache</done>
