# Plan 13-03: Persistent Voice Channels

## Context

Discord-style persistent voice channels are always-on rooms that users join/leave freely. Unlike call invitations (ring → accept → connect), voice channels show in the channel sidebar with connected user avatars. Users click to join, and others see them appear.

**What exists:**
- `ChannelType` in shared-types already includes `'voice'` type
- Voice permission bits exist: `CONNECT`, `SPEAK`, `VIDEO`, `MUTE_MEMBERS`, `DEAFEN_MEMBERS`
- Channel list UI on web and mobile already renders different channel types
- LiveKit integration from Plan 13-02 provides the SFU backbone
- Phoenix Presence is used elsewhere in the app for user online/offline status

**What's needed:**
1. Backend voice channel manager tracking per-channel occupancy
2. Phoenix Channel for voice state events (join/leave/mute/deafen)
3. Channel list UI to show voice channels with live occupancy
4. Voice channel panel (full-size view when in a voice channel)
5. Single-channel constraint (one voice channel at a time)
6. Mobile voice channel screen

## Tasks

### Task 1: Backend voice channel manager

**Files:** `apps/backend/lib/cgraph/webrtc/voice_channel_manager.ex`

**Actions:**
1. Create `VoiceChannelManager` module (uses Phoenix Presence for state tracking):
   - `join_voice_channel(channel_id, user_id, opts)`:
     - Auto-leave any previous voice channel for this user
     - Track user in Phoenix Presence for `voice:channel_id`
     - If LiveKit room doesn't exist for channel, create it
     - Return LiveKit token for the room
   - `leave_voice_channel(channel_id, user_id)`:
     - Untrack from Presence
     - If last user leaves, optionally clean up LiveKit room (or keep alive)
   - `get_voice_channel_members(channel_id)` → list of `{user_id, mute_state, deafen_state}`
   - `get_user_voice_state(user_id)` → `{channel_id, mute, deafen} | nil`
   - `update_voice_state(channel_id, user_id, state)` — update mute/deafen/video
2. Use ETS or Presence for tracking (Presence preferred — auto-cleanup on disconnect)
3. LiveKit room naming: `vc_{channel_id}` for voice channels

### Task 2: Backend voice state Phoenix Channel

**Files:** `apps/backend/lib/cgraph_web/channels/voice_state_channel.ex`, `apps/backend/lib/cgraph_web/channels/group_channel.ex`

**Actions:**
1. Create `VoiceStateChannel` (`"voice:channel_id"`):
   - `join/3` — verify user has CONNECT permission for this channel, call `VoiceChannelManager.join_voice_channel`, return LiveKit token
   - Handle `"mute"` / `"unmute"` / `"deafen"` / `"undeafen"` — update Presence metadata, broadcast to channel
   - Handle `"leave"` — graceful leave, Presence auto-tracks disconnect
   - Broadcast events: `voice_state_update` with `{user_id, channel_id, self_mute, self_deafen}`
2. Enhance `GroupChannel` to broadcast voice state to all group members:
   - When a user joins/leaves a voice channel, broadcast `voice_member_update` to `group:group_id`
   - Group members can see channel occupancy without joining the voice channel

### Task 3: Web voice state store + hook

**Files:** `apps/web/src/stores/voiceStateStore.ts`, `apps/web/src/modules/calls/hooks/useVoiceChannel.ts`

**Actions:**
1. Create `voiceStateStore` (Zustand):
   - State: `currentChannelId`, `currentRoomToken`, `members` (Map<channelId, VoiceMember[]>), `isMuted`, `isDeafened`, `isVideoOn`
   - Actions: `joinChannel(channelId)`, `leaveChannel()`, `toggleMute()`, `toggleDeafen()`, `toggleVideo()`
   - Subscribe to group channel `voice_member_update` events to update members map
2. Create `useVoiceChannel` hook:
   - Wraps voiceStateStore + useLiveKitRoom from Plan 13-02
   - On `joinChannel`: connect to `voice:channel_id` Phoenix Channel → get token → connect to LiveKit
   - Auto-leave previous channel when joining new one
   - On disconnect/unmount: leave gracefully
   - Expose: `participants`, `isMuted`, `isDeafened`, `isConnected`, controls

### Task 4: Web voice channel list item

**Files:** `apps/web/src/modules/groups/components/channel-list/voice-channel-item.tsx`

**Actions:**
1. Create `VoiceChannelItem` component:
   - Show channel name with speaker icon
   - Below channel name: list of connected user avatars (max 5 visible + "+N" overflow)
   - Each avatar shows mute indicator (red slash) if muted
   - Click → calls `voiceStateStore.joinChannel(channelId)`
   - If user is already in this channel, show "Connected" badge
   - If user is in a DIFFERENT voice channel, show "Switch?" confirmation
2. Integrate into existing channel list (replace generic channel item for `type === 'voice'`)

### Task 5: Web voice channel panel

**Files:** `apps/web/src/modules/groups/components/voice-channel-panel.tsx`

**Actions:**
1. Create `VoiceChannelPanel` overlay (shown at bottom of sidebar when connected):
   - Shows current voice channel name
   - Participant list with speaking indicators (from LiveKit `activeSpeakers`)
   - Control bar: mute, deafen, video, screen share, disconnect
   - Click channel name → scrolls to/highlights channel in list
   - Disconnect button → leaves voice channel + LiveKit room
2. Panel is persistent (stays visible while navigating text channels)
3. Connection quality indicator using LiveKit stats

### Task 6: Mobile voice channel screen

**Files:** `apps/mobile/src/screens/groups/voice-channel-screen.tsx`, `apps/mobile/src/stores/voiceStateStore.ts`

**Actions:**
1. Create mobile `voiceStateStore.ts` (mirrors web):
   - Same state shape and actions as web store
   - Adapts to mobile socket manager for channel events
2. Create `VoiceChannelScreen`:
   - Full-screen participant grid (LiveKit tracks via `@livekit/react-native`)
   - Active speaker highlight
   - Control bar: mute, deafen, speaker/earpiece toggle, disconnect
   - Participant list (collapsible)
   - Background audio continues when navigating away (using `InCallManager` or `VoIpAudioSession`)

5. Voice channel panel persists while browsing text channels
6. Mobile has a functional voice channel screen with LiveKit-backed audio
