## Context

The web customization pages were built with mock data during rapid UI development. The backend
APIs for all gamification features exist and return real data. The gap is purely a frontend
wiring issue тАФ the pages import `MOCK_*` constants instead of calling the API.

**Current mock data locations:**
- `pages/customize/progression-customization/mock-data.ts` тАФ MOCK_ACHIEVEMENTS, MOCK_LEADERBOARD,
  MOCK_QUESTS, MOCK_DAILY_REWARDS (also has `@todo(api) Create achievements API endpoints`)
- `pages/customize/identity-customization/constants.ts` тАФ MOCK_BORDERS, MOCK_TITLES, MOCK_BADGES
- `pages/customize/theme-customization/constants.ts` тАФ MOCK_THEMES
- `pages/forums/forum-admin/useForumAdminInit.ts` тАФ Lines 100, 116, 139, 163 all contain
  `// Mock` data for analytics, rules, mod queue, members

**Backend endpoints (verified):**
- `GET /api/v1/achievements` тЖТ AchievementController
- `GET /api/v1/leaderboard` тЖТ GamificationController
- `GET /api/v1/quests` тЖТ GamificationController
- `GET /api/v1/daily-rewards` тЖТ GamificationController
- `GET /api/v1/cosmetics/borders|titles|badges` тЖТ CosmeticsController
- `GET /api/v1/themes` тЖТ CosmeticsController
- `GET /api/v1/forums/:id/analytics|members` тЖТ ForumController
- `GET /api/v1/forums/:id/automod/rules` тЖТ ForumAutoModController
- `GET /api/v1/moderation/reports` тЖТ ModerationController
