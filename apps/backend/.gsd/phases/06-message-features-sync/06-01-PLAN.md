<objective>
Wire MessageEdit record creation into the edit_message flow and add the EditHistory shared type so clients can display previous versions.
Purpose: The MessageEdit schema and table exist but are never populated — this closes that gap so edit history is actually recorded and retrievable.
Output: Backend inserts MessageEdit rows on every edit; shared-types exports an EditHistory interface consumed by web and mobile in Wave 2.
</objective>

<context>
**What exists:**
- `MessageOperations.edit_message/3` at apps/backend/lib/cgraph/messaging/message_operations.ex — calls `update_message/2` but does NOT insert a `MessageEdit` record
- `MessageEdit` schema at apps/backend/lib/cgraph/messaging/message_edit.ex with fields: `previous_content`, `edit_number`, `edited_by_id`, `message_id`
- `Message` schema has `has_many :edits, MessageEdit` and `is_edited: true` flag set by `edit_changeset/2`
- `message_edits` table exists in DB (migration already run)
- Shared types at packages/shared-types/src/models.ts — `Message` has `isEdited` (L81) but no `EditHistory` or `edits` field

**Key references:**

- Backend edit handler: apps/backend/lib/cgraph_web/channels/conversation_channel.ex L233
- MessageEdit schema: apps/backend/lib/cgraph/messaging/message_edit.ex
- Shared types: packages/shared-types/src/models.ts </context>

       - Step 1 (`:count_edits`): Count existing MessageEdit records for this message to determine `edit_number`

    2. Return the updated message with edits preloaded
