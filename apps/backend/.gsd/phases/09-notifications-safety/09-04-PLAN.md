# Plan 09-04: Account Deletion Polish & Phase Integration

## Context

Account deletion (AUTH-08) is already fully implemented with soft delete → Oban hard delete, data export, and UI on both platforms. This plan ensures:

1. **New data from 09-01/09-02 is covered** — notification preferences and DND settings are deleted
2. **Full cascade verification** — all Phase 1-8 data is properly handled
3. **GDPR checklist** — data export completeness, push token cleanup, presence cleanup

## Tasks

### Cascade Updates

1. **Add notification_preferences to hard_delete cascade**
   - In `hard_delete_user.ex`, add cleanup for `notification_preferences` table
   - Delete all rows where `user_id = deleted_user_id`
   - This should be straightforward — add one more `Repo.delete_all` query

2. **Include notification preferences in data export**
   - In `data_export.ex`, add `notification_preferences` to the exported data sections
   - Format: list of `{target_type, target_id, mode, muted_until}`

3. **Push token cleanup on soft delete**
   - Verify: when user initiates deletion, all push tokens are revoked
   - Check `account_deletion_controller.ex` → does it call `PushTokens.delete_all_for_user/1`?
   - If not, add it during soft delete (not hard delete — we want immediate stop of push)

4. **Presence cleanup on deletion**
   - When soft delete occurs, broadcast user as offline via presence channel
   - Verify: `PresenceChannel` handles user deletion gracefully

### GDPR Compliance Verification

5. **Data export completeness audit**
   - Verify data export includes:
     - [x] User profile (name, email, avatar URL, bio)
     - [x] Messages (content, timestamps, conversation IDs)
     - [x] E2EE public keys (not private — those stay on device)
     - [x] Social data (friends, blocked users)
     - [x] Settings (all preferences)
     - [x] Notification preferences (from 09-01)
     - [x] Push token registrations (platform, device info — not the token itself)
     - [x] Login history / sessions
   - Fix any missing sections

6. **Hard delete completeness audit**
   - Verify hard delete covers:
     - Messages: content anonymized (not deleted — other users need conversation integrity)
     - E2EE keys: crypto material deleted
     - Profile: avatar deleted from storage, PII wiped
     - Social: friend relationships deleted, blocks deleted
     - Push tokens: deleted
     - Sessions: deleted
     - Notification preferences: deleted
     - Forum posts: anonymized (author replaced with "Deleted User")

### Integration Tests

7. **Write `phase9_verification_test.exs`**
   - Test: create user → set notification preferences → delete account → preferences gone
   - Test: create user → register push token → delete account → token cleaned up
   - Test: create user → delete account → 30 day grace period → hard delete runs → all PII gone
   - Test: create user → initiate deletion → cancel within grace period → account restored
   - Test: data export includes all expected sections

### Cross-Platform Verification

8. **Verify web delete account flow**
   - Settings → Delete Account → type "DELETE" → confirm password → account soft-deleted
   - Verify: user is logged out, redirected to landing page
   - Verify: push tokens revoked, presence cleared

9. **Verify mobile delete account flow**
   - Settings → Account → Delete Account → same confirmation flow
   - Verify: same cleanup as web

## Verification

- Delete account → verify all notification preferences removed from DB
- Delete account → verify push tokens cleared
- Delete account → verify other users see user as offline
- Request data export → verify all data categories present
- Cancel deletion within grace period → verify account fully restored
