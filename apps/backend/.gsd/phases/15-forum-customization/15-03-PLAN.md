# Plan 15-03: User Groups Admin + Per-Board Permissions UI

## Context

**FORUM-15 User Groups:** Backend is ~90% complete — `ForumUserGroup` (213L) with 30+ permissions, `UserGroups` context (124L) with CRUD + default groups, `MemberSecondaryGroup` (209L) with expiration and OR-logic stacking, `GroupAutoRule` (323L) with auto-assignment rules, `SecondaryGroupsController` (447L) with full API. **Web has ~10%** — no dedicated admin pages. Mobile has ~5%.

**FORUM-12 Per-Board Permissions:** Backend is ~90% complete — `BoardPermission` (285L) with 15 fields and inherit/allow/deny, `ForumPermission` (303L), `PermissionTemplate` (241L) with presets, `PermissionsController` (470L) with full CRUD + apply template + check. **Web has ~55%** — forum-level panel exists but no board-level overrides UI, no template management. Mobile has 0%.

This plan is primarily **frontend admin UI work** — the backend APIs are fully functional.

## Tasks

### Task 1: User groups store slice

**Files:** `apps/web/src/modules/forums/store/forumStore.userGroups.ts`

**Actions:**
1. Create store slice with state: `groups`, `members`, `secondaryGroups`, `autoRules`
2. Actions:
   - `fetchGroups(forumId)` — GET /forums/:id/user-groups
   - `createGroup(forumId, data)` — POST /forums/:id/user-groups
   - `updateGroup(groupId, data)` — PUT /forums/:id/user-groups/:gid
   - `deleteGroup(groupId)` — DELETE /forums/:id/user-groups/:gid
   - `reorderGroups(forumId, groupIds)` — PUT /forums/:id/user-groups/reorder
   - `fetchMembers(groupId)` — GET /forums/:id/user-groups/:gid/members
   - `assignSecondaryGroup(userId, groupId, expiry?)` — POST /secondary-groups/assign
   - `fetchAutoRules(forumId)` — GET /forums/:id/user-groups/auto-rules
   - `createAutoRule(forumId, data)` — POST /auto-rules
   - `evaluateRules(forumId)` — POST /secondary-groups/evaluate-rules
3. Wire to `SecondaryGroupsController` and group endpoints

### Task 2: User group manager page

**Files:** `apps/web/src/modules/forums/components/user-groups/user-group-manager.tsx`, `apps/web/src/pages/forums/forum-user-groups.tsx`

**Actions:**
1. `UserGroupManager` component:
   - List all groups sorted by position with drag-to-reorder
   - Each group row: name, color badge, member count, staff indicator, edit/delete buttons
   - "Create Group" button opens create form
   - Group editor: name, color, description, is_staff, position, is_default
   - Default groups (Admin, Moderator, Member, Guest) marked as system, non-deletable
2. Page route: `/forums/:forumId/admin/user-groups`
3. Delete confirmation dialog warns about member reassignment

### Task 3: Group permissions matrix

**Files:** `apps/web/src/modules/forums/components/user-groups/group-permissions-matrix.tsx`

**Actions:**
1. Render a matrix grid: rows = groups (sorted by position), columns = permissions (30+ boolean fields from ForumUserGroup)
2. Group permissions into categories: Content (can_post, can_reply, can_create_thread, can_edit_own, can_delete_own), Moderation (can_moderate, can_warn_users, can_manage_reports, can_ban), Admin (can_manage_forum, can_manage_boards, can_manage_groups)
3. Each cell is a checkbox toggle
4. Row header shows group name + color badge
5. Column header shows permission name with tooltip description
6. "Save All" button bulk-updates changed permissions
7. Highlight cells that differ from the group above (visual diff)

### Task 4: Secondary group panel

**Files:** `apps/web/src/modules/forums/components/user-groups/secondary-group-panel.tsx`

**Actions:**
1. Show list of available secondary groups with current member counts
2. "Assign to User" form: user search + group select + optional expiry date
3. "View Members" expandable list per secondary group showing user + assignment date + expiry
4. "Remove" button to revoke secondary group membership
5. Show how permissions stack (OR logic — most permissive wins)
6. Expiring-soon badge for memberships within 7 days of expiry

### Task 5: Auto-rule editor

**Files:** `apps/web/src/modules/forums/components/user-groups/auto-rule-editor.tsx`

**Actions:**
1. List existing auto-rules with rule type, criteria, target group, status (active/paused)
2. "Create Rule" form:
   - Rule type: milestone (post_count, thread_count, reputation), time_based (member_since), subscription (premium tier), custom (JSONB criteria)
   - Threshold value
   - Target group (select from secondary groups)
   - Active toggle
3. "Evaluate Now" button triggers rule evaluation for all forum members
4. Rule templates: "Active Poster" (50+ posts → VIP), "Veteran" (6 months → Senior), "Top Contributor" (1000+ karma → Elite)
5. Shows last evaluation time and members affected count

### Task 6: Permissions store slice

**Files:** `apps/web/src/modules/forums/store/forumStore.permissions.ts`

**Actions:**
1. Create store slice with state: `boardPermissions`, `templates`, `forumPermissions`
2. Actions:
   - `fetchBoardPermissions(boardId)` — GET /boards/:id/permissions
   - `updateBoardPermission(boardId, groupId, perms)` — PUT /boards/:id/permissions
   - `fetchTemplates(forumId)` — GET /forums/:id/permission-templates
   - `createTemplate(forumId, data)` — POST /forums/:id/permission-templates
   - `applyTemplate(boardId, templateId)` — POST /boards/:id/permissions/apply-template
   - `checkPermission(forumId, action)` — GET /forums/:id/permissions/check
3. Wire to `PermissionsController` endpoints

### Task 7: Board permissions panel

**Files:** `apps/web/src/modules/forums/components/forum-permissions/board-permissions-panel.tsx`, `apps/web/src/pages/forums/board-permissions.tsx`

**Actions:**
1. `BoardPermissionsPanel` component:
   - Board selector dropdown at top
   - For selected board: grid of groups × permissions with inherit/allow/deny radio buttons
   - "Inherit from forum" is the default state (shown as dimmed)
   - "Allow" overrides to yes, "Deny" overrides to no
   - Visual indicator of effective permission (inheritance chain resolution)
2. Page route: `/forums/:forumId/boards/:boardId/permissions`
3. "Reset to Inherit All" button clears board-level overrides

### Task 8: Permission template manager

**Files:** `apps/web/src/modules/forums/components/forum-permissions/permission-template-manager.tsx`

**Actions:**
1. List templates: system presets (Read Only, Full Access, Moderate, Post Only) + custom
2. "Create Template" form: name, description, set each permission, save
3. "Apply to Board" action: select boards, preview changes, confirm apply
4. "Duplicate Template" for creating variants
5. System templates shown as non-editable with "Duplicate" option

### Task 9: Permission matrix (cross-board view)

**Files:** `apps/web/src/modules/forums/components/forum-permissions/permission-matrix.tsx`

**Actions:**
1. Full 3D matrix view: boards (columns) × groups (rows) × permissions (accordion sections)
2. Cell shows effective permission (green=allowed, red=denied, gray=inherited)
3. Hover shows inheritance chain: "Allowed by Board Override" or "Inherited from Forum → Allowed"
4. Export as CSV for auditing
5. Search/filter by group or board name

### Task 10: Mobile user groups screen

**Files:** `apps/mobile/src/screens/forums/forum-user-groups-screen.tsx`

**Cross-plan note:** New mobile screens must be registered in `forums-navigator.tsx` and `types/index.ts`. Ensure screen name is added to the forums navigator stack.

**Actions:**
1. List groups with member counts and color badges
2. Tap group to view members and basic settings
3. Long-press to reorder (drag handle)
4. Admin can tap "Edit" to modify group name/color/basic permissions
5. Simplified permission view: key permissions as toggle switches (not full matrix)
6. Secondary group assignment via "Add Member to Group" action

### Task 11: Mobile board permissions screen

**Files:** `apps/mobile/src/screens/forums/board-permissions-screen.tsx`

**Actions:**
1. Board picker at top (horizontal scroll)
2. For selected board: list of groups with expandable permission sections
3. Each permission shows inherit/allow/deny segmented control
4. "Apply Template" button opens template list bottom sheet
5. Simplified compared to web — focuses on most common permissions

### Task 12: Shared types

**Files:** `packages/shared-types/src/forum-permissions.ts`, `packages/shared-types/src/forum-user-groups.ts`

**Actions:**
1. `forum-permissions.ts`: Export `BoardPermission`, `ForumPermission`, `PermissionTemplate`, `PermissionLevel` (inherit/allow/deny), `EffectivePermission`
2. `forum-user-groups.ts`: Export `ForumUserGroup`, `SecondaryGroupMembership`, `GroupAutoRule`, `AutoRuleType`

## Verification

1. User group manager lists groups with correct member counts and supports reorder
2. Group permissions matrix renders 30+ permissions for all groups with save
3. Secondary group assignment works with expiry and OR-logic stacking
4. Auto-rule evaluation assigns users to groups based on configured thresholds
5. Board permissions panel shows per-board overrides with inherit/allow/deny
6. Applying a permission template to a board sets all permissions correctly
7. Permission matrix shows effective permissions across all boards/groups
8. Mobile user groups screen displays groups and allows basic management
9. Mobile board permissions show permissions with usable controls
