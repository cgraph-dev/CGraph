# Plan 16-03: Progressive Disclosure System — Level-Gated Features

## Context

GAME-09 requires progressive disclosure: "XP first → marketplace later." This is the **only GAME-xx requirement with zero existing implementation.** The concept is: new users see only basic gamification (XP, streaks, achievements), and more complex features (shop, marketplace, battle pass, prestige) reveal themselves as the user levels up.

This follows Discord's approach where server features unlock with Nitro/boosts, and Google Play Games' level-gated achievements. Meta's gaming platform also uses progressive feature revelation.

**Key design principle:** Never hide features permanently. Show locked features with a "Reach Level X to unlock" message so users know what's coming. This creates aspiration without confusion.

<objective>
Build complete progressive disclosure system: backend level checking, API protection, frontend gate components, sidebar integration, and unlock celebrations.

Purpose: Progressive disclosure prevents overwhelming new users while creating aspiration for advanced features.
Output: FeatureGates module, LevelGatePlug, LevelGate React component, useLevelGate hook, unlock toasts.
</objective>

  ```elixir
  defmodule CGraph.Gamification.FeatureGates do
    @feature_requirements %{
      xp_system: 1,         # Always available
      streaks: 1,           # Always available  
      achievements: 1,      # Always available
      quests: 3,            # Early unlock
      daily_rewards: 3,     # Early unlock
      leaderboard: 5,       # Shows competitive element
      shop: 8,              # First spending opportunity
      cosmetics: 10,        # Avatar borders, themes
      titles: 12,           # Equippable titles
      animated_borders: 15, # Premium cosmetic
      marketplace: 15,      # User-to-user trading
      battle_pass: 18,      # Seasonal content
      events: 20,           # Seasonal events
      trading: 20,          # P2P marketplace
      prestige: 25          # Endgame content
    }
    
    def feature_requirements, do: @feature_requirements
    def required_level(feature), do: Map.get(@feature_requirements, feature)
    
    def unlocked?(user_level, feature) do
      case required_level(feature) do
        nil -> true  # Unknown features default to unlocked
        req -> user_level >= req
      end
    end
    
    def get_user_gates(user_level) do
      Map.new(@feature_requirements, fn {feature, req_level} ->
        {feature, %{
          unlocked: user_level >= req_level,
          required_level: req_level,
          current_level: user_level
        }}
      end)
    end
    
    def newly_unlocked_features(old_level, new_level) do
      Enum.filter(@feature_requirements, fn {_feature, req} ->
        old_level < req and new_level >= req
      end)
      |> Enum.map(fn {feature, _} -> feature end)
    end
  end
  ```

  <files>apps/backend/lib/cgraph_web/plugs/level_gate_plug.ex</files>
  <action>
  Create `CGraphWeb.Plugs.LevelGatePlug`:
  ```elixir
  defmodule CGraphWeb.Plugs.LevelGatePlug do
    import Plug.Conn
    alias CGraph.Gamification.FeatureGates
    
    def init(opts), do: opts
    
    def call(conn, feature: feature) do
      user = conn.assigns[:current_user]
      
      if is_nil(user) or FeatureGates.unlocked?(user.level, feature) do
        conn
      else
        required = FeatureGates.required_level(feature)
        conn
        |> put_status(403)
        |> Phoenix.Controller.json(%{
          error: "level_required",
          message: "Reach level #{required} to unlock this feature",
          required_level: required,
          current_level: user.level
        })
        |> halt()
      end
    end
  end
  ```
