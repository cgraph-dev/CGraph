**`FileTransfer` context:**

- `initiate_upload(user, attrs)` — validates file type (reject .exe, .bat, .cmd, .scr, .vbs, .com,
  .pif, .msi, .ps1), checks tier storage quota, creates file_transfer record. For files
  > 5MB, returns multipart upload config (upload_id, chunk_size, presigned URLs for each chunk). For
  > smaller files, returns single presigned PUT URL.
- `upload_chunk(transfer_id, chunk_number, etag)` — records chunk completion. Returns progress.
- `complete_upload(transfer_id, parts)` — completes S3 multipart upload, sets status to
  "processing". Triggers thumbnail generation for images/videos.
- `generate_download_url(transfer_id, user_id, opts)` — validates user has access to the
  conversation, generates a time-limited signed S3 URL (default: 1 hour). Increments download_count.
- `get_transfer(transfer_id)` — returns transfer record with status.
- `get_user_storage_usage(user_id)` — sums file_size for user's transfers. Used for quota check.
- `cancel_upload(transfer_id, user_id)` — aborts multipart upload, deletes S3 parts, marks failed.
- `generate_thumbnail(transfer)` — for images: resize to 200px max dimension via S3 Lambda or
  server-side (for MVP, use imagemagick via System.cmd if available, or skip and let client
  generate). For videos: extract first frame placeholder.

**`ChunkedUpload` module:**

- Manages multipart upload state via ExAws.S3 multipart API.
- `start_multipart(storage_key, content_type)` — initiates S3 multipart upload.
- `presign_chunk(upload_id, storage_key, chunk_number)` — generates presigned URL for each chunk.
- `complete_multipart(upload_id, storage_key, parts)` — finalizes multipart upload.
- `abort_multipart(upload_id, storage_key)` — cancels upload, cleans up parts.

**`DownloadToken` module:**

- `generate(transfer, user_id, ttl_seconds \\ 3600)` — creates a short-lived signed token using
  `Phoenix.Token.sign/3`. Token contains transfer_id + user_id + expiry.
- `verify(token)` — verifies token, returns `{:ok, %{transfer_id, user_id}}` or
  `{:error, :expired}`.
- Download URLs use these tokens instead of raw S3 URLs for access control.

Dangerous file types list: `.exe`, `.bat`, `.cmd`, `.scr`, `.vbs`, `.com`, `.pif`, `.msi`, `.ps1`,
`.wsf`, `.js` (as attachment), `.jar`. Allowed max size per tier: Free=25MB, Plus=50MB, Pro=100MB,
Ultimate=200MB. </action> <verify> `mix ecto.migrate` succeeds. `mix compile` succeeds. Context has
all listed functions. </verify> <done> File transfer schema, chunked upload support via S3
multipart, secure download tokens, file type validation, tier-based storage quotas, and E2EE-aware
encrypted metadata. </done> </task>

  **FileTransferController** (~180 lines):
  - `initiate` — POST /api/v1/files/upload — initiates upload. Body: `{file_name, file_size,
    file_mime_type, conversation_id, is_encrypted}`. Returns presigned URL(s) or multipart config.
  - `upload_chunk` — PUT /api/v1/files/:id/chunks/:chunk_number — records chunk completion.
  - `complete` — POST /api/v1/files/:id/complete — finalizes upload. Body: `{parts: [{number, etag}]}`.
  - `download` — GET /api/v1/files/:id/download — generates download URL. Validates conversation access.
  - `status` — GET /api/v1/files/:id — returns upload status and progress.
  - `cancel` — DELETE /api/v1/files/:id — cancels upload.
  - `storage_usage` — GET /api/v1/files/usage — returns user's storage usage vs tier limit.

**FileTransferJSON** — renders transfer status, progress, download URLs.

**Router:** Add under messaging_routes:

```elixir
scope "/files" do
  post "/upload", FileTransferController, :initiate
  get "/usage", FileTransferController, :storage_usage
  get "/:id", FileTransferController, :status
  put "/:id/chunks/:chunk_number", FileTransferController, :upload_chunk
  post "/:id/complete", FileTransferController, :complete
  get "/:id/download", FileTransferController, :download
  delete "/:id", FileTransferController, :cancel
end
```

- Logs cleanup counts.
