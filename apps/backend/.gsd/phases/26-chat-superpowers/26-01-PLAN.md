  <action>
  Create a migration for secret chat tables:

**`secret_conversations` table:**

- `id` binary_id primary key
- `initiator_id` references users (NOT NULL)
- `recipient_id` references users (NOT NULL)
- `status` string (active, terminated, expired) default "active"
- `self_destruct_seconds` integer (NULL = no self-destruct)
- `initiator_device_id` string (device binding)
- `recipient_device_id` string (device binding)
- `initiator_fingerprint` string (key fingerprint for verification)
- `recipient_fingerprint` string
- `terminated_at` utc_datetime_usec
- `terminated_by` references users
- timestamps

Unique index on `[initiator_id, recipient_id]` where status = 'active' (only one active secret chat
per pair).

**`secret_messages` table:**

- `id` binary_id primary key
- `secret_conversation_id` references secret_conversations (NOT NULL)
- `sender_id` references users (NOT NULL)
- `ciphertext` binary (NOT NULL) — the actual encrypted message blob
- `content_type` string default "text" (text, image, video, audio, file, voice, sticker)
- `nonce` binary — encryption nonce for AES-GCM
- `ratchet_header` binary — ratchet public key + message number + previous chain length
- `expires_at` utc_datetime_usec — computed from self_destruct_seconds at send time
- `read_at` utc_datetime_usec — when recipient read the message
- `file_metadata` map — encrypted file metadata (size, mime, thumbnail hash)
- timestamps(updated_at: false)

Index on `[secret_conversation_id, inserted_at]`. Index on `[expires_at]` for cleanup worker.

**Ecto schemas:** Create `SecretConversation` and `SecretMessage` schemas matching the tables.

**Context `SecretChat`:**

- `create_secret_conversation(initiator, recipient_id, opts)` — creates secret chat, validates no
  existing active one, returns `{:ok, secret_conversation}`. Opts include `device_id`.
- `list_secret_conversations(user)` — lists active secret conversations for user on this device.
- `get_secret_conversation(id, user_id)` — gets conversation if user is a participant.
- `send_secret_message(conversation, sender, attrs)` — stores ciphertext blob. attrs must include
  `ciphertext`, `nonce`, `ratchet_header`. Computes `expires_at` from conversation's
  `self_destruct_seconds`. Returns `{:ok, message}`.
- `list_secret_messages(conversation, opts)` — cursor-paginated, only returns messages not yet
  expired.
- `mark_secret_message_read(message_id, user_id)` — sets `read_at`, starts self-destruct countdown
  if timer is set.
- `set_self_destruct_timer(conversation, user_id, seconds)` — updates timer. Both participants are
  notified via PubSub.
- `destroy_secret_chat(conversation, user_id)` — sets status to "terminated", deletes ALL messages
  from DB (hard delete, not soft delete — this is a privacy feature). Sets `terminated_at` and
  `terminated_by`.
- `cleanup_expired_messages()` — deletes messages where `expires_at < now()`. Called by Oban worker.

Follow existing patterns from `CGraph.Messaging.Conversations` for query structure. Use
`CGraph.Repo` for writes. Business logic in context, never in controller. </action> <verify>
`mix ecto.migrate` succeeds. `mix compile` succeeds with no warnings on new files. </verify> <done>
Secret conversation and message tables exist in DB. Context module provides full CRUD with device
binding, self-destruct timer management, and hard-delete on termination. </done> </task>

  <action>
  **SecretChatController** (~150 lines):
  - `create` — POST /api/v1/secret-chats — creates secret chat. Body: `{recipient_id, device_id}`.
    Returns 201 with secret conversation data + key fingerprints.
  - `index` — GET /api/v1/secret-chats — lists user's active secret conversations.
  - `show` — GET /api/v1/secret-chats/:id — gets a specific secret conversation with last message preview.
  - `destroy` — DELETE /api/v1/secret-chats/:id — terminates and hard-deletes all messages.
  - `set_timer` — PUT /api/v1/secret-chats/:id/timer — sets self-destruct timer.
    Body: `{seconds: 5|30|60|300|3600|86400|604800|null}`.
    Broadcasts timer change to both participants via PubSub.

All endpoints validate user is participant. Controller delegates to `SecretChat` context. Use safe
error messages (no `inspect(reason)` leaks).

**SecretChatJSON** — renders secret conversation and message data. NEVER include `ciphertext` in
REST responses — that's for the channel only.

**SecretChatChannel:**

- Join `"secret_chat:" <> conversation_id` — validates user is participant AND device matches.
- `handle_in("new_message", payload, socket)` — payload contains
  `{ciphertext, nonce, ratchet_header, content_type, file_metadata}`. Stores via SecretChat context,
  broadcasts to other participant. Rate limited (same pattern as ConversationChannel).
- `handle_in("typing", payload, socket)` — typing indicators.
- `handle_in("message_read", %{"message_id" => id}, socket)` — marks read, triggers self-destruct
  countdown.
- `handle_in("screenshot_detected", _payload, socket)` — broadcasts screenshot alert to other
  participant (privacy signal).
- Catch-all handler for unrecognized events.

**UserSocket:** Add `channel "secret_chat:*", CGraphWeb.SecretChatChannel`.

**Router:** Add routes under messaging_routes:

```elixir
resources "/secret-chats", SecretChatController, only: [:index, :show, :create, :delete]
put "/secret-chats/:id/timer", SecretChatController, :set_timer
```
