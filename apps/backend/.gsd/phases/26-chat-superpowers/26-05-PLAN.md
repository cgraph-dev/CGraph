  <action>
  **Harden `Calls` module** — ensure full call state machine:
  States: `idle → ringing → connecting → connected → ended`
  Transitions:
  - `initiate_call(caller, callee_id, opts)` — creates call record with state "ringing".
    Opts: `{type: "voice"|"video", from_secret_chat: bool, conversation_id}`.
    Validates callee exists, not blocked, not in another call.
    Sends push notification to callee (via Notifications context).
    Sets 30-second ring timeout (Process.send_after → auto-reject as "missed").
  - `accept_call(call_id, callee_id)` — transitions to "connecting". Clears ring timeout.
  - `reject_call(call_id, user_id, reason)` — transitions to "ended". Reason: "rejected", "busy",
    "timeout". Notifies caller via channel.
  - `end_call(call_id, user_id)` — transitions to "ended". Records duration. Notifies both parties.
  - `toggle_video(call_id, user_id, enabled)` — toggles video stream. Notifies peer.
  - `toggle_mute(call_id, user_id, muted)` — toggles audio mute. Notifies peer.

**Call timeout handling:**

- Ring timeout: 30 seconds. If callee doesn't accept, mark as "missed".
- Connection timeout: 10 seconds after accept. If ICE fails, mark as "failed".
- Max call duration: 4 hours (safety limit). Auto-end with warning at 3h50m.

**`CallQuality` module** (NEW):

- `report_metrics(call_id, user_id, metrics)` — stores quality snapshot. Metrics:
  `{jitter_ms, packet_loss_pct, bitrate_kbps, round_trip_ms, codec, resolution}`.
- `get_call_quality(call_id)` — returns quality summary for a call.
- `get_average_quality(user_id, days \\ 30)` — average quality over time.
- Store in ETS for real-time metrics (flushed to DB on call end). Use lightweight approach — no
  separate table, embed quality summary in call_history record as JSON map.
