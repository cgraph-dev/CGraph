  Create migration for sticker tables:

**`sticker_packs` table:**

- `id` binary_id primary key
- `name` string NOT NULL (unique)
- `title` string NOT NULL (display title, e.g., "Fluffy Cats")
- `description` text
- `author` string
- `thumbnail_url` string NOT NULL
- `category` string NOT NULL (animals, emotions, memes, gaming, holidays, custom, premium)
- `sticker_type` string default "static" (static, animated, video)
- `is_premium` boolean default false
- `is_official` boolean default false
- `is_animated` boolean default false
- `coin_price` integer default 0 (0 = free, >0 = costs coins)
- `download_count` integer default 0
- `sticker_count` integer default 0
- `sort_order` integer default 0
- `published` boolean default true
- `creator_id` references users (NULL for official packs)
- `group_id` references groups (NULL for public packs, set for group-exclusive)
- timestamps

Index on `[category]`, `[is_premium]`, `[published]`, `[creator_id]`, `[group_id]`.

**`stickers` table:**

- `id` binary_id primary key
- `sticker_pack_id` references sticker_packs (NOT NULL, on_delete cascade)
- `emoji_shortcode` string (e.g., ":wave:", for quick search/mapping)
- `file_url` string NOT NULL (S3/R2 URL to the sticker image)
- `thumbnail_url` string
- `file_type` string default "webp" (webp, apng, lottie, gif)
- `file_size` integer
- `width` integer
- `height` integer
- `sort_order` integer default 0
- timestamps(updated_at: false)

Index on `[sticker_pack_id, sort_order]`.

**`user_sticker_packs` table:**

- `id` binary_id primary key
- `user_id` references users (NOT NULL)
- `sticker_pack_id` references sticker_packs (NOT NULL)
- `sort_order` integer default 0
- timestamps(updated_at: false)

Unique index on `[user_id, sticker_pack_id]`.

**Ecto schemas:** Standard CGraph patterns for all three schemas.

**Context `CGraph.Stickers`:**

- `list_store_packs(opts)` — paginated, filterable by category, type, is_premium. Includes
  sticker_count and preview stickers (first 5).
- `get_pack(pack_id)` — returns pack with all stickers.
- `search_packs(query)` — search by name/title.
- `add_pack(user, pack_id)` — adds pack to user's collection. If pack has coin_price > 0, deducts
  coins via `CGraph.Gamification.deduct_coins/2`. Returns {:error, :insufficient_coins} if not
  enough.
- `remove_pack(user, pack_id)` — removes from collection.
- `list_user_packs(user_id)` — returns user's installed packs with stickers.
- `get_sticker(sticker_id)` — returns single sticker with pack info.
- `list_recently_used(user_id, limit \\ 20)` — recently sent stickers (query messages table for
  content_type "sticker" by this user, extract sticker IDs).
- `increment_download_count(pack_id)` — atomic increment.
- `create_pack(attrs)` — admin/creator function for creating custom packs.
- `add_sticker_to_pack(pack, attrs)` — adds a sticker to a pack with file upload. </action> <verify>
  `mix ecto.migrate` succeeds. `mix compile` succeeds. Context module has all listed functions.
  </verify> <done> Sticker system database schema created with packs, individual stickers, and user
  collections. Context provides full CRUD including store browsing, coin-gated downloads, and

  <action>
  **StickerController** (~200 lines):
  - `store` — GET /api/v1/stickers/store — browse sticker store. Query params: `category`,
    `type` (static/animated), `premium` (true/false), `page`, `per_page`. Returns packs with
    preview stickers (first 5 per pack).
  - `search` — GET /api/v1/stickers/search?q=... — search packs by name/title/emoji shortcode.
  - `show_pack` — GET /api/v1/stickers/packs/:id — full pack with all stickers.
  - `my_packs` — GET /api/v1/stickers/my-packs — user's installed sticker packs with all stickers.
  - `add_pack` — POST /api/v1/stickers/packs/:id/add — add pack to user's collection.
    For coin-priced packs, deducts coins and returns error if insufficient.
  - `remove_pack` — DELETE /api/v1/stickers/packs/:id/remove — remove from collection.
  - `recently_used` — GET /api/v1/stickers/recent — recently sent stickers.
  - `trending` — GET /api/v1/stickers/trending — top packs by download_count.

**StickerJSON** — renders pack data with stickers, thumbnails, download counts.

**Router:** Add sticker routes under messaging_routes:

```elixir
scope "/stickers" do
  get "/store", StickerController, :store
  get "/search", StickerController, :search
  get "/my-packs", StickerController, :my_packs
  get "/recent", StickerController, :recently_used
  get "/trending", StickerController, :trending
  get "/packs/:id", StickerController, :show_pack
  post "/packs/:id/add", StickerController, :add_pack
  delete "/packs/:id/remove", StickerController, :remove_pack
end
```
