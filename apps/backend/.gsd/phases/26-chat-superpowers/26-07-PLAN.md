  <action>
  **Audit every handle_in handler in ConversationChannel.** For each feature, verify the full
  round-trip works. Fix any gaps found:

**1. Message Editing (`edit_message`):**

- Verify: broadcasts `message_edited` with `%{id, content, edited_at}` to all participants.
- Ensure only sender can edit. Edit window: 15 minutes after send (configurable).
- Edit history preserved in `message_edits` table (already exists via MessageEdit).
- FIX if missing: broadcast the `edited_at` timestamp so clients show "edited" indicator.

**2. Message Deletion (`delete_message`):**

- Verify delete_for_everyone broadcasts `message_deleted` with `%{id}` to all participants.
- Verify delete_for_me only soft-deletes for requesting user (not broadcast).
- Ensure deleted messages in DB set `content: nil, deleted_at: DateTime.utc_now()`.
- FIX if missing: distinguish between soft-delete (for_me) vs broadcast-delete (for_everyone) in the
  channel handler. Accept `%{"message_id" => id, "mode" => "for_me" | "for_everyone"}`.

**3. Reply/Quote (`new_message` with reply_to):**

- Verify: when `reply_to_id` is provided, the broadcast includes `reply_to` with a preview of the
  original message: `%{id, content_preview (first 100 chars), sender_username}`.
- FIX if missing: preload `reply_to` association and include preview in broadcast payload.

**4. Forward Message:**

- Add `handle_in("forward_message", %{"message_id" => id, "to_conversation_id" => target}, socket)`.
- Delegates to `MessageOperations.forward_message/3`.
- Creates a new message in target conversation with `forwarded_from: original_message_id`.
- Broadcasts to target conversation channel.
- Validates: user must be member of target conversation.

**5. Typing Indicators:**

- Verify: `handle_in("typing", ...)` broadcasts `typing` to others with auto-expiry.
- FIX if missing: include `user_id` and `username` in typing broadcast.
- Add `handle_in("stop_typing", ...)` for immediate stop (vs waiting for timeout).

**6. Link Preview Generation:**

- After `new_message` is sent, detect URLs in content using regex `~r/https?:\/\/[^\s<]+/`.
- For each URL found, enqueue `Workers.FetchLinkPreview` Oban job:
  `%{message_id: msg.id, url: url}`.
- When preview is fetched, update message's `link_preview` field and broadcast
  `link_preview_updated` event: `%{message_id, link_preview: %{url, title, description, image}}`.
- LinkPreviewService and FetchLinkPreview worker already exist — ensure they're wired in.

**7. Delivery Receipts:**

- On `after_join`, mark all messages in conversation as "delivered" for joining user.
- On `msg_ack`, track per-device delivery (DeliveryTracking already exists).
- Broadcast `delivery_receipt` to sender: `%{message_id, delivered_to: user_id, delivered_at}`.

**8. Read Receipts:**

- Verify `mark_read` broadcasts `read_receipt` to sender:
  `%{message_ids: [...], read_by: user_id, read_at}`.
- Add read receipt opt-out: check user privacy setting before broadcasting. If user has
  `read_receipts_enabled: false`, don't broadcast (still track internally). </action> <verify>

  ConversationChannel: edit with history, delete (for_me/for_everyone), reply with preview, forward
