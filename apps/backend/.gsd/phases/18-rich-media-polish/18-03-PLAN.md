# Plan 18-03 — UI Polish & Component Library

## Goal

Elevate CGraph's visual quality and developer experience: expand Storybook from 15 to 30+ stories,
add skeleton loading states for every page, ensure all screens have empty/error states, standardize
animations across both platforms using the shared animation-constants package, and document the
component library.

## Why This Ordering

Tasks proceed: (0) animation token standardization, (1) page transitions + error boundaries, (2)
skeleton loading for all pages, (3) empty state audit + new states, (4) Storybook expansion, (5)
mobile UI polish, (6) component catalog documentation.

## Tasks

### Task 0 — Animation Token Standardization

**What:** Extend `packages/animation-constants/` with pre-composed transition presets that both web
and mobile consume. Currently has `durations.ts` (44L), `easings.ts` (59L), `springs.ts` (48L),
`stagger.ts` (25L). Add a `transitions.ts` module with ready-to-use framer-motion / Reanimated
transition objects. Then audit top-20 most-used animation files, replacing hardcoded values with
imports from the package.

**Where:**

- `packages/animation-constants/src/transitions.ts` (new)
- `packages/animation-constants/src/index.ts` (re-export)
- Top-20 animation files in `apps/web/src/` and `apps/mobile/src/`

**How:**

Create `transitions.ts`:

```typescript
import { durations } from './durations';
import { easings } from './easings';
import { springs } from './springs';

// Pre-composed transition presets for framer-motion
export const transitions = {
  pageEnter: {
    initial: { opacity: 0, y: 8 },
    animate: { opacity: 1, y: 0 },
    exit: { opacity: 0, y: -8 },
    transition: { duration: durations.normal / 1000, ease: easings.easeOut },
  },
  modalSlideUp: {
    initial: { opacity: 0, y: '100%' },
    animate: { opacity: 1, y: 0 },
    exit: { opacity: 0, y: '100%' },
    transition: { type: 'spring', ...springs.gentle },
  },
  drawerSlideIn: {
    initial: { x: '100%' },
    animate: { x: 0 },
    exit: { x: '100%' },
    transition: { type: 'spring', ...springs.snappy },
  },
  fadeIn: {
    initial: { opacity: 0 },
    animate: { opacity: 1 },
    transition: { duration: durations.fast / 1000 },
  },
  listItemStagger: {
    initial: { opacity: 0, y: 12 },
    animate: { opacity: 1, y: 0 },
  },
  springBounce: {
    transition: { type: 'spring', ...springs.bouncy },
  },
  scalePress: {
    whileTap: { scale: 0.97 },
    transition: { type: 'spring', ...springs.snappy },
  },
} as const;

// Reanimated-compatible presets (numeric values for RN)
export const rnTransitions = {
  pageEnter: { duration: durations.normal, easing: 'easeOut' },
  modalSlideUp: { damping: springs.gentle.damping, stiffness: springs.gentle.stiffness },
  fadeIn: { duration: durations.fast },
} as const;
```

Re-export from `index.ts`.

**Audit phase** — grep for hardcoded `duration:`, `delay:`, `damping:`, `stiffness:` in the top-20
most-edited animation files. Replace with imports. Document exceptions where custom values are
intentional (e.g., waveform-specific timing).

**Verify:** Package builds. No regression in existing animations. Spot-check 5 pages for smooth
transitions.

### Task 1 — Page Transitions + Error Boundaries

**What:** Add consistent page-level transitions using framer-motion `AnimatePresence` and wrap all
route components with error boundaries using a standardized fallback.

**Where:**

- `apps/web/src/shared/components/page-transition.tsx` (new)
- `apps/web/src/shared/components/error-fallback.tsx` (new or extend)
- `apps/web/src/App.tsx` or router file (wrap routes)

**How:**

Create `page-transition.tsx`:

```tsx
import { motion, AnimatePresence } from 'framer-motion';
import { transitions } from '@cgraph/animation-constants';
import { useLocation } from 'react-router-dom';

export function PageTransition({ children }: { children: React.ReactNode }) {
  const location = useLocation();
  return (
    <AnimatePresence mode="wait">
      <motion.div key={location.pathname} {...transitions.pageEnter} className="flex-1">
        {children}
      </motion.div>
    </AnimatePresence>
  );
}
```

Create `error-fallback.tsx`:

```tsx
export function ErrorFallback({
  error,
  resetErrorBoundary,
}: {
  error: Error;
  resetErrorBoundary: () => void;
}) {
  return (
    <div className="flex min-h-[400px] flex-col items-center justify-center p-8 text-center">
      <ShieldAlertIcon className="text-destructive mb-4 h-12 w-12" />
      <h2 className="mb-2 text-xl font-semibold">Something went wrong</h2>
      <p className="text-muted-foreground mb-6 max-w-md">
        {error.message || 'An unexpected error occurred.'}
      </p>
      <div className="flex gap-3">
        <Button onClick={resetErrorBoundary}>Try Again</Button>
        <Button variant="outline" onClick={() => window.history.back()}>
          Go Back
        </Button>
      </div>
    </div>
  );
}
```

Wrap route-level components with `<ErrorBoundary FallbackComponent={ErrorFallback}>` and wrap the
router outlet with `<PageTransition>`.

**Mobile** — add `react-native-reanimated` entering/exiting transitions to screen navigations:

```tsx
// In navigation config
screenOptions={{
  animation: 'fade',
  animationDuration: durations.normal,
}}
```

**Verify:** Navigate between pages → smooth fade transition. Break a component intentionally → error
fallback shows with "Try Again" button → click retries.

**How:**

Create `page-skeleton.tsx` — a generic wrapper:

```tsx
export function PageSkeleton({
  isLoading,
  skeleton,
  children,
}: {
  isLoading: boolean;
  skeleton: React.ReactNode;
  children: React.ReactNode;
}) {
  if (isLoading) return <>{skeleton}</>;
  return <>{children}</>;
}
```

Create page-specific skeletons. Each matches the layout shape of the real page:

**`channel-list-skeleton.tsx`:**

- Repeating rows: circle (avatar) + two rectangles (name + last message) × 10 items

**`forum-skeleton.tsx`:**

- Board header rectangle

**`settings-skeleton.tsx`:**

- Sidebar: list of rectangles × 8
- Content: heading rectangle + form field rectangles × 5

Wire into pages: each page component should render its skeleton during loading:

```tsx

import Animated, { FadeIn, FadeOut } from 'react-native-reanimated';

export function EmptyState({ icon, title, description, actionLabel, onAction }: EmptyStateProps) {

    <Animated.View entering={FadeIn.duration(300)} style={styles.container}>
```
