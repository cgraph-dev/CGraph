# Plan 14-04: Mobile Forum Wiring — BBCode Renderer, Search, Store, Delete Flows

## Context

The mobile forum has 10 screens (~3,637 LOC), 5 shared components (~1,290 LOC), and real-time hooks (`useForumChannel`, `useThreadChannel`). However, several gaps exist:

1. **No BBCode renderer** — The backend will now parse BBCode→HTML (Plan 14-01), but mobile can't use HTML WebViews for performance. Need a native RN BBCode renderer that produces `<Text>` components.

2. **No centralized Zustand store** — All state is local to screens via hooks and direct API calls. This means no cross-screen state sharing, no optimistic updates, and no cache.

3. **Search not wired to full-text** — `searchService.ts` has `searchForums()` but it hits the old ILIKE endpoint. Need to wire to the new full-text search from Plan 14-02.

4. **No delete flows** — No delete confirmation modals for forums, threads, posts, or comments at the screen level.

5. **Module index incomplete** — `CreatePostScreen` and `PluginMarketplaceScreen` not exported.

6. **Create post doesn't support inline polls** — `poll-widget.tsx` (452L) exists for display but isn't integrated into the creation flow.

**What exists (mobile):**
- `post-screen.tsx` (570L) — Thread/post view with replies, no delete UI
- `forum-screen.tsx` (263L) — Forum threads listing
- `create-post-screen/` (185+L) — Post creation with preview + type selector
- `poll-widget.tsx` (452L) — Full poll UI for display/voting
- `vote-buttons.tsx` (189L) — Upvote/downvote component
- `useForumChannel` / `useThreadChannel` — Real-time hooks (wired)
- `searchService.ts` — Has `searchForums()` but uses old endpoint

## Tasks

### Task 1: Create BBCode renderer component

**Files:** `apps/mobile/src/components/forums/bbcode-renderer.tsx`

**Actions:**
1. Create `BBCodeRenderer` component that takes `bbcode: string` prop
2. Parse BBCode tags and render as React Native components:
   - `[b]` → `<Text style={{fontWeight: 'bold'}}>` 
   - `[i]` → `<Text style={{fontStyle: 'italic'}}>`
   - `[u]` → `<Text style={{textDecorationLine: 'underline'}}>`
   - `[s]` → `<Text style={{textDecorationLine: 'line-through'}}>`
   - `[url=href]` → `<Text onPress={()=>Linking.openURL(href)} style={linkStyle}>`
   - `[img]src[/img]` → `<Image source={{uri: src}} />`
   - `[quote]` → Styled `<View>` with left border and gray background
   - `[quote=author]` → Quote view with author citation
   - `[code]` → Monospace `<Text>` in code block `<View>`
   - `[list][*]` → Bulleted list items
   - `[color=X]` → `<Text style={{color: X}}>` (validate color)
   - `[size=N]` → `<Text style={{fontSize: sizeMap[N]}}>`
   - `[spoiler]` → Collapsed `<Pressable>` that reveals content on tap
   - `[center]` → `<View style={{alignItems: 'center'}}>`
3. Handle nesting with recursive parsing
4. Handle plain text and `\n` as line breaks
5. XSS-safe: only render validated URLs (http/https), strip JavaScript URIs
6. Alternative approach: if backend sends `content_html`, use the `content` (raw BBCode) for native rendering or `content_html` for display via a lightweight HTML renderer

### Task 2: Create Zustand forum store

**Files:** `apps/mobile/src/stores/forumStore.ts`, `apps/mobile/src/services/forumService.ts`

**Actions:**
1. Create `forumService.ts` API client:
   - `listForums(params)` → GET /api/v1/forums
   - `getForum(id)` → GET /api/v1/forums/:id
   - `createForum(data)` → POST /api/v1/forums
   - `updateForum(id, data)` → PUT /api/v1/forums/:id
   - `deleteForum(id)` → DELETE /api/v1/forums/:id
   - `listBoards(forumId)` → GET /api/v1/forums/:fid/boards
   - `listThreads(boardId, params)` → GET /api/v1/forums/:fid/boards/:bid/threads
   - `getThread(threadId)` → GET /api/v1/forums/:fid/boards/:bid/threads/:tid
   - `createThread(boardId, data)` → POST /api/v1/forums/:fid/boards/:bid/threads
   - `createComment(postId, data)` → POST /api/v1/forums/:fid/posts/:pid/comments
   - `editComment(postId, commentId, content)` → PUT /api/v1/.../comments/:cid
   - `deleteComment(postId, commentId)` → DELETE /api/v1/.../comments/:cid
   - `votePost(postId, value)` → POST /api/v1/forums/:fid/posts/:pid/vote
   - `searchForums(query, filters)` → GET /api/v1/search/forums
2. Create `forumStore.ts` Zustand store:
   - State: `forums[]`, `currentForum`, `boards[]`, `threads[]`, `currentThread`, `comments[]`, `searchResults[]`, `searchQuery`, `loading`, `error`
   - Actions: `fetchForums`, `fetchForum`, `createForum`, `updateForum`, `deleteForum`, `fetchBoards`, `fetchThreads`, `fetchThread`, `createThread`, `createComment`, `editComment`, `deleteComment`, `vote`, `searchForums`, `clearSearch`
   - Optimistic updates for voting and comment creation
