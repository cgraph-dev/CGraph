# Plan 14-01: BBCode Parser + Poll API + Attachment Uploads

## Context

The forum backend has extensive schema and context infrastructure (~7,500 LOC across 64+ files) but three critical pieces are stubs or missing:

1. **BBCode parser** — `thread.ex` and `thread_post.ex` both have `maybe_render_html/1` that only does `html_escape + newline→<br>`. No actual BBCode tag parsing exists. The web frontend already has a full BBCode editor with toolbar (colors, sizes, smilies, formatting).

2. **Poll API routes** — `ThreadPoll` schema (81 lines), `PollVote` schema (33 lines), and `polls.ex` context (110 lines) all exist with `create_thread_poll/2`, `vote_poll/3`, `has_voted?/2`, `get_poll_results/1`. But **no controller or routes** exist — polls are completely unreachable via HTTP.

3. **Attachment uploads** — `ThreadAttachment` schema (110 lines) has full validation (file type, 10MB limit, dimensions) but **no controller, no routes, no upload handler**. The web frontend has `attachment-uploader/` component and `uploadAttachment` store action.

**What exists:**
- `forums/thread.ex` (444L) — `maybe_render_html/1` is a stub at L139-148
- `forums/thread_post.ex` (127L) — `maybe_render_html/1` stub at L108-116
- `forums/polls.ex` (110L) — Full context: create, vote, results, validation
- `forums/thread_poll.ex` (81L) — Schema with question, options, is_multiple_choice, closes_at
- `forums/poll_vote.ex` (33L) — Vote record with option_ids
- `forums/thread_attachment.ex` (110L) — Schema with file validation, dimensions, thumbnails
- `forum_routes.ex` (119L) — No poll or attachment routes

**What's needed:**
1. BBCode→HTML module with XSS protection
2. PollController + routes wired to existing context
3. ThreadAttachmentController + upload route
4. Thread creation to accept inline poll params

## Tasks

### Task 1: Create BBCode parser module

**Files:** `apps/backend/lib/cgraph/forums/bbcode.ex`, `apps/backend/test/cgraph/forums/bbcode_test.exs`

**Actions:**
1. Create `CGraph.Forums.BBCode` module with `to_html/1` function
2. Parse standard BBCode tags:
   - `[b]bold[/b]` → `<strong>bold</strong>`
   - `[i]italic[/i]` → `<em>italic</em>`
   - `[u]underline[/u]` → `<u>underline</u>`
   - `[s]strike[/s]` → `<del>strike</del>`
   - `[url=href]text[/url]` → `<a href="href" rel="nofollow noopener" target="_blank">text</a>`
   - `[url]href[/url]` → auto-linked
   - `[img]src[/img]` → `<img src="src" loading="lazy" />`
   - `[quote]text[/quote]` → `<blockquote>text</blockquote>`
   - `[quote=author]text[/quote]` → `<blockquote><cite>author</cite>text</blockquote>`
   - `[code]text[/code]` → `<pre><code>text</code></pre>`
   - `[list][*]item[/list]` → `<ul><li>item</li></ul>`
   - `[color=red]text[/color]` → `<span style="color:red">text</span>` (validate color value)
   - `[size=4]text[/size]` → `<span style="font-size:1.2em">text</span>` (clamp 1-7)
   - `[center]text[/center]` → `<div style="text-align:center">text</div>`
   - `[spoiler]text[/spoiler]` → `<details><summary>Spoiler</summary>text</details>`
3. Sanitize output:
   - HTML-escape input text BEFORE processing BBCode tags
   - Validate URL schemes (only http/https/mailto)
   - Strip `javascript:` and data URIs from url/img tags
   - No `<script>`, `onerror`, `onclick`, or other event handler injection
   - Validate color values against CSS color names and hex pattern
4. Handle edge cases: unclosed tags (leave as text), nested tags, empty tags
5. Convert newlines to `<br>` for non-code blocks
6. Write comprehensive tests for all tags, nesting, and XSS vectors

### Task 2: Wire BBCode parser into thread and thread_post schemas

**Files:** `apps/backend/lib/cgraph/forums/thread.ex`, `apps/backend/lib/cgraph/forums/thread_post.ex`

**Actions:**
1. In `thread.ex`, replace `maybe_render_html/1` stub (L139-148):
   ```elixir
   defp maybe_render_html(changeset) do
     case get_change(changeset, :content) do
       nil -> changeset
       content -> put_change(changeset, :content_html, CGraph.Forums.BBCode.to_html(content))
     end
   end
   ```
2. Same replacement in `thread_post.ex` L108-116
3. Ensure both create and update changesets trigger `maybe_render_html/1`

**Files:** `apps/backend/lib/cgraph_web/controllers/api/v1/poll_controller.ex`, `apps/backend/lib/cgraph_web/controllers/api/v1/poll_json.ex`, `apps/backend/lib/cgraph_web/router/forum_routes.ex`

**Actions:**

   - `close/2` — POST /threads/:thread_id/poll/close — close poll early (author or mod)

3. Add routes to `forum_routes.ex`:
   ```elixir
   # Under thread scope
   get "/poll", PollController, :show
   post "/poll", PollController, :create
   put "/poll", PollController, :update
   post "/poll/vote", PollController, :vote
   post "/poll/close", PollController, :close
   ```

## Verification

After completion:
1. `CGraph.Forums.BBCode.to_html("[b]hello[/b]")` returns `<strong>hello</strong>`
2. `BBCode.to_html("[url=javascript:alert(1)]xss[/url]")` sanitizes the URL
3. Creating a thread with `content: "[b]Bold[/b] text"` populates `content_html` with `<strong>Bold</strong> text`
4. `POST /api/v1/forums/:fid/boards/:bid/threads/:tid/poll` creates a poll
5. `POST /api/v1/forums/:fid/boards/:bid/threads/:tid/poll/vote` records a vote
6. Thread creation with `{..., poll: {question: "...", options: [...]}}` creates thread + poll atomically
7. `POST /api/v1/forums/:fid/boards/:bid/threads/:tid/attachments` uploads a file and returns attachment metadata
8. BBCode test suite passes with all tag types and XSS prevention cases
