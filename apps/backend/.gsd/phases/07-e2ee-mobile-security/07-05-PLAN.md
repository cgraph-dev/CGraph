<task type="auto">
  <name>Task 1: Wire encrypt-on-send and decrypt-on-receive into mobile chatStore</name>
  <files>
    apps/mobile/src/stores/chatStore.ts
  </files>
  <action>
    1. In the mobile `chatStore.ts`, find the `sendMessage` function (or equivalent — the function that sends a new message via API or Phoenix Channel).

    2. Add encrypt-before-send for 1:1 (direct) conversations:
       - Import the e2eeStore: `import { useE2EEStore } from '../lib/crypto/store/e2eeStore'`
       - Before sending, check: `e2eeStore.getState().isInitialized && conversation.type === 'direct'`
       - If true: call `e2eeStore.getState().encryptMessage(recipientId, content)`
       - Use the returned encrypted payload: set `content` to the ciphertext, add `is_encrypted: true`, include protocol metadata (ephemeral_public_key, nonce, protocol_version, etc.)
       - On encryption failure: throw error, do NOT fall back to plaintext (critical security behavior matching web)

    3. Find the message receive handler (the function called when a new message arrives via Phoenix Channel or API polling):
       - Check `message.isEncrypted` or `message.is_encrypted`
       - If encrypted AND not from current user:
         a. Call `e2eeStore.getState().decryptMessage(message.senderId, encryptedPayload)`
         b. Replace message content with decrypted plaintext
         c. Keep `isEncrypted: true` flag for UI indicator
         d. On decrypt failure: set `content = "⚠️ Unable to decrypt"`, `decryptionFailed = true`

    4. If the mobile `Message` interface doesn't have `isEncrypted` / `decryptionFailed` fields, add them (the research shows `isEncrypted: boolean` and `encryptedContent: string | null` already exist at lines 47-48).

    5. Handle offline message queue: encrypted messages received while app is backgrounded (and biometric-locked) should be stored encrypted in WatermelonDB. Decrypt when app foregrounds and e2eeStore initializes. Use the existing WatermelonDB bridge pattern from Phase 6.

    IMPORTANT: Match the web's security behavior — never show ciphertext, never fall back to plaintext on encrypt failure.

  </action>
  <verify>
    - `pnpm --filter mobile typecheck` passes
    - `grep -n "encryptMessage\|decryptMessage\|is_encrypted" apps/mobile/src/stores/chatStore.ts` shows encrypt/decrypt integration
    - `pnpm --filter mobile lint` passes
  </verify>
  <done>
    Mobile chatStore encrypts before send and decrypts on receive for 1:1 conversations via e2eeStore (pq-bridge). Messages stored encrypted in WatermelonDB when offline.
  </done>
