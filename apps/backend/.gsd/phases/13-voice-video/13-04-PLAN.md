# Plan 13-04: Call E2EE + Mobile LiveKit

## Context

All CGraph messaging uses E2EE. Voice/video calls must also be encrypted end-to-end. LiveKit has built-in E2EE support using SFrame (Secure Frames) via `ExternalE2EEKeyProvider`. Additionally, mobile needs LiveKit SDK integration for group calls.

**What exists:**
- Existing E2EE infrastructure (PQXDH + Triple Ratchet) for messaging
- LiveKit `livekit-client` SDK (from Plan 13-02) has `E2EEManager` with `ExternalE2EEKeyProvider`
- Backend has per-user E2EE key infrastructure
- Web calls show a lock icon for encrypted 1:1 messages (from Phase 7)

**What's needed:**
1. SFrame E2EE for LiveKit calls (web)
2. Mobile LiveKit SDK installation + integration
3. Mobile call E2EE
4. Key distribution: per-room shared secret via existing secure channels
5. Encryption indicator UI on both platforms

## Tasks

### Task 1: Backend per-room encryption key management

**Files:** `apps/backend/lib/cgraph/webrtc/call_encryption.ex`, `apps/backend/lib/cgraph_web/channels/call_channel.ex`

**Actions:**
1. Create `CallEncryption` module:
   - `get_or_create_room_key(room_id)`:
     - Generate 256-bit random key for the room (AES-256)
     - Store in ETS with room lifecycle (key dies when room dies)
     - Return base64-encoded key
   - `rotate_room_key(room_id)`:
     - Generate new key, broadcast rotation event to all participants
     - Used when participants change (member leaves → rotate to protect forward secrecy)
   - `encrypt_room_key_for_participant(room_key, participant_public_key)`:
     - ECDH + AES-GCM wrap the room key for each participant
     - Return encrypted key blob
2. Modify `call_channel.ex` join response to include:
   - `e2ee_key`: per-room key (encrypted to the joining participant's public key)
   - `e2ee_enabled`: true

### Task 2: Web SFrame E2EE integration

**Files:** `apps/web/src/lib/webrtc/callEncryption.ts`, `apps/web/src/lib/webrtc/livekitService.ts`

**Actions:**
1. Create `callEncryption.ts`:
   - `setupE2EE(room: Room, roomKey: Uint8Array)`:
     - Create `ExternalE2EEKeyProvider`
     - Set key material from room key
     - Enable E2EE on the Room: `room.setE2EEEnabled(true)`
   - `rotateKey(room: Room, newKey: Uint8Array)`:
     - Call `keyProvider.setKey(newKey, keyIndex)`
     - LiveKit handles ratcheting internally
   - `isEncrypted(room: Room)`: boolean check
   - Key derivation: `HKDF-SHA256(roomKey, salt="livekit-e2ee", info=roomName)` → 256-bit frame key
2. Modify `livekitService.ts`:
   - After `Room.connect()`, call `setupE2EE()` with the room key from backend
   - Listen for `RoomEvent.E2EEStateChanged` to update UI
3. Modify `useLiveKitRoom.ts`:
   - Add `isE2EEEnabled` state
   - Fetch room key from call channel join response
   - Pass to encryption setup

### Task 4: Mobile LiveKit SDK installation + service

**Files:** `apps/mobile/package.json`, `apps/mobile/src/lib/webrtc/livekitService.ts`

**Actions:**
1. Add dependencies:
   - `@livekit/react-native`

2. Create mobile `livekitService.ts`:
   - `connect(url, token, opts)` — connect to LiveKit room
   - `publishAudioTrack()` — create and publish local audio
   - `publishVideoTrack(cameraFacing)` — create and publish local video
   - `disconnect()` — leave room and cleanup
   - Track event handlers: `onTrackSubscribed`, `onTrackUnsubscribed`
   - Speaker detection: `onActiveSpeakersChanged`
3. Use `registerGlobals()` from `@livekit/react-native` for WebRTC polyfill

## Verification

After completion:
1. Web group calls have E2EE enabled (green lock visible)
2. SFrame encryption is verified by checking encrypted frames in WebRTC internals
3. Key rotation occurs when a participant leaves
4. Mobile can join group calls via LiveKit SFU (not just P2P)
5. Mobile shows encryption indicator during calls
6. If encryption key is wrong, media is garbled (not plaintext)
