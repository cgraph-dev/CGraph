## Context

Four mobile screens bypass the API layer entirely:

1. **Notifications Inbox** — `getMockNotifications()` creates fake notifications with a
   `setTimeout` delay to simulate loading. The notification store exists but is never used
   by the screen.

2. **User Wall** — `MOCK_POSTS` is a hardcoded array of fake post objects. Zero API calls.
   Backend has user profile/activity endpoints.

3. **Forum List** — Uses a try/catch where the catch block falls back to `getMockForums()`,
   meaning any API error shows fake forums instead of an error state.

4. **Call History** — `currentUserId = 'current-user'` hardcoded instead of reading from
   the auth store. This means call history filtering shows incorrect results.

## Tasks

1. **Wire Notifications screen**
   - Import and use `useNotificationStore` or notification service
   - Replace `getMockNotifications()` with `notificationService.getNotifications()`
   - Wire pull-to-refresh to store refetch
   - Add loading skeleton, error state, empty state
   - Delete `getMockNotifications` function

2. **Wire User Wall screen**
   - Create or use existing user activity API call (`GET /api/v1/users/:id/wall`)
   - Replace `MOCK_POSTS` with API response data
   - Add loading, error, and empty states
   - Delete `MOCK_POSTS` constant

3. **Fix Forum List error handling**
   - Remove `getMockForums()` from catch block
   - Show error UI with retry button on API failure
   - Optionally: use cached/stale data from store if available
   - Delete `getMockForums` function

4. **Fix Call History currentUserId**
   - Import `useAuthStore` (or equivalent auth hook)
   - Replace `const currentUserId = 'current-user'` with `const currentUserId = useAuthStore().user?.id`
   - Add guard for unauthenticated state
