## Context

Mobile store facades were created as a clean abstraction layer but many return stub values:

- `useCommunityFacade().forums` → `[] as unknown[]` with TODO "wire to forum store"
- `useMarketplaceFacade().balance` → hardcoded `0` with TODO "wire to gamification store coins"
- `useUIFacade()` → sidebar, modal methods are all no-ops

Voice recording is completely stubbed — the hook returns inert values and the component export
is commented out. `expo-av` is likely needed for the actual implementation.

The forums feature directory is the only one without a `hooks/` subdirectory.

Voice call screen simulates audio levels with `Math.random()` instead of reading WebRTC stats.

## Tasks

1. **Wire community facade to forum store**
   - Import forum store
   - Replace `[] as unknown[]` with `forumStore.getState().forums`
   - Ensure reactive updates (use zustand selector or equivalent)

2. **Wire marketplace facade to gamification store**
   - Import gamification store
   - Replace hardcoded `0` with `gamificationStore.getState().coins`
   - Ensure balance updates reactively

3. **Fix UI facade**
   - Evaluate which UI facade methods make sense on mobile:
     - `isSidebarOpen` / `setSidebarOpen` — likely remove (web-only concept)
     - `activeModal` / `showModal` / `hideModal` — wire to React Navigation modal or state
   - Remove methods that don't apply to mobile with a comment

4. **Implement voice recording hook**
   - Add `expo-av` to mobile dependencies
   - Implement `useVoiceRecording()`:
     - `startRecording`: request permission, create `Audio.Recording`, `.prepareToRecordAsync()`,
       `.startAsync()`, track duration with interval
     - `stopRecording`: `.stopAndUnloadAsync()`, get URI, return for message send
     - Cleanup on unmount
   - Handle permission denied gracefully

5. **Create VoiceMessageRecorder component**
   - UI: idle state (mic button), recording state (duration + stop), preview state (play + send)
   - Uses `useVoiceRecording()` hook
   - Uncomment export in `components/index.ts`

6. **Create forum hooks directory**
   - Create `features/forums/hooks/index.ts`
   - Create `useForumList.ts` — wraps forum store/service for list operations
   - Create `useForumDetail.ts` — wraps forum store/service for single forum operations

7. **Fix voice call audio levels**
   - Replace `Math.random()` with `RTCRtpReceiver.getStats()` audio level reading
   - Or use WebRTC `getStats()` → audioLevel from `RTPReceiverStatsReport`
   - Fallback to 0 if stats unavailable (not random)
