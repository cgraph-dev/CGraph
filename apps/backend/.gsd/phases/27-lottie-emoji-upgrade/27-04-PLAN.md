<objective>
Add Lottie-based avatar borders, chat effects, and animation upgrades to the gamification system.

Purpose: CGraph's gamification has 150+ avatar borders using CSS animations (pulse, rotate, shimmer,
particles, etc.). This plan adds Lottie as a new animation type: circular Lottie animations that
wrap around avatar images. This unlocks far richer border effects (fire, lightning, aurora, etc.)
that CSS can't achieve. Also adds Lottie-based chat effects (send/receive animations) and creates
the folder structure for storing Lottie border assets. Existing CSS borders are preserved.

Output: LottieBorderRenderer component, Lottie border type in avatar_border schema, priv/data/
lottie_borders/ directory, updated cosmetics controller, border preview, chat effects. </objective>

  Create `LottieBorderRenderer` — a circular Lottie animation that wraps an avatar:

```tsx
export function LottieBorderRenderer({
  lottieUrl,
  avatarSize, // e.g. 48, 64, 96, 128
  borderWidth = 3,
  lottieConfig, // {loop, speed, segment}
  children, // avatar image as children
}: LottieBorderProps) {
  const containerRef = useRef<HTMLDivElement>(null);
  const totalSize = avatarSize + borderWidth * 2;

  // The Lottie animation is rendered in a square container around the avatar
  // Using CSS clip-path: circle() to mask it into a ring shape
  // Inner mask cuts out the avatar area, outer mask clips to circle

  return (
    <div className="relative" style={{ width: totalSize, height: totalSize }}>
      {/* Lottie animation layer (behind avatar) */}
      <div
        ref={containerRef}
        className="absolute inset-0"
        style={{
          clipPath: `circle(${totalSize / 2}px at center)`,
          WebkitMaskImage: `radial-gradient(circle ${avatarSize / 2}px at center, transparent ${avatarSize / 2 - 1}px, black ${avatarSize / 2}px)`,
        }}
      />
      {/* Avatar centered on top */}
      <div
        className="absolute"
        style={{
          top: borderWidth,
          left: borderWidth,
          width: avatarSize,
          height: avatarSize,
          borderRadius: '50%',
          overflow: 'hidden',
        }}
      >
        {children}
      </div>
    </div>
  );
}
```

**Performance constraints:**

- Use canvas renderer for avatars < 64px (cheaper than SVG for small sizes)
- Use SVG renderer for 64px+ (better quality at larger sizes)
- Max 2 concurrent Lottie border animations in viewport (use IntersectionObserver)
- Border animations auto-pause when scrolled out of view
- Respect `prefers-reduced-motion` — show static border color instead </action> <verify>Render
  avatar with Lottie border. Animation plays as ring around avatar. Scroll out of view → animation
  pauses. Reduced motion → static border.</verify> <done>LottieBorderRenderer renders circular
  Lottie animations around avatars with performance guardrails.</done> </task>
