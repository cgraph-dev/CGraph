<objective>
Build the backend Lottie animation infrastructure — schemas, context, manifest, caching, and API.

Purpose: CGraph is upgrading from static Unicode emoji to animated Noto Emoji (Lottie). This plan
creates the foundational backend layer: a new `CGraph.Animations` context with Lottie asset schemas,
a manifest mapping Unicode codepoints to Google Fonts CDN URLs
(`https://fonts.gstatic.com/s/e/notoemoji/latest/{codepoint}/lottie.json`), Cachex caching, and REST
API endpoints. It also adds `lottie_url` and `animation_format` fields to existing custom emoji
schemas so the entire emoji system is Lottie-aware.

Output: New `CGraph.Animations` context, Lottie schema + migrations, manifest module, cache layer,
Lottie REST controller, updated custom emoji schemas, full test coverage. </objective>

```elixir
defmodule CGraph.Animations.Lottie do
  use Ecto.Schema
  import Ecto.Changeset

  @primary_key {:id, :binary_id, autogenerate: true}
  @foreign_key_type :binary_id
  schema "lottie_assets" do
    field :codepoint, :string        # e.g. "1f600" — hex Unicode codepoint
    field :emoji, :string             # e.g. "😀" — the actual emoji character
    field :name, :string              # e.g. "grinning face"
    field :category, :string          # e.g. "Smileys & Emotion"
    field :subcategory, :string       # e.g. "face-smiling"
    field :keywords, {:array, :string}
    field :lottie_url, :string        # Full CDN URL to lottie.json
    field :webp_url, :string          # Full CDN URL to 512.webp (static fallback)
    field :gif_url, :string           # Full CDN URL to 512.gif (animated fallback)
    field :file_size, :integer        # Lottie JSON file size in bytes (for preload budgets)
    field :duration_ms, :integer      # Animation duration in ms
    field :asset_type, :string        # "emoji" | "border" | "effect" | "sticker"
    field :source, :string            # "noto" | "custom" | "premium"
    field :is_active, :boolean, default: true
    timestamps(type: :utc_datetime)
  end
end
```

```
https://fonts.gstatic.com/s/e/notoemoji/latest/{codepoint}/512.webp     (static WebP fallback)

```

**Implementation:**

- Load `priv/data/emoji_unicode16.json` at compile time (same as `Emoji` module)
- For each emoji, compute codepoint hex from the emoji character
- Build compile-time map: `%{"1f600" => %{emoji: "😀", name: "grinning face", ...}}`
- Functions:
  - `get_url(codepoint, format \\ :lottie)` — returns CDN URL for given format
  - `has_animation?(codepoint)` — true if codepoint is in the Noto animated set
  - `all_animated_codepoints()` — returns list of all animated codepoint hex strings
  - `cdn_base_url()` — configurable via `config :cgraph, :lottie_cdn_base_url`
  - `emoji_to_codepoint(emoji_string)` — converts "😀" to "1f600"
  - `codepoint_to_emoji(hex)` — converts "1f600" to "😀"
  - `enrich_emoji(emoji_map)` — adds `lottie_url`, `webp_url`, `gif_url` to an emoji map

**Config:** Add to `config/config.exs`:

```elixir
config :cgraph, :lottie_cdn_base_url,
  "https://fonts.gstatic.com/s/e/notoemoji/latest"
```

```elixir
defmodule CGraph.Animations.LottieCache do
  @cache_name :lottie_cache
  @ttl :timer.hours(24)

  def get_or_fetch(codepoint, fetch_fn) do
    case Cachex.get(@cache_name, codepoint) do
      {:ok, nil} ->
        result = fetch_fn.()
        Cachex.put(@cache_name, codepoint, result, ttl: @ttl)
        result
      {:ok, cached} -> cached
    end
  end

  def invalidate(codepoint), do: Cachex.del(@cache_name, codepoint)
  def warm_cache(entries), do: Enum.each(entries, fn {k, v} -> Cachex.put(@cache_name, k, v, ttl: @ttl) end)
  def stats, do: Cachex.stats(@cache_name)
end
```

Register `:lottie_cache` in `CGraph.CacheSupervisor` (`lib/cgraph/supervisors/cache_supervisor.ex`)
alongside the existing `:cgraph_cache`, `:session_cache`, `:token_cache`, and
`:two_factor_challenges` caches. </action>
<verify>`CGraph.Animations.LottieCache.get_or_fetch("1f600", fn -> :test end)` returns `:test`.
