<objective>
Harden WebSocket reconnection with session resumption and circuit breaker safeguards.

Purpose: The `@cgraph/socket` package already has `exponentialBackoffWithJitter()` ‚Äî a well-tested
backoff function with property-based tests. The PhoenixClient uses it for `reconnectAfterMs`.
However, three gaps remain:

1. No session resumption: `sessionId` and `lastSequence` are stored on disconnect but NOT sent as
   reconnect params, so clients restart from scratch on every reconnect.
2. No circuit breaker: PhoenixClient will reconnect forever, draining battery on mobile.
3. Channel rejoins use a simple linear backoff instead of jitter.

Output: PhoenixClient reconnects with session resumption, stops after max attempts, and channel
rejoins use the jitter function. All existing tests still pass. </objective>
