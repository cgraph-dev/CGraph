# CGraph v0.7.29 Release Notes

**Release Date:** January 9, 2026

This release focuses on architecture improvements, code quality, and developer experience. It fixes a critical React hooks violation that was causing mobile app crashes and introduces patterns and tooling to prevent similar issues in the future.

---

## 🐛 Bug Fixes

### React Hooks Violation Fixed

The mobile app was experiencing the error:
```
Error: Rendered more hooks than during the previous render
```

**Root Cause:** In `ConversationScreen.tsx`, a `useCallback` hook was defined after an early return statement (`if (isLoading) return <LoadingView />`). React requires that all hooks are called unconditionally and in the same order on every render.

**Fix:** Moved all hooks before any conditional returns. The component now properly follows React's Rules of Hooks.

**Prevention:** Added ESLint configuration with `react-hooks/rules-of-hooks: error` to catch these violations at build time.

## ✨ New Features

### Extracted Conversation Components

The massive 4,945-line `ConversationScreen.tsx` has been refactored into focused, memoized components. This improves:

- **Maintainability** — Each component has a single responsibility
- **Testability** — Components can be tested in isolation
- **Performance** — Memoized components reduce unnecessary re-renders
- **Reusability** — Components can be used in other screens

**New Components:**

| Component | Description |
|-----------|-------------|
| `MessageActionsMenu` | Discord/Telegram-style action sheet for message options |
| `ReactionPickerModal` | Full emoji picker with categories (Smileys, Gestures, Hearts, Symbols) |
| `EmptyConversation` | Empty state with animated wave and conversation starters |
| `AttachmentPreviewModal` | Preview images/videos before sending |
| `ImageViewerModal` | Full-screen image gallery with swipe navigation |
| `VideoPlayerModal` | Full-screen video player with expo-video |
| `TypingIndicator` | Animated bouncing dots for typing status |
| `AttachmentPicker` | Bottom sheet for camera/gallery/file selection |
| `MessageInput` | Full-featured message composition input |
| `AnimatedMessageWrapper` | Smooth entrance animations for messages |
| `AnimatedReactionBubble` | Tap animation for reaction bubbles |

**Usage:**
```tsx
import { MessageActionsMenu, ReactionPickerModal } from '@/components/conversation';
```

### Standardized API Response Types

New TypeScript types for consistent API response handling:

```typescript
import { ApiResponse, PaginatedResponse, isSuccessResponse } from '@/types';

// Type-safe API response handling
const response: ApiResponse<User> = await api.getUser(id);

if (isSuccessResponse(response)) {
  // response.data is guaranteed to exist and be User type
  console.log(response.data.username);
}
```

**Available Types:**
- `ApiResponse<T>` — Single item response wrapper
- `PaginatedResponse<T>` — List with pagination metadata
- `ApiError` — Structured error information
- `PaginationMeta` — Page/cursor information

### React Profiler Monitoring

Development utility for measuring render performance:

```tsx
import { ProfilerWrapper } from '@/components/dev';

// Wrap components to monitor performance
<ProfilerWrapper id="MessageList" thresholdMs={16}>
  <MessageList messages={messages} />
</ProfilerWrapper>
```

**Features:**
- Logs slow renders (configurable threshold)
- Stores metrics in memory for analysis
- Summary statistics via `getProfilerStats()`
- HOC version: `withProfiler(Component, 'id')`
- Automatically disabled in production

## 🏗️ Architecture Improvements

### Backend Context Refactoring

The `Cgraph.Messaging` context (837 lines) has been split into focused sub-contexts:

| Sub-Context | Responsibility |
|-------------|----------------|
| `Conversations` | Conversation CRUD, participant management |
| `Messages` | Message CRUD, pinning, editing |
| `Reactions` | Adding/removing reactions |
| `ReadReceipts` | Read status and delivery tracking |
| `Search` | Full-text message search |

**Backwards Compatible:** The main `Cgraph.Messaging` module delegates to sub-contexts, so existing code continues to work.

```elixir
# Old (still works)
Cgraph.Messaging.create_message(user, conversation, attrs)

# New (also works, more specific)
Cgraph.Messaging.Messages.create_message(user, conversation, attrs)
```

### Zustand Slices Pattern

New slice creators for the chat store enable better code organization:

```typescript
import { createConversationsSlice, createMessagesSlice } from '@/stores/slices';

// Combine slices into a single store
const useChatStore = create<ChatStore>()((...args) => ({
  ...createConversationsSlice(...args),
  ...createMessagesSlice(...args),
  ...createTypingSlice(...args),
  ...createReactionsSlice(...args),
}));
```

**Benefits:**
- Separation of concerns
- Easier testing
- Better TypeScript inference
- Incremental migration path

## 🛠️ Developer Experience

### ESLint Configuration for Mobile

The mobile app now has a comprehensive ESLint configuration:

```javascript
// eslint.config.js highlights
rules: {
  'react-hooks/rules-of-hooks': 'error',  // Catch hook violations
  'react-hooks/exhaustive-deps': 'warn',  // Prevent stale closures
  '@typescript-eslint/no-unused-vars': 'warn',
  'no-console': ['warn', { allow: ['warn', 'error', 'info'] }],
}
```

**Run linting:**
```bash
cd apps/mobile
pnpm lint
```
