# CGraph v0.9.0 Release Notes

**Release Date**: January 2025

## Overview

CGraph v0.9.0 is a major security and real-time features release that introduces Double Ratchet end-to-end encryption for forward secrecy, enhanced authentication security with HTTP-only cookies, and complete online presence tracking.

## 🔐 Security Improvements

### Double Ratchet E2EE (Signal Protocol)

The encryption system has been upgraded from X3DH-only to full Signal Protocol implementation:

- **Forward Secrecy**: Past messages remain secure even if keys are compromised
- **Break-in Recovery**: Future messages become secure after compromise
- **Per-Message Keys**: Each message uses a unique encryption key that is immediately discarded
- **Out-of-Order Handling**: Messages can be received and decrypted out of order
- **Session Persistence**: Ratchet state is stored in IndexedDB for durability

**New Files:**
- `apps/web/src/lib/crypto/sessionManager.ts` - Session management with Double Ratchet
- `apps/web/src/lib/crypto/doubleRatchet.ts` - Core Double Ratchet implementation

**New Store Methods:**
```typescript
const { 
  encryptWithRatchet, 
  decryptWithRatchet, 
  hasRatchetSession,
  destroyRatchetSession,
  getRatchetSessionStats 
} = useE2EEStore();

// Encrypt with forward secrecy
const secureMessage = await encryptWithRatchet(recipientId, "Hello!");

// Decrypt with session auto-establishment
const plaintext = await decryptWithRatchet(message, senderIdentityKey);
```

### HTTP-Only Cookie Authentication

Authentication tokens are now stored in HTTP-only cookies instead of localStorage:

- **XSS Protection**: Tokens cannot be accessed by JavaScript
- **Automatic Handling**: Cookies are sent automatically with requests
- **CSRF Protection**: Combined with SameSite=Lax cookies

**Migration Notes:**
- Frontend API calls require `withCredentials: true` (already configured)
- WebSocket connections use a separate session token from sessionStorage
- The `tokenStorage` helper now uses sessionStorage for WebSocket auth only

## 🟢 Online Presence Tracking

Complete friend online status tracking is now available:

### Socket Integration

```typescript
import { socketManager } from '@/lib/socket';

// Join the presence lobby on login
socketManager.joinPresenceLobby();

// Check if a friend is online
const isOnline = socketManager.isFriendOnline(userId);

// Get all online friends
const onlineFriends = socketManager.getOnlineFriends();
```

### React Hooks

```typescript
import { usePresence, useUserOnline } from '@/hooks';

// Track all online friends
function FriendList() {
  const { onlineFriends, isConnected, onlineCount } = usePresence();
  // ...
}

// Track a specific user's status
function UserStatus({ userId }) {
  const isOnline = useUserOnline(userId);
  return <Badge color={isOnline ? 'green' : 'gray'} />;
}
```

### Events

The presence system broadcasts the following events:
- `friend_online` - Friend came online
- `friend_offline` - Friend went offline  
- `status_update` - Friend changed status (away, dnd, etc.)

## 🔄 Migration Guide

### From v0.8.x

1. **E2EE Sessions**: Existing X3DH sessions continue to work. New conversations will use Double Ratchet automatically.

2. **Token Storage**: No action needed. The new HTTP-only cookie system handles authentication automatically.

3. **Presence**: Add `socketManager.joinPresenceLobby()` after socket connection if you want real-time friend status.

### API Changes

- `useE2EEStore` now has `useDoubleRatchet` state (default: `true`)
- New methods: `encryptWithRatchet`, `decryptWithRatchet`, `hasRatchetSession`
- Legacy `encryptMessage`/`decryptMessage` still work for backwards compatibility
