# CGraph v0.7.32 Release Notes

**Release Date**: January 9, 2026  
**Type**: Major Feature Release

This release introduces enterprise-grade scalability features including distributed search, WebRTC
calling, and intelligent rate limiting. These are the largest infrastructure additions since the
initial release.

---

## Highlights

- **WebRTC Voice/Video Calling** — Peer-to-peer calls with up to 10 participants
- **Meilisearch Integration** — Blazing fast full-text search with typo tolerance
- **Distributed Rate Limiting** — Redis-backed rate limiting with Lua atomicity
- **Sampled Presence** — Scalable presence for channels with millions of users

---

## New Features

### WebRTC Voice/Video Calling

Complete WebRTC implementation for real-time voice and video calls.

| Feature         | Implementation                |
| --------------- | ----------------------------- |
| 1:1 Voice Calls | Peer-to-peer via DTLS-SRTP    |
| 1:1 Video Calls | Peer-to-peer with H.264/VP8   |
| Group Calls     | Up to 10 participants         |
| Screen Sharing  | Full screen or window capture |
| Signaling       | Phoenix Channels              |

**Modules Added:**

- `Cgraph.WebRTC` — Room management and ICE configuration
- `Cgraph.WebRTC.Room` — Room state with participant tracking
- `Cgraph.WebRTC.Participant` — Participant media state
- `CgraphWeb.CallChannel` — Phoenix Channel for signaling

**Channel Events:**

```elixir
# Signaling
"signal:offer"        # SDP offer
"signal:answer"       # SDP answer
"signal:ice_candidate" # ICE candidate exchange

# Media control
"media:mute" / "media:unmute"
"media:video_on" / "media:video_off"
"media:screen_share"

# Call management
"call:ring"
"call:leave"
"call:end"
```

### Meilisearch Integration

Replaced PostgreSQL full-text search with Meilisearch for production-grade search.

| Capability         | Details                                 |
| ------------------ | --------------------------------------- |
| Typo tolerance     | Handles misspellings automatically      |
| Faceted search     | Filter by date, channel, user           |
| Sub-50ms response  | For most queries                        |
| Automatic fallback | PostgreSQL when Meilisearch unavailable |

**Modules Added:**

- `Cgraph.Search.SearchEngine` — Meilisearch client with retry logic
- `Cgraph.Search.Indexer` — Background document indexing

**Usage:**

```elixir
# Automatic - Search context tries Meilisearch first
Search.search_users("john doe", limit: 20)
Search.search_messages("project update", channel_id, current_user)

# Manual reindexing
Search.Indexer.full_reindex(:users)
Search.Indexer.full_reindex(:messages)
```

### Distributed Rate Limiting

Redis-backed rate limiting with Lua scripts for atomic operations.

| Feature           | Implementation                      |
| ----------------- | ----------------------------------- |
| Sliding window    | Precise request counting            |
| Atomic operations | Lua scripts prevent race conditions |
| Tier awareness    | Different limits per subscription   |
| Multi-region      | Shared state across nodes           |

**Modules Added:**

- `Cgraph.RateLimiter.Distributed` — Redis-backed rate limiter

**Configuration:**

```elixir
# Rate limits by tier
config :cgraph, Cgraph.RateLimiter.Distributed,
  default_limits: %{
    free: {60, 60_000},      # 60 requests per minute
    pro: {300, 60_000},      # 300 requests per minute
    enterprise: {1000, 60_000}
  }
```

### Sampled Presence

HyperLogLog-based presence for massive channels.

| Channel Size | Sample Rate | Memory |
| ------------ | ----------- | ------ |
| <100         | 100% (full) | ~10KB  |
| 100-1K       | 50%         | ~5KB   |
| 1K-10K       | 10%         | ~1KB   |
| 10K-100K     | 1%          | ~12KB  |
| >100K        | 0.1%        | ~12KB  |

**Modules Added:**

- `Cgraph.Presence.Sampled` — HyperLogLog presence approximation

**Usage:**

```elixir
# Get approximate online count in O(1)
{:ok, 45_123} = Presence.Sampled.approximate_count(channel_id)

# Get full summary
summary = Presence.Sampled.get_summary(channel_id)
# => %{total: 1_234_567, online: 45_123, typing: 23, approximate: true}
```

## Architecture Changes

### Supervision Tree

Added new children to the application supervisor:

```elixir
children = [
  # ... existing services
  Cgraph.WebRTC,                  # Call room management
  Cgraph.Presence.Sampled,        # Large channel presence
  Cgraph.RateLimiter.Distributed, # Redis-backed rate limiting
  Cgraph.Search.Indexer,          # Background search indexing
]
```

### Channel Routing

Added call channel to UserSocket:

```elixir
# user_socket.ex
channel "call:*", CgraphWeb.CallChannel
```

### Search Context

Rewrote `Cgraph.Search` to use Meilisearch with automatic fallback:

```elixir
def search_users(query, opts \\ []) do
  case SearchEngine.search(:users, query, opts) do
    {:ok, results} -> process_meilisearch_results(results)
    {:error, _} -> search_users_postgres(query, opts)  # Fallback
  end
end
```

### Rate Limiter Plug

Updated `RateLimiterV2` plug to use distributed backend:

```elixir
def do_rate_limit_check(key, tier) do
  scope = tier_to_scope(tier)
  DistributedRateLimiter.check(key, scope, tier)
end
```

## Migration Guide

### From v0.7.31

1. **Update dependencies:**

   ```bash
   mix deps.get
   ```

2. **Configure Meilisearch (optional):**
   - Without Meilisearch: Search falls back to PostgreSQL automatically
   - With Meilisearch: Set `MEILISEARCH_URL` and run initial indexing

3. **Configure TURN servers (for calling):**
   - 1:1 calls work with public STUN servers
   - For NAT traversal in enterprise environments, configure TURN

4. **No database migrations required**
