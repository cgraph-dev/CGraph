# CGraph v0.9.5 Release Notes

**Release Date:** January 2026  
**Type:** Security & Infrastructure Release

---

## 🔐 Security Hardening

### Critical Fixes

1. **Presence Privacy Protection**
   - Fixed broadcast leak where presence updates were sent globally
   - User presence now scoped to per-user PubSub topics
   - Only contacts receive presence updates

2. **Stripe Webhook Verification**
   - Fixed configuration key mismatch for webhook signing secret
   - Properly validates all incoming Stripe webhooks

3. **IP Spoofing Prevention**
   - Added trusted proxy CIDR enforcement
   - Only accepts X-Forwarded-For from Cloudflare and private networks
   - Configurable CIDR allowlist: `103.21.244.0/22`, `103.22.200.0/22`, etc.

4. **MIME Type Spoofing Prevention**
   - Added magic byte validation for uploaded files
   - Prevents content-type header manipulation attacks
   - Validates file signatures match declared MIME types

5. **Client Presence Synchronization**
   - Web: Added `presence_update` and `contact_presence` event handlers
   - Mobile: Auto-joins user channel for presence events
   - Consistent presence state across all clients

### Enhanced Security Features

- HTTP-only session cookies with Secure and SameSite flags
- Trusted proxy validation in RateLimiterV2 plug
- Distributed Redis-backed rate limiting

## 🔧 Configuration Changes

### New Environment Variables

```bash
STRIPE_SECRET_KEY     - Stripe API secret key
STRIPE_WEBHOOK_SECRET - Stripe webhook signing secret (was: signing_secret)
STRIPE_PRICE_IDS      - JSON map of tier names to Stripe price IDs
```

### Backend Config

```elixir
# config/runtime.exs
config :cgraph, :stripe,
  secret_key: System.get_env("STRIPE_SECRET_KEY"),
  signing_secret: System.get_env("STRIPE_WEBHOOK_SECRET")  # Fixed key name
```
