## 🔐 Signal Protocol Double Ratchet

**The gold standard in end-to-end encryption**, now in CGraph. We've implemented the full Signal Protocol Double Ratchet algorithm — the same cryptographic foundation that secures Signal, WhatsApp, and other industry-leading messengers.

### What's Included

- **X3DH Key Agreement** — Extended Triple Diffie-Hellman for secure session establishment
- **ECDH P-384** — NIST-approved elliptic curve (stronger than P-256)
- **AES-256-GCM** — Authenticated encryption with associated data
- **HKDF** — HMAC-based Key Derivation Function
- **Forward Secrecy** — Past messages stay secure even if keys are compromised
- **Break-in Recovery** — Session automatically heals after key compromise
- **Out-of-Order Messages** — Handles delayed/reordered messages securely
- **Post-Quantum Ready** — Placeholder for CRYSTALS-Kyber upgrade path

### Security Properties

| Property | Implementation |
|----------|----------------|
| Forward Secrecy | New DH key pair each exchange |
| Break-in Recovery | Automatic key refresh |
| Key Erasure | Message keys deleted after use |
| Constant-time Operations | Timing attack protection |
| Secure Random | Web Crypto API CSPRNG |

### Quick Start

```typescript
import { DoubleRatchetEngine } from '@/lib/crypto/doubleRatchet';

const alice = new DoubleRatchetEngine({ enableAuditLog: true });
await alice.initializeAlice(sharedSecret, bobPublicKey);

const encrypted = await alice.encryptMessage(
  new TextEncoder().encode('Hello, Bob!')
);
```

## 🧠 AI Message Intelligence Engine

**Enterprise-grade AI features** for smarter conversations. All processing can run locally for maximum privacy.

### Features

| Feature | Description |
|---------|-------------|
| **Smart Replies** | Context-aware reply suggestions with confidence scores |
| **Sentiment Analysis** | 8 emotions (joy, sadness, anger, fear, surprise, disgust, trust, anticipation) |
| **Content Moderation** | Spam, scam, harassment, phishing detection |
| **Language Detection** | 20+ languages with script detection |
| **Topic Extraction** | Automatic categorization and trending topics |
| **Conversation Insights** | Engagement metrics, response patterns, summaries |

### Privacy-First Design

- **Local ML by default** — No data leaves the device
- **Optional cloud AI** — Enhanced accuracy when privacy allows
- **No message storage** — Analysis is ephemeral
- **GDPR/CCPA compliant** — Full data protection compliance

### Quick Start

```typescript
import { AIMessageEngine } from '@/lib/ai/AIMessageEngine';

const ai = new AIMessageEngine({ localMLOnly: true });

// Smart replies
const replies = await ai.generateSmartReplies(message);

// Sentiment analysis
const sentiment = await ai.analyzeSentiment('I love this!');
// { overall: 'positive', emotions: { joy: 0.88, ... } }

// Content moderation
const result = await ai.moderateContent(suspiciousText);
// { isApproved: false, flags: [...], suggestedAction: 'block' }
```

## 🔊 Spatial Audio Engine

**VR/AR-ready 3D positional audio** with HRTF support. Perfect for immersive voice chat experiences.

### Features

| Feature | Description |
|---------|-------------|
| **3D Positioning** | Full x, y, z spatial placement |
| **HRTF** | Head-Related Transfer Function for realistic spatialization |
| **Audio Zones** | Configurable reverb environments (room, hall, cave, outdoor) |
| **Voice Activity Detection** | Real-time speech detection with callbacks |
| **Noise Cancellation** | Integration hooks for Krisp/RNNoise |
| **Attenuation Curves** | Inverse, linear, exponential distance models |

### Quick Start

```typescript
import { SpatialAudioEngine, SpatialAudioRoom } from '@/lib/audio/SpatialAudioEngine';

const engine = new SpatialAudioEngine({ hrtfEnabled: true });

// Add audio source at position
engine.createSource({
  position: { x: 2, y: 0, z: -3 },
  stream: userAudioStream,
});

// Update listener position (e.g., from VR headset)
engine.updateListenerPosition({ x: 0, y: 1.7, z: 0 });
engine.updateListenerOrientation(headsetForward, headsetUp);
```
