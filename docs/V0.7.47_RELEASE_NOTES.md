# CGraph v0.7.47 Release Notes

**Release Date:** January 11, 2026  
**Type:** Security Hardening Release

---

## Summary

This release addresses multiple critical security vulnerabilities discovered during a comprehensive security audit. It includes fixes for race conditions, brute force protection, safe parameter parsing, and adds database indexes for improved performance at scale.

---

## 🚨 Critical Security Fixes

### 1. Race Condition in Coin Spending (CVE-LOW-001)

**Severity:** CRITICAL  
**Impact:** Double-spending of coins possible through concurrent requests

**Description:**
The `Gamification.spend_coins/2` function had a time-of-check-to-time-of-use (TOCTOU) vulnerability where the balance check occurred before the transaction lock, allowing concurrent requests to bypass balance verification.

**Fix:**
Added `SELECT FOR UPDATE` row-level locking within the transaction:

```elixir
Repo.transaction(fn ->
  locked_user = 
    from(u in User, where: u.id == ^user.id, lock: "FOR UPDATE")
    |> Repo.one!()
  
  if locked_user.coins < amount do
    Repo.rollback(:insufficient_funds)
  end
  # ... proceed with deduction
end)
```

**File:** `apps/backend/lib/cgraph/gamification.ex`

---

### 2. Premium Subscribe Demo Bypass (CVE-LOW-002)

**Severity:** CRITICAL  
**Impact:** Free premium access in production through demo parameter

**Description:**
The premium subscription endpoint accepted a `demo=true` parameter that bypassed payment verification without environment checks.

**Fix:**
Demo mode now requires explicit environment variable AND non-production environment:

```elixir
def can_use_demo_mode? do
  System.get_env("PREMIUM_DEMO_MODE") == "true" &&
    Mix.env() != :prod
end
```

**File:** `apps/backend/lib/cgraph_web/controllers/premium_controller.ex`

---

### 3. 2FA Brute Force Vulnerability (CVE-LOW-003)

**Severity:** CRITICAL  
**Impact:** Unlimited 2FA code guessing attempts

**Description:**
Two-factor authentication endpoints had no rate limiting, allowing attackers to brute force 6-digit TOTP codes (1,000,000 combinations).

**Fix:**
Created `TwoFactorRateLimiter` plug with progressive lockout:

| Threshold | Action |
|-----------|--------|
| 5 failures in 5 minutes | 15-minute lockout |
| 3 lockout periods | 24-hour extended lockout |
| Successful verification | All counters reset |

**File:** `apps/backend/lib/cgraph_web/plugs/two_factor_rate_limiter.ex`

---

### 4. Uncaught Integer Parsing Exceptions (CVE-LOW-004)

**Severity:** CRITICAL  
**Impact:** Server crash on malformed pagination parameters

**Description:**
20+ controllers used unsafe `String.to_integer()` on user input without try/catch, allowing attackers to crash endpoints with payloads like `?page=abc`.

**Fix:**
Created `ParamParser` helper with safe parsing functions:

```elixir
# Before (vulnerable)
page = Map.get(params, "page", "1") |> String.to_integer()

# After (safe)
page = parse_int(params["page"], 1, min: 1)
```

**File:** `apps/backend/lib/cgraph_web/helpers/param_parser.ex`
